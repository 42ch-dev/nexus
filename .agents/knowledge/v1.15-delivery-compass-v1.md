### Locked architecture decisions (v3)

Architecture audit confirmed: all remaining daemon local API routes are **infrastructure** required even in preset-driven model. No API routes need adding or removing beyond what T6 already cleaned up. Key decisions from [v1.15-daemon-api-workspace-write-architecture.md](v1.15-daemon-api-workspace-write-architecture.md):

| ID | Decision | Summary |
|----|----------|---------|
| D1 | Workspace file writes by ACP agent | All user-visible workspace file writes go through ACP agent `fs/write_text_file` via daemon tool execution API. CLI and daemon do not write business files directly. |
| D2 | `context_update` hooks stay core-context-only | Hooks do NOT support direct file I/O. File writes are always ACP agent actions. |
| D3 | `inner_graph` nodes create ACP sessions | Nodes create ACP sessions; agent writes via tool API (existing pattern). |
| D4 | Research output path defined in preset prompt | Path strategy lives in preset prompt templates, not hardcoded in daemon. |
| D5 | `Stories/`/`References/` created by preset, not init | Workspace init is skeleton-free; directories created on-demand by ACP agent (T8.5). |
| D6 | No new workspace write API needed | ACP tool execution API already covers all file write needs. |
| D7 | Daemon workspace init becomes minimal | Only creates `.nexus42/` state; no business directory scaffolding. |

**Module boundary summary**: `nexus-orchestration` = preset engine logic (no direct I/O); `nexus42d` = HTTP API + workspace state + ACP tool execution; `nexus42` = CLI UX + workspace init (minimal). Workspace writes flow: `preset inner_graph → ACP session → agent tool request → daemon ACP tool execute → fs/write_text_file → audit log`.

### In scope (frozen)

- Add `crates/nexus-orchestration/embedded-skills/` (versioned with the repo); on `nexus42` **install** and **upgrade**, copy/sync contents to **`$HOME/.nexus42/skills/`** (idempotent; user-local conflicts must surface an explicit operator report; conflicts are non-fatal by default, so skill sync succeeds with conflicts listed in output — see §8).
- `crates/nexus-home-layout` MUST define the canonical helper **`user_skills_dir(home) -> PathBuf`** returning **`$HOME/.nexus42/skills/`**. All user-local skill path references MUST use this helper instead of hardcoded strings.
- V1.15 MVP ships with one seed embedded skill: **`novel-writing-assistant`**. Additional embedded skills can be added in later versions.
- **Preset roles**:
  - Remove `recommended_models` (ACP cannot switch models per session policy).
  - Add **`recommended_skills`** (or equivalent schema field) validated to paths readable by the ACP agent: either under the project skill root or under `$HOME/.nexus42/skills/`.
  - On **ACP new session**, inject recommended skills into initial session context.
  - If a skill lives only under `$HOME/.nexus42/skills/`, ensure readability by **symlink** into default project skill location **`{$workspace_dir}/.agents/skills/`** (double **s**) or the configured ACP-readable skill root.
- **Workspace semantics**:
  - **`{$workspace_dir}`** is the real project directory; **`$HOME/.nexus42/...`** is creator/runtime data (operational DB, config, caches) — keep the boundary strict.
  - Workspace init MUST NOT create **`Stories/`** or **`References/`** directories. Only functional directories needed for ACP-readable skills or local state are permitted (for example `.agents/skills/` only when needed). **All** user-visible artifacts are created by **preset policy**, not CLI `init` defaults.
- **novel-writing** preset:
  - Default story output path: **`Stories/<StoryRef>/<chapter-id>.md`**.
  - Association to `world` / platform identifiers lives in **local DB or global creator config**, not inferred solely from directory names.
- **Research flow** (preset-driven, not CLI `research`):
  - Default functional output root: **`{$workspace_dir}/.nexus42/references/<run-id>/`** with at least **`report.md`** and optional **`artifacts/`** subtree.
  - The research preset MUST define a minimum output contract: **`{$workspace_dir}/.nexus42/references/<run-id>/report.md`** as the required report, plus optional **`artifacts/`** subtree for additional findings.
  - A dedicated research preset (for example **`embedded-presets/research`**) SHOULD be created as part of O2 before removing legacy `nexus42 research` commands.
- **Remove CLI command groups** in V1.15 (hard remove, no deprecation window): `nexus42 research *`, `nexus42 manuscript *`, `nexus42 publish *` — replaced by **preset / orchestration** entrypoints.
- **sync module** inside `embedded-presets/novel-writing` (or equivalent preset package): defines how workspace artifacts map to platform publish/sync operations (structured deltas / bundles per existing contracts — implementation follows `v1-spec` + `@42ch/nexus-contracts`).

## 2) V1.15 positioning

V1.15 moves the product from **“CLI verbs for every workflow”** to **“presets own workflows; CLI installs skills and runs orchestration”**, while tightening the **filesystem + ACP** story for long-form writing.

### Core principles

1. **Orchestration is the product surface** for writing/research/publish flows.
2. **ACP-first**: skills are injected for session bootstrapping, not a parallel command universe.
3. **Filesystem discipline**: empty-by-default workspace; artifacts are policy-driven.
4. **Evidence-bound ship**: every theme exits with tests + docs (see §9).
