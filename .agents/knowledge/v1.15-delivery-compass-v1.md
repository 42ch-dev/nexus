### Locked delivery posture

| Field | Value |
|-------|--------|
| **Delivery track** | V1.15 |
| **Rhythm** | Orchestration-first refactor (pre-release; **no backward-compat requirement**) |
| **Primary line** | Theme O1 — **Embedded skills + preset/session integration** |
| **Secondary line** | Theme O2 — **Command surface convergence** (remove `research` / `manuscript` / `publish` CLI groups; strategy-only) |
| **Tertiary line** | Theme O3 — **novel-writing** layout + **sync** module (workspace → platform) |

### In scope (frozen)

- Add `crates/nexus-orchestration/embedded-skills/` (versioned with the repo); on `nexus42` **install** and **upgrade**, copy/sync contents to **`$HOME/.nexus42/skills/`** (idempotent; user-local conflicts must surface explicit errors — see §8).
- **Preset roles**:
  - Remove `recommended_models` (ACP cannot switch models per session policy).
  - Add **`recommended_skills`** (or equivalent schema field) validated to paths readable by the ACP agent: either under the project skill root or under `$HOME/.nexus42/skills/`.
  - On **ACP new session**, inject recommended skills into initial session context.
  - If a skill lives only under `$HOME/.nexus42/skills/`, ensure readability by **symlink** into default project skill location **`{$workspace_dir}/.agents/skills/`** (or the configured ACP-readable skill root).
- **Workspace semantics**:
  - **`{$workspace_dir}`** is the real project directory; **`$HOME/.nexus42/...`** is creator/runtime data (operational DB, config, caches) — keep the boundary strict.
  - Initial workspace: **no fixed `Stories/` or `References/` skeleton**; only minimal functional dirs (e.g. `.agents/skills` as needed). **All** user-visible artifacts are created by **preset policy**, not CLI `init` defaults.
- **novel-writing** preset:
  - Default story output path: **`Stories/<StoryRef>/<chapter-id>.md`**.
  - Association to `world` / platform identifiers lives in **local DB or global creator config**, not inferred solely from directory names.
- **Research flow** (preset-driven, not CLI `research`):
  - Default functional output root: **`{$workspace_dir}/.nexus42/references/<run-id>/`** with at least **`report.md`** and optional **`artifacts/`** subtree.
- **Remove CLI command groups** in V1.15 (hard remove, no deprecation window): `nexus42 research *`, `nexus42 manuscript *`, `nexus42 publish *` — replaced by **preset / orchestration** entrypoints.
- **sync module** inside `embedded-presets/novel-writing` (or equivalent preset package): defines how workspace artifacts map to platform publish/sync operations (structured deltas / bundles per existing contracts — implementation follows `v1-spec` + `@42ch/nexus-contracts`).

### Out of scope (frozen)

- **DF-03** third-party preset registry / signing / publish — backlog XL.
- **DF-12** dual-outbox full merge — any future.
- **DF-13** / **DF-16** (entitlements consumption UI, Stripe) — V2.0+.
- Coordinated **breaking wire** without the public contracts release process (if wire change is needed, it is a **separate** decision — default is avoid).

4. `status.json` + this compass remain consistent when implementation runs.

## 2) V1.15 positioning

V1.15 moves the product from **“CLI verbs for every workflow”** to **“presets own workflows; CLI installs skills and runs orchestration”**, while tightening the **filesystem + ACP** story for long-form writing.

### Core principles

1. **Orchestration is the product surface** for writing/research/publish flows.
2. **ACP-first**: skills are injected for session bootstrapping, not a parallel command universe.
3. **Filesystem discipline**: empty-by-default workspace; artifacts are policy-driven.
4. **Evidence-bound ship**: every theme exits with tests + docs (see §9).
