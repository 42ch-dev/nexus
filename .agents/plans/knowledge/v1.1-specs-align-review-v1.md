## 3. Detailed findings

### 3.1 P0 / contract — Entitlements wire not in JSON Schema SSOT

- **Spec**: `schema/entitlements-wire-v1.md` defines `MeEntitlementsResponseV1` and `OfficialCreatorQuotaResponseV1` (field-level SSOT for `GET /me/entitlements` and `GET /official-creator/quota` bodies).  
- **Strategy**: `codegen-strategy-v1.md` §3 lists entitlements as objects that **may** be added to the public nexus schema tree; §6 **minimal freeze set** item **5** explicitly includes these routes.  
- **Repo**: No `schemas/platform/*entitlement`* (or equivalent) files; platform consumers cannot import generated DTOs for these bodies from `@42ch/nexus-contracts` yet.  
- **Risk**: Platform hand-writes Zod/TS types or duplicates shapes — long-term drift vs v1-spec.  
- **Remediation direction**: Add two JSON Schemas under `schemas/platform/`, `$ref` shared enums from `common.schema.json` where possible, run `pnpm run codegen`, bump versions per `AGENTS.md` if the wire is new to the package.

### 3.2 Documentation SSOT drift

- **[architecture-alignment-review-v1.md](architecture-alignment-review-v1.md) §2.2** — `User` row still states “not in schemas/domain”; **false** as of current tree. **ForkBranch** “partial / forked_from only” is also **false** — schema includes `parent_branch_id`.  
- **[v1.1-overview-v2.md](v1.1-overview-v2.md)** — §2 plan counts, §2.3 “Todo” table (wire sprint, explore parity, publish parity), and §3 open residual summary are **stale** relative to `status.json` (2026-04-11). Readers should prefer `status.json` until overview is refreshed.

### 3.3 Architecture / product — intentional partials

- **SQLite path**: Single global `state.db` under `~/.nexus42/` vs cli-spec workspace-scoped path — acceptable per prior architecture review **if** multi-workspace remains V1.2+; track with ADR or cli-spec amendment when promoting multi-workspace.  
- **Domain vs sync aggregates**: Spec describes full aggregate list; OSS domain layer focuses on world/creator/memory/manuscript graph — sync/outbox/workspace/agent profile remain **contract- and sync-layer-centric**. Call this out in docs to avoid repeated “gap” false positives.

### 3.4 Closed or well-covered areas (sanity check)

- **WorldStatus / MembershipStatus** string enums: addressed in prior alignment waves (see architecture review §1.2).  
- **Platform plans 16–20** wire shapes: covered by sprint + coverage matrix **Done** rows.  
- **Context assembly**, **bundle / conflict** schemas: present under `schemas/platform/` and `schemas/cli-sync/` consistent with v1-spec pointers in `README.md`.

### 3.5 pre-freeze-spec-log.md

- File is **frozen archive**; remaining product nuance (e.g. **W3** creator profile privacy tiers) is reflected in platform execution + Explore card schemas — no additional OSS schema gap identified in this pass beyond §3.1.
