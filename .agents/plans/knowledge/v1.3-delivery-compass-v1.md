## 1) Purpose and SSOT boundary

This document is the implementation-period SSOT for V1.3 planning decisions and execution guardrails for the `nexus` OSS repository, based on confirmed PM brainstorming outcomes and the platform v1.3-spec.

- Authoritative for: V1.3 scope classification, wave decomposition, residual governance strategy, cross-repo dependencies, and handoff rules.
- Not authoritative for: field-level wire contracts and behavior details defined in v1-spec mother documents and ADRs.
- Conflict resolution:
  - If conflict with plan execution status, `status.json` in `{PLAN_DIR}` wins for machine state.
  - If conflict with architecture or contract semantics, v1-spec / ADR source files win.

## 2) Confirmed decisions (locked)

- **V1.3 scope (nexus side)**: Creator Self-Registration CLI (M20-M23 from v1.3-spec §9) + full V1.2 residual governance (35 items).
- **Plan architecture**: Option A — Feature-first + severity-driven residual batches (serial, single track).
- **Residual governance strategy**: Option A — severity-driven (medium first, then low, then nit; accept/fix classification per item).
- **Branch strategy**: `feature/v1.3-creator-register-and-residuals` from `main`; single track, no worktree needed.
- **Non-goals**: Entitlements CLI client, CLI+Platform e2e, OpenAPI work, Stripe integration.
- **Cross-repo dependency**: Nexus CLI `creator register` depends on platform completing M10 (challenge/verify API endpoints) first.

## 5) Residual governance rules

### Accept vs Fix classification

| Criteria | Fix | Accept |
|----------|-----|--------|
| Code correctness bug | ✅ | |
| Security concern | ✅ | |
| Performance issue (measurable) | ✅ | |
| Dead code (clearly unused) | ✅ | |
| Design intent (documented) | | ✅ + comment |
| Known limitation (documented) | | ✅ + comment |
| Deferred to future version | | ✅ + archive |
| Style / naming preference | Case-by-case | Case-by-case |

### Close and archive

- Fixed items: code change + test → close residual in `status.json` → archive in `archived/residuals/`
- Accepted items: code comment or knowledge doc note → close → archive
- All 35 items must reach 0 open by V1.3 Done
