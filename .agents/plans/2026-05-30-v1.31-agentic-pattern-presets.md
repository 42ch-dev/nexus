## 1. Problem Statement

V1.31 P1+P2 de-stub capabilities, but without demonstrator presets there is no proof that **Agentic Design Patterns compose** in the preset engine. P3 delivers two embedded presets exercising reflection, memory-augmentation, and rule-gated checkpoints — without lifting `ConditionalNotYetSupported`.

---

## 2. Goals

### A1 — Preset `reflection-loop`

Location: `crates/nexus-orchestration/embedded-presets/reflection-loop/`

1. States: `draft` → `revise` → `summarize` → `done` (linear; no conditional next).
2. Uses: `acp.prompt`, `judge.llm` (exit_when llm_judge), `context.summarize`.
3. Prompt templates under `prompts/` with Handlebars vars from `preset.input`.
4. `requires_capabilities` lists all deps; passes `preset validate`.

### A2 — Preset `memory-augmented`

Location: `crates/nexus-orchestration/embedded-presets/memory-augmented/`

1. States: `recall` → `generate` → `persist` → `done`.
2. Uses: `creator.read_memory`, `judge.rule` (exit_when rule — e.g. allow proceed), `acp.prompt`, `creator.write_memory`.
3. Input vars: `topic`, optional `memory_keyword`.
4. Passes `preset validate`.

### A3 — Integration tests

1. Loader discovers both presets via `include_dir!` (no Rust registration change beyond auto-discovery if already glob-based).
2. Graph build tests: no validation errors.
3. At least one orchestration integration test per preset using mock worker / test hooks (deterministic where LLM required).

### A4 — Manual runbook (doc in preset README or plan note)

1. Steps: `creator demo seed` → schedule add with preset → orchestrate run → verify memory/judge side effects.
