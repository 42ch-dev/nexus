# Plan: V1.16 system and platform command consolidation

**Plan ID**: `2026-05-12-v1.16-system-platform-topology`
**Theme**: B1 — Command surface refactor
**Milestone**: M2
**Effort**: M
**Working branch**: `feature/v1.16-system-platform-topology` (from `feature/v1.16`)
**Depends on**: `2026-05-12-v1.16-command-surface-contract`

---

## Goal

Create the `platform` top-level command group and extend `system` top-level,
migrating 12+ legacy top-level commands into these two V2 groups.
All legacy commands remain callable via `hide` for backward compatibility.

## Scope

### In scope

1. **Create `platform` top-level** with subcommands:
   - `platform auth` — migrated from `auth` (login/logout/status/profiles)
   - `platform explore` — migrated from `explore`
   - `platform context` — migrated from `context`
   - `platform publish` — new stub for future use

2. **Extend `system` top-level** with subcommands:
   - `system version` — new (CLI version info)
   - `system doctor` — migrated from `doctor`
   - `system completion` — new (shell completion generation)
   - `system config` — migrated from `config` (get/set/unset/path)
   - `system debug` — migrated from `debug` (dump-workspace/replay-delta)
   - `system preset` — already exists as `system preset`
   - `system db` — migrated from `db`
   - `system identity` — migrated from `identity`
   - `system runtime-mode` — migrated from `runtime-mode`

3. **Hide legacy top-level commands** (keep callable):
   - `auth`, `explore`, `context`, `config`, `debug`, `doctor`
   - `db`, `preset`, `identity`, `runtime-mode`

4. **Update contract tests**:
   - Un-ignore `v2_target_platform_subcommands`
   - Un-ignore `v2_target_system_subcommands`
   - Update `current_state_visible_command_groups` (count changes)

### Out of scope

- `creator` extensions (soul/memory/kb — Plan 4)
- `world` → `sync world` migration (Plan 4)
- `schedule` repositioning (Plan 4 or later)
- Removing hidden legacy commands (later cleanup plan)
