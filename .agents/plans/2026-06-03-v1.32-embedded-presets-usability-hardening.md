## 1. Problem Statement

V1.31 shipped embedded presets as binary runtime assets, but stricter validation is expected to expose usability gaps that the old validation surfaces missed. Known areas include likely `kb-extract` inner graph wiring, `memory-augmented` using `exit_when: rule` without an explicit rule expression (`TD-V131-08`), and lack of a durable smoke suite for embedded assets.

P2 repairs shipped embedded presets against the P1 quality gate without expanding orchestration semantics.

---

## 2. Goals

### B1 — Embedded preset smoke suite

1. Add a test that discovers every embedded preset shipped via `include_dir!` / system preset scan path.
2. Validate each embedded preset through the P1 shared validation gate.
3. Keep the smoke suite deterministic and independent of live LLM/provider calls.

### B2 — `kb-extract` wiring repair

1. Inspect `kb-extract` graph/inner graph declarations under the stricter validator.
2. Fix likely inner graph wiring or stale references if validation exposes them.
3. Preserve V1.29/V1.30 queue extraction behavior; do not change capability semantics except to make the preset loadable and runnable.

### B3 — V1.31 demonstrator preset hardening

1. Keep `reflection-loop` valid under P1 checks.
2. Keep `memory-augmented` valid under P1 checks.
3. Resolve or explicitly document `TD-V131-08`: if empty `exit_when: rule` remains intentionally always-true, make that behavior explicit in YAML/spec/tests; otherwise add the explicit rule expression.

### B4 — User-facing preset runbooks

1. Ensure embedded preset README/runbook snippets are accurate enough for manual smoke runs.
2. Avoid documenting commands that do not exist.

---

## 3. Non-goals

- Adding new embedded presets
- Implementing conditional routing or branching
- Changing `kb.extract_work`, `creator.*`, `judge.*`, or `context.summarize` semantics beyond preset wiring needs
- Closing `SEC-V131-01` (P3 owns it)
- Moving DF-29/DF-31 out of backlog
- Platform integration

1. `kb-extract`, `novel-writing`, `reflection-loop`, and `memory-augmented` all pass strict validation.
2. Embedded preset smoke tests fail when a compiled preset asset is structurally invalid.
