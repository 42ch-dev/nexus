### 🟢 Suggestion

#### S-001: Clarify optimization scope in `HANDLEBARS` doc comment
**Location**: `crates/nexus-orchestration/src/tasks/mod.rs:1036-1041`

The doc comment states the `OnceLock` registry "avoids per-call allocation overhead." While true, `Handlebars::render_template` compiles the template string on-the-fly every call (it does not cache compiled templates in the registry). The actual win is avoiding repeated registry object allocation and escape-fn setup, not template compilation caching.

**Recommendation**: Update the doc comment to be precise:
```rust
/// Static Handlebars registry — avoids per-call registry allocation and
/// escape-function setup overhead. Templates are still compiled per-render.
```

**Rationale**: Prevents future maintainers from incorrectly assuming template compilation is cached, which could lead to premature optimization attempts or confusion about expected performance gains.

---

#### S-002: Consider `#[cfg(test)]` visibility annotation for `handlebars_registry`
**Location**: `crates/nexus-orchestration/src/tasks/mod.rs:1047`

`handlebars_registry()` is currently `pub(crate)`-equivalent (no explicit visibility, so private to the module). It is only called by `render_core_context_template` and the unit test `core_context_template_repeated_renders_are_deterministic`.

**Recommendation**: No action required unless the module structure changes. Marking as observation only — the current encapsulation is correct.

---

#### S-003: Document why `Handlebars<'static>` is safe to share across threads
**Location**: `crates/nexus-orchestration/src/tasks/mod.rs:1040`

`Handlebars` is not automatically `Sync` in all configurations (it depends on helper/decorator registrations). The current registry only sets `no_escape` and uses no custom helpers, making it implicitly `Sync`. If a future change registers a non-`Sync` helper, compilation will fail, which is the correct guardrail.

**Recommendation**: Add a brief safety note:
```rust
// SAFETY: This registry uses only built-in helpers and a Send+Sync escape fn,
// making the Handlebars instance Sync. Custom helper registration would require
// explicit Sync bounds.
```
