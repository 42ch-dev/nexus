### 🟢 Suggestion

#### S-001: skill_sync.rs — no path traversal validation on skill file content
- **Location**: `crates/nexus-orchestration/src/skill_sync.rs`
- **Observation**: `sync_embedded_skills()` copies files from embedded skills to `$HOME/.nexus42/skills/<slug>/`. The skill slug is validated by the manifest loader (alphanumeric + hyphens), which prevents directory traversal in the destination path. However, there is no explicit validation that embedded skill content doesn't contain malicious `../` references in file content.
- **Impact**: Low — embedded skills are compiled into the binary at build time and are under developer control, not user input. No external attacker can inject skill content.
- **Recommendation**: No action needed for V1.15. Consider adding a note in docs that embedded skills are trusted build-time content.

#### S-002: skill_link.rs — symlink target validation is implicit
- **Location**: `crates/nexus-orchestration/src/skill_link.rs`
- **Observation**: `ensure_skill_link()` validates that the target exists as a directory before creating the symlink, and checks if existing symlinks point to the correct target. However, it does not validate that the target is under `$HOME/.nexus42/skills/` (i.e., a caller could theoretically create a link to any directory).
- **Impact**: Low — only called from `ensure_preset_skill_links()` which passes `user_skills_dir()` as the base. The path comes from `nexus_home_layout::user_skills_dir()` which returns `$HOME/.nexus42/skills/`.
- **Recommendation**: No action needed — the call site constrains the base path correctly.
