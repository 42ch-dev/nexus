## Task T1: Review Classification Quality Signal — FINDINGS

### R-Q1 (Critical): Memory leak in `quality_signal()` function

**File**: `crates/nexus-domain/src/review_quality.rs`  
**Line**: 43

```rust
let entry = seen.entry(Box::leak(lower.into_boxed_str())).or_insert(0);
```

**Problem**: The `Box::leak(lower.into_boxed_str())` pattern creates a `'static` lifetime reference to a heap-allocated string by leaking it intentionally. However, this function is intended to be a **pure function** used in a classification pipeline that could process many digests. Each call to `quality_signal()` leaks memory proportional to the number of unique tokens in the input.

**Impact**: Over time, repeated classification of digests will accumulate leaked allocations. For a daemon or CLI processing many pending reviews, this could cause unbounded memory growth.

**Severity**: Critical — memory leak in hot path

**Recommendation**: Use `std::collections::HashSet<String>` instead of `HashMap<&str, usize>` with leaked boxes. The lowercase string can be stored directly as an owned `String` in the HashSet:

```rust
let mut seen: HashSet<String> = HashSet::new();
for tok in &tokens {
    let lower = tok.to_lowercase();
    seen.insert(lower);  // owned String, no leak
}
```

**Expected Fix Commit**: Same commit or same PR batch as T1.

### R-Q2 (Warning): Test coverage boundary for quality thresholds

**File**: `crates/nexus-domain/src/review_quality.rs`

**Observation**: The `is_high_signal()` function uses hardcoded thresholds:
- `unique_ratio >= 0.35`
- `alpha_ratio >= 0.65`
- `repeated_token_ratio <= 0.45`

The tests verify these thresholds against specific inputs, but there's no test for **boundary conditions** (e.g., exactly 0.35 unique ratio, or 0.45 repeated token ratio). This is acceptable for an initial implementation but could be documented as a known limitation.

**Severity**: Low (suggestion)

### R-Q9 (Warning): `workspace_path().unwrap_or_default()` fallback behavior

**File**: `crates/nexus42d/tests/integration.rs`  
**Line**: 653

**Observation**: The test falls back to an empty string if `workspace_path()` returns `None`:
```rust
let workspace_root = state.workspace_path().unwrap_or_default();
```

If `workspace_root` is empty, `workspace_dir.join("tmp")` would create a `tmp` directory in the current working directory during tests. This could cause test pollution if the current directory is not intended to be a test workspace.

**Severity**: Low (suggestion) — likely benign in CI but worth validating
