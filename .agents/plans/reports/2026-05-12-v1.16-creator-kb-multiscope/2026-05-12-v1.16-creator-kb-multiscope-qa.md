### F-003 — Corrupt index silently treated as empty (QC2 Warning) — OPEN

**Status**: Not fixed in current commit range.
- `read_kb_index()` at line 343 still uses `unwrap_or_default()` for corrupt JSON
- Missing vs corrupt index not distinguished — both return `KbIndex::default()`
- If `index.json` contains malformed JSON, all existing entries become invisible and the next `kb add` overwrites with a single-entry index
- Atomic write was added (W2 improvement), but the corrupt-vs-missing distinction was not implemented

**Impact**: Data loss scenario if index file is corrupted (disk error, partial write before atomic fix).
**Recommendation**: Distinguish `index_path.exists()` check (already at line 337) from parse failure. On parse failure, log a warning and return error rather than silently defaulting.

### F-004 — Entry ID collision (QC2 Warning) — PARTIALLY ADDRESSED

**Status**: Diversifier added but no create-new guard.
- `generate_entry_id()` now mixes millis + subsec_nanos + diversifier (lines 351-360)
- Collision probability reduced but not eliminated
- `std::fs::copy(file, &dest)` at line 527 still silently overwrites existing file
- No `create_new` or existence check on destination before copy

**Impact**: In edge cases (clock skew, extreme rapid calls), a new entry could overwrite an existing entry file while the index contains both IDs.
**Recommendation**: Add `dest.exists()` check before copy, or use `OpenOptions::new().create_new(true)`.
