## Suggestions

### S1: Stale comment in `delta_bundle.rs` test

**File**: `crates/nexus-sync/src/delta_bundle.rs`, line ~397

```rust
// LATEST_SCHEMA_VERSION may be higher due to local schemas like daemon-status-v2
```

**Issue**: This comment references `daemon-status-v2` as a local schema that *may* raise `LATEST_SCHEMA_VERSION`. After WS5, `daemon-status-v2` is no longer codegen'd (it moved to `src/local/`), so `LATEST_SCHEMA_VERSION` dropped from 2 → 1. The comment's premise is now inverted — local schemas can no longer affect this constant since they're outside codegen. The assertion `assert_eq!(bundle.schema_version, 1)` remains correct, but the explanatory comment is misleading.

**Recommendation**: Update the comment to reflect the post-WS5 reality:

```rust
// Bundle uses schema_version=1 (sync wire contract).
// LATEST_SCHEMA_VERSION now tracks only wire schemas (local schemas are not codegen'd).
```

**Severity**: Suggestion — no functional impact; test still passes.

---

### S2: `LATEST_SCHEMA_VERSION` semantic shift not documented in CHANGELOG

**File**: `packages/nexus-contracts/CHANGELOG.md`

**Issue**: The CHANGELOG lists removed types but does not mention that `LATEST_SCHEMA_VERSION` dropped from 2 → 1. TypeScript consumers of `@42ch/nexus-contracts` who read `LATEST_SCHEMA_VERSION` for protocol versioning may be surprised by the change. While `nexus-platform` (the sole consumer) likely doesn't depend on this value, the semantic shift is worth noting for future consumers.

**Recommendation**: Add a line under the `### Changed` section:

```markdown
- `LATEST_SCHEMA_VERSION` changed from `2` → `1` (DaemonStatusV2 was the only v2 schema; it is now a hand-written local type).
```

**Severity**: Suggestion — low risk; platform confirmed zero usage of removed types.

## Positive Observations

1. **Type safety improvement**: `DaemonStatusV2` and `LocalIdentity` went from untyped `String` enums (codegen) to proper Rust enums with `#[serde(rename_all = "snake_case")]` — eliminating a class of runtime serialization bugs.
2. **Clean `runtime.rs` refactor**: The `nexus42d` daemon status handler replaced `serde_json::json!` blob construction with typed struct literals, improving type safety and IDE discoverability.
3. **Consistent import discipline**: Call sites use explicit `use nexus_contracts::local::<module>::<Type>` rather than wildcard re-exports at the crate root, making the wire/local boundary visible at use sites as recommended in §3.2.
4. **`LocalIdentity` conversion simplification**: The typed `IdentityType` enum removes the need for `FromStr` error handling at the domain-contract boundary, simplifying both `TryFrom` and `From` implementations.
5. **`enum_conversions.rs` clean update**: Only the import path changed; the conversion logic itself remains intact, demonstrating good separation between generated types and hand-maintained conversions.
