```
├── mod.rs              # Re-exports domain::*, acp_runtime::*, meta::*

│   └── registry_manifest.rs
└── domain/
    ├── mod.rs          # Re-exports all domain local types
    ├── agent_profile.rs
    ├── local_identity.rs
    ├── manuscript_state.rs
    ├── outbox_entry.rs
    ├── reference_source.rs
    ├── runtime_mode.rs
    └── workspace_binding.rs
```

**Assessment**: The structure is clean and follows the spec's §3 placement convention. Each module has:
- `#[derive(Debug, Clone, Serialize, Deserialize)]` (minimum)
- `#[serde(rename_all = "snake_case")]` for structs
- Round-trip serialization tests in `#[cfg(test)]` blocks
- Re-exports at the `mod.rs` level for ergonomic imports

**One minor issue**: The `local/mod.rs` uses `pub use meta::*;` which re-exports meta types at the `local::` namespace level. This is fine for now but could cause naming collisions if meta types grow. Not a blocking issue.
