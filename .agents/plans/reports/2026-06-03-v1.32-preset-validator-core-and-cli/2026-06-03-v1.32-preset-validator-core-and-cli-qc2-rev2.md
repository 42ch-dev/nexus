### 🟢 Suggestion
- S-001 (carry-over polish): Consider redacting or relativizing the bundle root in `MissingAsset` messages (or provide a separate "debug" field). Update the missing-asset tests to assert `!e.contains("/private/") && !e.contains("/var/")` (or equivalent portable check) for consistency with the new dotdot W2 test. This would fully close the sanitization item from round 1.
- S-002: The input `req.path` in `validate_preset` is still used raw (no early canonicalize, no allow-list under user preset dirs). While by design for a local "validate this candidate YAML/bundle" surface, add a comment documenting the threat model and why arbitrary FS paths are accepted here (vs. e.g. load time). Optional: reject paths containing `..` at the handler level before FS ops (defense in depth).
- S-003: `check_symlink_escapes` and `check_asset_file_existence` both do `bundle_root.join(rel_path)` + canonicalize/exists. Consider factoring a small helper that returns (canonicalized, within_root) or similar to avoid duplicated join logic. Minor maintainability.
- S-004: The legacy `validate_template_files_in_sandbox` in loader (still used by `load_preset` from dir) duplicates some canonicalize logic from the new facade. Future cleanup could make the facade's asset checks the single impl and call it from load_preset when a root is present (non-breaking since P1 is additive).
