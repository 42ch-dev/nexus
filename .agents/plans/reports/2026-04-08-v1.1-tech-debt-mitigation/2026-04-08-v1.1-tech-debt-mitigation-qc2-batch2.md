#### Finding M-1 (Medium): Script/Style Removal is Fragile

**Location**: `research.rs:471-556` (`strip_html_tags` function)

**Issue**: The custom HTML tag stripper uses character-by-character parsing which may fail on:
- Malformed HTML (unclosed tags)
- Nested script tags
- Script tags with attributes containing `>` character

```rust
// Current implementation - vulnerable to malformed HTML
if i + 7 <= chars.len() {
    let tag_start: String = chars[i..i+7].iter().collect();
    let tag_lower = tag_start.to_lowercase();
    if tag_lower == "<script" {
        // Skip until </script>
```

**Risk**: If a PDF extraction produces HTML-like content or a fetched URL returns malformed HTML, the stripper may:
- Include script content in extracted text
- Hang on unclosed tags
- Produce incorrect output

**Recommendation**: Use a proper HTML parser crate like `scraper` or `select` as noted in the existing comment (line 469).

**Remediation Priority**: Medium - Current implementation works for well-formed HTML but should be hardened.

        ));
    }
}

#### Outbox Integration ✅

```rust
pub fn mark_conflicted_with_retry(
    &self,
    outbox_entry_id: &str,
    error: &str,
    replay_after: &RetryAfterPolicy,

    let now = chrono::Utc::now();
    let next_retry_at = match retry_after {
        RetryAfterPolicy::AtTime(t) => Some(t.to_rfc3339()),
        RetryAfterPolicy::AfterSeconds(secs) => {
            let target = now + chrono::Duration::seconds(*secs as i64);
            Some(target.to_rfc3339())
        }
        RetryAfterPolicy::None => None,
    };
    // Stores next_retry_at for replay() to check
}
```

#### Finding L-2 (Low): Missing Max Retry Limit for Partial Apply

**Location**: `partial_apply.rs:183-195`
