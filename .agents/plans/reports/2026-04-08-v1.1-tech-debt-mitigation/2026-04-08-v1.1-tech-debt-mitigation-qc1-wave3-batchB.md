#### W-1: CI drift check shell script readability
- **Issue**: The `schema-drift-check` job in `.github/workflows/ci.yml` uses complex shell one-liners with multiple `grep`, `sed`, and `tr` pipes. While functional, this pattern reduces maintainability and makes debugging harder if a check fails unexpectedly.
- **Location**: `.github/workflows/ci.yml:119-177` (all three steps)
- **Recommendation**: Consider extracting schema comparison logic into a small script in `tooling/` directory (e.g., `tooling/check-schema-drift.sh`) for better readability and testability. The script can be called from CI and also run locally during development.
- **Severity**: Warning (non-blocking, quality improvement)
- **Source**: Manual inspection + architecture review

#### W-2: TempDir ownership semantics in test_utils.rs
- **Issue**: `create_test_workspace()` and `create_initialized_test_workspace()` return `TempDir` as the first element of the tuple, but the documentation correctly emphasizes that "the caller must keep `temp_dir` alive for the duration of the test". This ownership pattern is correct but subtle — developers might accidentally drop `TempDir` early by destructuring incorrectly or ignoring the first tuple element.
- **Location**: `crates/nexus42d/src/test_utils.rs:11-23, 40-43` (documentation)
- **Recommendation**: Add a more prominent warning in the function signature or use a newtype wrapper (e.g., `TestWorkspaceGuard`) that makes the ownership requirement explicit in the type system. Current documentation is good, but could be reinforced with a compile-time hint.
- **Severity**: Warning (non-blocking, future safety improvement)
- **Source**: Manual inspection + Rust ownership pattern analysis

#### W-3: LATEST_SCHEMA_VERSION type mismatch (u32 vs &str)
- **Issue**: The CI check compares `LATEST_SCHEMA_VERSION` (u32) with `SCHEMA_VERSION` (&str) numerically after extraction. While the shell script handles this correctly (`"$CONTRACTS_VERSION" != "$CLI_VERSION"` after string extraction), the type mismatch suggests a potential future bug if schema versions evolve to non-integer strings (e.g., `"v2.1"`).
- **Location**: CI check at `.github/workflows/ci.yml:171-177` vs generated contracts `crates/nexus-contracts/src/generated/mod.rs:84`
- **Recommendation**: Standardize schema version representation across all packages (either all u32 or all &str). If u32 is preferred, change CLI/daemon `SCHEMA_VERSION` to u32 constants; if &str is preferred, change generated `LATEST_SCHEMA_VERSION` to &str. This prevents future comparison bugs.
- **Severity**: Warning (non-blocking, design consistency)
- **Source**: Manual inspection + cross-package type analysis
