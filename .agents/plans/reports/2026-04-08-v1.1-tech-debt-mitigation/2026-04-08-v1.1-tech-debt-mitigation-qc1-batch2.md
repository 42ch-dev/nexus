## Findings

### 🟡 Warning

#### W-B2-1: Database migration silently swallows errors
**Location**: `crates/nexus42/src/db/mod.rs:33`
**Issue**: Migration uses `let _ = conn.execute("ALTER TABLE ...", [])` which silently ignores errors. While this is intentional to support idempotent migrations, failures other than "column already exists" are also swallowed.
**Impact**: If the ALTER TABLE fails due to disk full, permissions, or schema corruption, the migration will silently fail and subsequent operations will error on missing `content` column.
**Recommendation**: Check error kind and only ignore `SqliteFailure(SqliteFailure { code: DuplicateColumn, ... })`. Re-raise other errors.
**Severity Rationale**: Low probability in typical dev environments, but could cause confusing errors in production. Proper error handling is trivial to add.

#### W-B2-2: unwrap_or(0) for conflict count query
**Location**: `crates/nexus42/src/manuscript/manager.rs:465`
**Issue**: `check_sync_conflicts()` uses `unwrap_or(0)` on the query result, treating any error as "no conflicts". This could mask database corruption or permission issues.
**Code**:
```rust
let count: u64 = conn
    .query_row(
        "SELECT COUNT(*) FROM outbox_entries WHERE delivery_state = 'conflicted'",
        [],
        |row| row.get(0),
    )
    .unwrap_or(0);
```
**Impact**: If the outbox_entries table is missing or corrupt, strict mode promotion will succeed incorrectly, allowing promotions with undetected sync conflicts.
**Recommendation**: Replace with proper error propagation using `?` operator or return a dedicated error type.
**Severity Rationale**: Affects correctness of a user-facing validation feature. Unlikely in normal operation but violates "errors should be explicit" principle.

#### W-B2-3: External tool dependency not validated at runtime
**Location**: `crates/nexus42/src/commands/research.rs` (PDF extraction)
**Issue**: The `--extract-text` feature depends on external tools (`pdftotext`, `curl`). The implementation uses `Command::new("pdftotext")` without checking if the tool exists. Failures are silently caught with `ok()` and logged.
**Impact**: Users without these tools installed will see extraction silently fail with no actionable error message.
**Recommendation**: Add a `check_dependencies()` function that runs `pdftotext -v` and `curl --version` on first use, caching the result. Provide clear error messages if tools are missing.
**Severity Rationale**: Feature works but fails silently. Improves UX and aligns with "explicit error handling" principle.

### 🟢 Suggestion

#### S-B2-1: Add validation for content_hash format
**Location**: `crates/nexus42/src/manuscript/metadata.rs:31`
**Issue**: The `content_hash` field is an `Option<String>` with no format validation. The implementation uses `sha256:` prefix, but there's no guarantee future writes will maintain this format.
**Recommendation**: Add a custom deserializer or validation method that ensures `content_hash` matches `^sha256:[a-f0-9]{64}$` regex. Alternatively, use a newtype `ContentHash(String)` with validation in constructor.
**Benefit**: Prevents typos or malformed hashes from being stored.

#### S-B2-2: RetryAfterPolicy should implement Display
**Location**: `crates/nexus-sync/src/outbox.rs:74-82`
**Issue**: `RetryAfterPolicy` enum lacks a `Display` implementation, making it harder to log or include in error messages.
**Recommendation**: Implement `std::fmt::Display` to produce human-readable strings like "retry after 2026-04-08T16:00:00Z" or "retry after 60s".
**Benefit**: Improves observability and debugging.

#### S-B2-3: Consider adding content length limit
**Location**: `crates/nexus42/src/commands/research.rs` (PDF/URL extraction)
**Issue**: Extracted text is stored without size limit. A large PDF or URL could consume excessive database storage.
**Recommendation**: Add a configurable `--max-content-size` flag (default: 1MB) and truncate extracted text with a marker `[truncated at 1MB]`. Store truncation status in a new column `content_truncated BOOLEAN`.
**Benefit**: Prevents runaway storage consumption.

#### S-B2-4: Precheck auth validation should include world_id
**Location**: `crates/nexus-sync/src/precheck.rs:256-305`
**Issue**: `precheck_bundle_with_auth()` validates `submitting_creator_id` matches `authenticated_creator_id`, but doesn't check that the creator has permissions for the target `world_id`. The multi-creator flag is passed but not cross-referenced with world membership.
**Recommendation**: Add a `check_world_membership()` step that verifies the authenticated creator has at least `Collaborator` role in the bundle's `world_id`. This can be a V1.1+ enhancement.
**Benefit**: Completes the multi-creator governance model.
