### 🟢 Suggestion

**S-B2-1: CI column extraction is fragile for complex SQL schemas**
- **Location**: `.github/workflows/ci.yml` lines 141-150
- **Issue**: The `grep -oP '^\s+\K\w+'` pattern extracts the first word from each line within the `CREATE TABLE` range. This works for simple `column_name TYPE,` definitions but would break if: (a) SQL comments use `--` instead of `//`, (b) multi-line constraints or CHECK clauses are added, (c) column names contain underscores followed by keywords (unlikely but possible).
- **Impact**: Low — current schema is simple. Future schema additions may need more robust parsing.
- **Fix**: Consider a simple Python/Rust script for complex schema comparison, or add a comment documenting this limitation. Acceptable for V1.1.

**S-B2-2: `create_initialized_test_workspace()` not yet adopted by any consuming test**
- **Location**: `crates/nexus42d/src/test_utils.rs` lines 43-64
- **Issue**: The `create_initialized_test_workspace()` helper is defined and self-tested, but no existing test outside `test_utils.rs` uses it. Tests that seed `workspace_meta` (e.g., in middleware.rs, workspace/mod.rs) still do so manually.
- **Impact**: None for correctness — helper works. But it suggests incremental migration is still pending.
- **Fix**: Continue migrating tests in middleware.rs, workspace/mod.rs, auth_middleware.rs in a follow-up batch. Non-blocking for V1.1.
