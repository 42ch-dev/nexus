### 🟡 Warning

#### QC3-B2-W1: retry_count stored in two places (SQLite column + JSON state)

- **Location**: `crates/nexus-sync/src/outbox.rs:494-508` and `crates/nexus-sync/src/partial_apply.rs:198`
- **Issue**: `persist_partial_apply_state()` stores `retry_count` both as a separate SQLite column AND inside the serialized `state_json`. This creates a potential inconsistency if the JSON is updated without updating the column, or vice versa.
- **Current code**:
  ```rust
  self.conn.execute(
      "INSERT OR REPLACE INTO partial_apply_states
          (outbox_entry_id, state_json, recorded_at, retry_count)
       VALUES (?1, ?2, ?3, ?4)",
      params![outbox_entry_id, state_json, now, state.retry_count],
  )?;
  ```
- **Impact**: If `increment_retry()` is called on a `PartialApplyState` object and then `persist_partial_apply_state()` is called, the SQLite column will be correct. But if code ever reads `retry_count` directly from SQLite instead of deserializing the JSON, it may get stale data.
- **Fix**: Remove the redundant `retry_count` column and always deserialize from `state_json`. Or use the SQLite column as the authoritative source and ignore the JSON field.
- **Confidence**: Medium

#### QC3-B2-W2: No foreign key constraint between partial_apply_states and outbox_entries

- **Location**: `crates/nexus-sync/src/outbox.rs:124-129`
- **Issue**: The `partial_apply_states` table has `outbox_entry_id TEXT PRIMARY KEY` but no `REFERENCES outbox_entries(outbox_entry_id)` foreign key constraint.
- **Impact**: Orphaned partial apply states could exist if an outbox entry is deleted but its partial apply state remains. This could also happen if `persist_partial_apply_state()` is called for a non-existent outbox entry.
- **Fix**: Add `FOREIGN KEY (outbox_entry_id) REFERENCES outbox_entries(outbox_entry_id) ON DELETE CASCADE` to the schema definition.
- **Confidence**: Medium

### 🟢 Suggestion

#### QC3-B2-S1: AuthContext::unauthenticated() returns is_multi_creator=false

- **Location**: `crates/nexus-sync/src/precheck.rs:47-53`
- **Observation**: The `unauthenticated()` factory returns `is_multi_creator: false`, which means auth match validation is skipped even in multi-creator worlds when no auth context is provided. This appears intentional for backward compatibility (`precheck_bundle()` uses this), but worth documenting.
- **Suggestion**: Add a comment explaining this is intentional for legacy callers, and that production multi-creator code should always use `AuthContext::multi_creator()`.
- **Confidence**: High

#### QC3-B2-S2: HTML stripping in URL extraction is basic

- **Location**: `crates/nexus42/src/commands/research.rs:457-460`
- **Observation**: `strip_html_tags()` is a basic regex-based approach. For better content extraction, a proper HTML parser (e.g., `scraper` crate) could be used.
- **Suggestion**: For V1.2, consider adding `scraper` or `select` crate for better HTML→text conversion. Current implementation is acceptable for V1.1 MVP.
- **Confidence**: Low

#### QC3-B2-S3: External tool availability check runs on every extraction

- **Location**: `crates/nexus42/src/commands/research.rs:372-379, 429-435`
- **Observation**: Each PDF/URL extraction first checks if `pdftotext`/`curl` is available by running `--version`. This adds overhead for every extraction.
- **Suggestion**: Consider caching the tool availability check (e.g., once per session) or checking once at startup.
- **Confidence**: Low
