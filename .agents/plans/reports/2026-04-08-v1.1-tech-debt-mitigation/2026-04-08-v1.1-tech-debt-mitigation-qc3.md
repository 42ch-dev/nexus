## Findings

### 🟡 Warning

#### W-QC3-1: Pool configuration not tunable at runtime
- **Issue**: `OutboxPool::new()` accepts `max_size` parameter but there is no mechanism to adjust pool configuration after creation. SQLite WAL mode provides good concurrent read/write performance, but production workloads may require tuning pool size based on load.
- **Impact**: Runtime pool size cannot be adjusted without recreating the pool
- **Fix**: Consider adding `OutboxPool::with_max_size()` or accepting pool configuration via environment variable with runtime override
- **Evidence**: `pool.rs:39-46` - pool built with `max_size(max_size)` but no runtime reconfiguration
- **Severity**: Warning (address before merge)
- **Confidence**: High
