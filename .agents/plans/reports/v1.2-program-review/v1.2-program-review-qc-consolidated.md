## 2. Blocking Items

### 🔴 H1: Daemon client `call_assemble` URL path mismatch

**Source**: QC#2
**Severity**: high
**Files**: `crates/nexus42/src/api/daemon_client.rs:186`, `crates/nexus42d/src/api/mod.rs:105`

**Issue**: CLI constructs URL as `/api/v1/context/assemble`, but daemon serves `/v1/local/context/assemble`.

**Impact**: HTTP 404 at runtime for two-stage assembly calls. User-visible failure.

**Fix**: Change daemon_client path to `/v1/local/context/assemble`.

---

### 🔴 H2: Clone command calls non-existent daemon endpoint

**Source**: QC#2
**Severity**: high
**Files**: `crates/nexus42/src/commands/clone.rs:124`

**Issue**: CLI posts to `/v1/local/world/clone`, but no route registered in daemon.

**Impact**: HTTP 404 when daemon is running. The `--dry-run` works but real clone fails.

**Fix Options**:
1. Implement daemon handler and route
2. Remove command from public CLI until endpoint ready
3. Mark as experimental/unstable with warning

## 3. Critical Findings (Non-blocking for Merge)

### 🟠 C-001: Review classification purely length-based

**Source**: QC#1
**Severity**: critical (architecture)
**Files**: `crates/nexus-domain/src/review.rs:150-224`

**Issue**: `classify_pending_review()` uses only `raw_digest.len()` to decide Drop/Fragment/Promote. No content-quality signal.

**Impact**: Garbage content may be promoted to long-term memory; dense valuable content may be dropped. Long-term memory quality degradation.

**Mitigation**: Not blocking for V1.2 MVP (per QC#3 assessment — "not a blocking security or correctness issue"). Track for V1.3.

**Recommendation**: Add entropy/unique-word-count ratio before length check.
