#### S-FQC3-001: Double YAML parsing in `load_preset()`
**Location:** crates/nexus-orchestration/src/preset/loader.rs:199-208

**Problem:** The YAML content is parsed twice in `load_preset()`:
1. Line 199: `serde_yaml::from_str(&yaml)` for sandbox validation
2. Line 208: `load_preset_from_str(&yaml, caps)` parses it again

This means every filesystem preset load pays the serde_yaml deserialization cost twice. For embedded presets this doesn't apply (they use `load_preset_from_str` directly), but for user-installed filesystem presets this will add overhead.

**Suggested Fix:** Parse once and reuse the `PresetManifest`. Either:
1. Have `load_preset_from_str` accept an already-parsed manifest
2. Or create an internal helper that takes `&str` and an optional `PresetManifest`

**Risk:** LOW (performance optimization only, no correctness impact)

#### S-FQC3-002: Unnecessary String allocations for dot-paths in validation
**Location:** crates/nexus-orchestration/src/preset/loader.rs:273-314

**Problem:** In `validate_template_files_in_sandbox()`, every template_file path triggers a `format!()` call to build a diagnostic dot-path like `"states[{}].exit_when.template_file"`. These allocations happen even on the happy path (no validation errors).

For a preset with many states and inner graph nodes, this creates many temporary Strings that are immediately discarded if validation passes.

**Suggested Fix:** Consider two alternatives:
1. Only build the diagnostic string when actually pushing a `ValidationProblem`
2. Use `Cow<'static, str>` or a closure-based approach to defer allocation until error time

**Risk:** LOW (micro-optimization, measurable only with many template_files)

---

#### S-FQC3-003: `canonicalize()` silently ignored for non-existent files
**Location:** crates/nexus-orchestration/src/preset/loader.rs:319-336

**Problem:** The sandbox validation via `canonicalize()` is silently skipped when the file doesn't exist (line 331-335). The comment correctly notes that structural checks (`..`, `/` prefix, null bytes) already ran in `assert_template_file_safe()`.

However, this means:
- Symlinks pointing outside the bundle won't be caught if the symlink target doesn't exist
- Filesystem errors (permission issues, I/O errors) are silently swallowed

**Suggested Fix:** Distinguish between `NotFound` and other IO errors. Only swallow `NotFound`, and report other errors as validation problems. This improves error observability without affecting the security model.

Example:
```rust
Err(e) if e.kind() == std::io::ErrorKind::NotFound => { /* ok, skip */ }
Err(e) => { /* report as validation problem */ }
```

**Risk:** LOW (error reporting improvement only)

---

#### S-FQC3-004: `canonicalize()` syscall per template_file
**Location:** crates/nexus-orchestration/src/preset/loader.rs:319

**Problem:** `canonicalize()` is a syscall that accesses the filesystem. For a preset with many template_file references (e.g., 20+ states with llm_judge + context_update hooks), this means 20+ syscalls per preset load.

**Suggested Fix:** Since the structural validation (`assert_template_file_safe`) already catches obvious traversal patterns, consider making the canonicalize sandbox check optional for embedded presets (which are trusted and compiled into the binary). User-installed presets should still run this defense-in-depth check.

**Risk:** LOW (performance optimization, would only affect trusted embedded presets)
