## Findings

### R1 — `hyper` / `http-body-util` Direct Dependencies in `nexus42d`

**Severity**: `medium`
**File**: `crates/nexus42d/Cargo.toml` lines 41–43
**Evidence**:
```toml
hyper = "1"
hyper-util = { version = "0.1", features = ["tokio"] }
http-body-util = "0.1"
```

These three deps are declared directly in `nexus42d/Cargo.toml` but are **not present in the workspace `Cargo.toml`** `[workspace.dependencies]` table. Per `AGENTS.md` §Development Workflow, workspace deps should be declared centrally.

**Expected**: workspace `Cargo.toml` should declare:
```toml
hyper = "1"
hyper-util = { version = "0.1", features = ["tokio"] }
http-body-util = "0.1"
```
And `nexus42d/Cargo.toml` should use:
```toml
hyper = { workspace = true }
hyper-util = { workspace = true }
http-body-util = { workspace = true }
```

**Note**: `hyper 1.x` is plausible for Axum 0.7 integration. This is a `medium` hygiene issue, not a functional defect.

### R2 — `statig` Not Using Workspace Dep

**Severity**: `low`
**File**: `crates/nexus42d/Cargo.toml` line 44
**Evidence**:
```toml
statig = { version = "0.3", features = ["async"] }
```

`statig = "0.3"` is declared directly rather than as `statig = { workspace = true }`. While the version is pinned (correct), having it in the workspace deps table would ensure version consistency across the workspace.

---

### R3 — `tokio::time::sleep` in Worker IPC Test (Timing Dependency)

**Severity**: `low` (test quality / potential flakiness)
**File**: `crates/nexus-orchestration/tests/worker_ipc_roundtrip.rs` line 18
**Evidence**:
```rust
tokio::time::sleep(Duration::from_millis(100)).await;
```
Used to let the dummy worker start and be ready to read. In a slow CI environment this could cause intermittent failures. The 100ms value is conservative but reasonable for a local subprocess startup.

**Mitigation**: Acceptable as-is for a skeleton phase. A proper fix (event-based readiness) can be done when WS3 wires real workers.

---

### R4 — `tokio::time::timeout` in Crash Detection Test

**Severity**: `low` (test quality)
**File**: `crates/nexus-orchestration/tests/worker_ipc_roundtrip.rs` lines 47, 56
**Evidence**:
```rust
let evt = tokio::time::timeout(Duration::from_secs(5), events.recv())...
```
5-second timeout for crash event detection is generous. Acceptable for now but worth noting the test suite could be slow if many crash scenarios occur.
