## Warning Findings (Must Fix)

### W-001: Unreachable Error Branches in `validate_handle()`

- **Cross-review**: QC1 W-001 only
- **Scope**: WS-B
- **Source**: `crates/nexus42/src/commands/creator.rs`, lines 156–176
- **Issue**: The `validate_handle()` function uses an if-else chain to provide detailed error messages, but the regex `^[a-z0-9][a-z0-9._-]{2,13}[a-z0-9]$` already enforces all structural constraints (length 4–15, valid chars, start/end rules). This makes most if-else branches unreachable dead code. Users never see the specific "too short" / "too long" / "must start with" messages.
- **Impact**: Dead code; misleading error feedback UX.
- **PM Decision**: **Fix in this wave**. Dev must simplify `validate_handle()` to avoid unreachable branches:
  - **Preferred**: Keep regex as the single source of truth (matches frozen spec exactly). Remove if-else chain. Return a single clear error message: `"Handle must be 4–15 characters, start and end with a letter or digit, and contain only lowercase letters, digits, dots, hyphens, and underscores."`
  - **Alternative**: Remove regex and use only if-else chain (preserves detailed errors but risks divergence from frozen spec regex).
