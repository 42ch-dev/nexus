### 🟡 Warning

**W-A01: Duplicate `yaml_value_depth` violates single truth source for depth semantics**
- **Issue**: Identical `yaml_value_depth` implementations exist in both `nexus-orchestration/src/preset/loader.rs` (lines 379-401) and `nexus42/src/commands/preset.rs` (lines 1250-1272). This creates a maintenance hazard: if depth measurement semantics change (e.g., handling of empty sequences, key depth in mappings), both implementations must be updated synchronously. The CLI's `validate_preset` and the orchestration loader's `load_preset_from_str_with_limits` share the same depth-checking logic but diverge in implementation location.
- **Impact**: Architecture erosion risk — future contributors may fix depth logic in one location without realizing the other exists, leading to inconsistent behavior between `preset validate` and actual preset loading.
- **Fix**: Export `yaml_value_depth` as `pub` from `nexus-orchestration/src/preset/loader.rs` and import it in `nexus42/src/commands/preset.rs`. Alternatively, move the function to a shared utility location if it becomes more generally useful.
- **Source**: `crates/nexus-orchestration/src/preset/loader.rs:379-401`, `crates/nexus42/src/commands/preset.rs:1250-1272`
- **Severity**: warning (maintainability/architecture)

**W-A02: Cross-crate build-time dependency without explicit contract**
- **Issue**: `crates/nexus42/build.rs` scans `../nexus-orchestration/embedded-presets/` at compile time, creating a build-time dependency between crates. While this is a valid pattern for compile-time data generation, there is no explicit contract (e.g., AGENTS.md note or documentation) documenting that nexus-orchestration's `embedded-presets/` directory structure is a stable interface for downstream crates. If nexus-orchestration reorganizes its preset layout (e.g., moves to `presets/embedded/`), nexus42's build.rs will silently fail or produce incorrect output.
- **Impact**: Implicit coupling — changes in nexus-orchestration may break nexus42 build without clear indication.
- **Fix**: Add a brief note in `crates/nexus-orchestration/AGENTS.md` documenting that `embedded-presets/` directory structure is a stable interface for downstream build-time consumers. Consider adding a symlink or explicit path constant if the relationship becomes more complex.
- **Source**: `crates/nexus42/build.rs:15-19`, `crates/nexus-orchestration/AGENTS.md` (missing documentation)
- **Severity**: warning (architecture/dependency direction)

### 🟢 Suggestion

**S-A01: build.rs `rerun-if-changed` path could be absolute**
- **Issue**: `cargo:rerun-if-changed=../nexus-orchestration/embedded-presets` uses a relative path with `..`. Cargo resolves this relative to the package root, which works in current versions but is less robust than an absolute path derived from `CARGO_MANIFEST_DIR`.
- **Fix**: Compute the absolute path using `CARGO_MANIFEST_DIR` and emit `cargo:rerun-if-changed={absolute_path}`.
- **Source**: `crates/nexus42/build.rs:48`
- **Severity**: suggestion (robustness)

**S-A02: `UserPresetScanResult` index field is private but could benefit from accessor**
- **Issue**: The `index: HashMap<String, usize>` field in `UserPresetScanResult` is private (appropriate), but there is no accessor method like `preset_ids()` that would allow consumers to iterate over the indexed IDs without accessing the Vec directly. This is a minor extensibility gap.
- **Fix**: If future consumers need to enumerate preset IDs without accessing the Vec, add a `pub fn preset_ids(&self) -> impl Iterator<Item = &String>` method.
- **Source**: `crates/nexus-orchestration/src/user_preset_dir.rs:38-39`
- **Severity**: suggestion (extensibility)

**S-A03: `is_scan_cache_fresh` mtime precision limitation could be documented**
- **Issue**: The mtime comparison (`current_mtime != cached_mtime`) is correct, but filesystems with 1-second mtime resolution can miss sub-second modifications. This is acceptable for presets but worth documenting for future maintainers.
- **Fix**: Add a one-line doc comment noting the 1-second-resolution limitation and its acceptability for preset use cases.
- **Source**: `crates/nexus-orchestration/src/user_preset_dir.rs:255-260`
- **Severity**: suggestion (documentation)
