## QC3 Review — Primary Accent: Performance & Reliability

### R1 — Linkage matrix enforcement (Critical) ✅

**Finding**: The CI job `verify-no-acp-in-daemon` checks `crates/nexus42d/Cargo.toml` with:

```bash
rg -n '^\s*(nexus-acp-host|agent-client-protocol)\s*=' crates/nexus42d/Cargo.toml
```

- ✅ Enforces that `nexus42d` does not depend on `nexus-acp-host` or `agent-client-protocol`.
- ✅ Matches spec §11.3 linkage matrix rule.
- ✅ `nexus42d/Cargo.toml` confirmed clean: no `nexus-acp-host`, no `agent-client-protocol`.
- ⚠️ **Suggestion (low)**: The regex could theoretically match a comment containing `"nexus-acp-host = "` on the same line (e.g. `# NOTE: nexus-acp-host = { path = "..." } # TODO`). In practice this is extremely unlikely; the rule is sound. Could be hardened with `--no-ignore` flag or by also checking for comment-prefixed lines explicitly.

### R2 — Compat shim completeness ✅

**Finding**: `crates/nexus42/src/acp/mod.rs` now contains:

```rust
pub use nexus_acp_host::*;
pub mod session_capture;
```

All `crate::acp::*` consumers in `nexus42` continue to compile:

| Consumer file | Imports | Status |
|---|---|---|
| `errors.rs` | `crate::acp::AcpError` | ✅ via `nexus_acp_host::error::AcpError` |
| `session.rs` | `SessionEntry, SessionManager` | ✅ |
| `agent.rs` | `AgentEntry, DistributionExt, RegistryClient, REGISTRY_URL, AgentSpawner, Platform, CacheMeta, NpxDistribution, BinaryDistribution, Distribution` | ✅ |
| `policy.rs` | `DefaultPolicy, PermissionPolicy` | ✅ |

The `nexus_acp_host::registry` module is a public submodule (`pub mod registry;` in `lib.rs`), so `crate::acp::registry::DistributionExt` resolves correctly through the glob re-export.

### R3 — `session_capture` retention rationale ✅

**Finding**: `session_capture.rs` correctly stays in `nexus42` because it imports `crate::api::daemon_client::DaemonClient`, which lives in `nexus42` and is not moving. The compat shim comment documents this explicitly. This matches spec §11.5.

### R4 — `agent-client-protocol` version pinning ✅

**Finding**: `nexus-acp-host/Cargo.toml` pins `agent-client-protocol = "=0.10.4"` (exact version). This matches spec §1.2 and prevents silent semver regressions from ACP SDK upgrades.

### R5 — Test imports correctly updated ✅

**Finding**: Both moved test files update imports from `nexus42::acp::*` to `nexus_acp_host::*`. Tests in `client.rs` and `localset_bridge.rs` also updated. The `daemon_client` parameter removal from test helpers is consistent with the API change.

### R6 — Phase-1 `acp-worker` stub (Correct for Phase 1) ⚠️

**Finding**: `crates/nexus42/src/commands/acp_worker.rs` is a minimal stub that prints a single JSON-RPC `worker/initialize` reply and exits. This is explicitly documented as Phase-1 only, with Phase-2 (WS2) replacing it with the stdin/stdout JSON-RPC main loop.

✅ Correct for Phase 1 intent.
⚠️ **Suggestion**: No integration test exercises the stub end-to-end. Given the stub's trivial nature (just prints JSON and exits), manual verification per spec §11.4 Step 9 is acceptable for Phase 1.
