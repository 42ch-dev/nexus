### đźźˇ Warning
1. **CLI `link` bypasses domain validation for `platform_creator_id`**  
   `link_identity` calls `nexus_local_db::link_to_platform` directly, skipping `LocalIdentity::link_to_platform` which validates the platform ID against the `CreatorId` regex (`^ctr_[a-zA-Z0-9]+$`). This allows invalid values to be persisted in SQLite, violating the JSON Schema contract.  
   -> Route the `link` command through the domain aggregate so validation is enforced before the DB update.  
   *Ref: `crates/nexus42/src/commands/identity.rs` (~ln 270); `crates/nexus-domain/src/local_identity.rs` (~ln 119-126).*

2. **Silent fallback to `Anonymous` on invalid wire enum value**  
   `From<nexus_contracts::LocalIdentity>` uses `.unwrap_or(LocalIdentityType::Anonymous)` when converting `identity_type`. An invalid string received over the wire is silently coerced to `Anonymous` rather than surfacing an error.  
   -> Change the conversion to `TryFrom` (or add an `Unknown` variant with logging) so callers must handle malformed contract data explicitly.  
   *Ref: `crates/nexus-domain/src/local_identity.rs` (~ln 153-154).*

3. **`--name` is silently ignored for anonymous identities**  
   The CLI argument parser accepts `--name` for both `--anonymous` and `--persistent`, but `LocalIdentity::create_anonymous()` hardcodes `display_name: None`. The schema explicitly allows `display_name` for anonymous identities, so user input is discarded without feedback.  
   -> Either propagate the name into `create_anonymous()` or reject the combination at the CLI layer with a clear error.  
   *Ref: `crates/nexus42/src/commands/identity.rs` (~ln 212); `crates/nexus-domain/src/local_identity.rs` (~ln 86).*

4. **Behavioral inconsistency: `use_identity` rejects anonymous IDs that `resolve_active_identity` accepts**  
   `use_identity` requires the ID to exist in the DB, which excludes ephemeral anonymous identities. By contrast, the (currently dead-code) `resolve_active_identity` treats `ctr_anon*` as valid even when absent from the DB. This means a user cannot run `nexus42 identity use <anon-id>` to switch back to an anonymous identity, even though the config may store one.  
   -> Align `use_identity` with the resolution logic in `resolve_active_identity`, or document that anonymous identities cannot be reused across sessions.  
   *Ref: `crates/nexus42/src/commands/identity.rs` (`use_identity` vs `resolve_active_identity`).*

5. **DB `link_to_platform` collapses distinct errors into a generic `QueryReturnedNoRows` wrapper**  
   All failure branches (not found, already linked, unexpected state) return `LocalDbError::Rusqlite(rusqlite::Error::QueryReturnedNoRows)`. This strips semantic information and forces the CLI to emit a generic "local database error" message.  
   -> Introduce dedicated `LocalDbError` variants such as `IdentityNotFound` and `AlreadyLinked`, and map them to user-friendly CLI errors.  
   *Ref: `crates/nexus-local-db/src/identity.rs` (~ln 179-195).*
