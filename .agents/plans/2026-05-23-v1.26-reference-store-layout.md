## 1. Problem Statement

`reference_sources` today stores optional inline `content` TEXT in SQLite while `nexus-knowledge::ReferenceSource` remains in-memory only. Product needs:

1. **Registry in SQLite** — unit metadata, mutability, scan state, hash, pointer to body.
2. **Body on disk** — User-scoped reference corpus as MD under `~/.nexus42/creators/<creator_id>/references/units/<id>/body.md`.

DF-43 accepted local-db as production owner; V1.26 implements the split and documents layout.

---

## 2. Goals

1. Add normative layout doc (ADR under `.agents/knowledge/` or extend `local-db-schema.md` §reference).
2. Add `nexus-home-layout` helpers: `creator_references_root`, `reference_unit_dir`, `reference_body_path`.
3. SQLite migration: `content_path TEXT`, `source_mutability TEXT NOT NULL DEFAULT 'static'`; deprecate inline `content` for new rows.
4. Update daemon `GET /v1/local/references` to return new fields; optional `GET /v1/local/references/{id}`.
5. Minimal CLI: `creator reference register | list | show` (or extend existing surface if present).
6. Update deferred tracker: DF-43 → implementation in V1.26; add **DF-44** reference body externalization.

---

## 3. Non-goals

- Full `nexus-knowledge` KnowledgeEntry SQLite.
- Platform sync of reference bodies.
- PDF/url fetch pipelines for `refreshable` sources (register + placeholder body OK).
- Changing work-scope `kb/` file index.
