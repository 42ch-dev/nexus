## 1. Problem Statement

`nexus42 acp agent use` prints **Coming soon** ([`cmd_agent_use`](../../crates/nexus42/src/commands/acp/mod.rs)). Users cannot persist a default ACP agent choice per workspace. This is the V1.27 **P2 half-round** item (grill decision #10).

---

## 2. Goals

1. Implement `acp agent use <agent-ref>` to record the default agent for the **active creator + workspace** (or global per creator — pick one in A1 and document).
2. Persist config under `~/.nexus42` using [`nexus-home-layout`](../../crates/nexus-home-layout) path helpers (no hardcoded paths).
3. Subsequent ACP commands that need a default agent read this config (minimum: `acp agent show` or help text confirms selection; extend only where already plumbed).
4. Remove Coming soon messaging; update [`cli_agent.rs`](../../crates/nexus42/tests/cli_agent.rs) expectations.
5. Validate `agent-ref` format (registry id or local name per existing ACP list commands).

---

## 3. Non-goals

- Real ACP connection in [`agent_slot.rs`](../../crates/nexus42/src/commands/acp_worker/agent_slot.rs) (DF-41).
- Multi-agent worker hosting changes (DF-01 shipped).
- Platform registry network calls from this command (local config only).
- Wire permission or session APIs.
