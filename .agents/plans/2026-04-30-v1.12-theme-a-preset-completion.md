# V1.12 Theme A — Preset Module Completion

## Summary

Complete the preset module residual closure work for V1.12: fix four preset-related residuals and update the stale orchestration AGENTS.md description. The goal is to make preset listing usable without a running daemon, harden user-supplied YAML loading, align CLI UX around `_`-prefixed preset names, add explicit preset validation, and make user preset lookup efficient at scale.

## Background

- **V1.12 delivery compass**: `.agents/knowledge/v1.12-delivery-compass-v1.md`
- **Theme A scope**: 4 preset residuals plus 1 documentation fix.
- **Verified compass facts**: `resolve_preset()` already supports user presets through `scan_user_presets` / `find_user_preset`; `preset validate` does not exist yet; `crates/nexus-orchestration/AGENTS.md` still describes user-installed presets as not implemented.
- **Residual target**: R-M1-W03, R-M1-W04, R-M1-W05, R-M1-W06. DOC-FIX is intentionally not a residual.

## Design Decisions

1. **Preset listing without daemon**: Embedded presets should be listed from compile-time or bundled data instead of requiring daemon API access.
2. **YAML limits are explicit and generous**: Use a default maximum depth of approximately 10 and maximum file size of approximately 1MB, with clear user-facing errors when exceeded.
3. **`_` prefix UX alignment**: Prefer accepting `_`-prefixed names during `preset init` and warning during scan/list validation rather than silently skipping entries.
4. **`preset validate` as the explicit diagnostic path**: Add a CLI subcommand that reports invalid YAML and structural preset issues without requiring the user to infer failures from `preset list`.
5. **Indexed lookup**: Build a `HashMap` index during preset scan, use O(1) lookup by preset ID, and invalidate any cache by file modification time.
6. **Documentation follows implementation reality**: Update `crates/nexus-orchestration/AGENTS.md` to describe the current preset resolution flow and user preset support.

## Out of Scope

- DF-03 Preset Registry or preset sharing/distribution.
- DF-11 Handlebars template engine.
- New preset feature design beyond closing the listed residuals.
- New crate dependencies unless separately approved.
