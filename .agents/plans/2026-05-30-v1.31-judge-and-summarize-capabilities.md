## 1. Problem Statement

Three orchestration capabilities remain stubs:

- **DF-32** `judge.rule` — only `always_true` / `always_false`
- **DF-33** `judge.llm` — parses GO/NOGO from input prompt text (no LLM call)
- **DF-34** `context.summarize` — truncation / `[SUMMARIZE_STUB]` canned response

**DF-37**: task-level stub fallback when worker handle missing should be reduced now that worker IPC is production-ready (V1.29 kb-extract path).

---

## 2. Goals

### J0 — Architect: worker-handle injection design

1. Define shared accessor injected into `JudgeLlm`, `ContextSummarize`, and optionally `AcpPrompt` standalone mode.
2. Pattern: registry holds `Arc<WorkerHandleProvider>` or reuses existing worker manager bridge from daemon boot.
3. Document in plan report before implement.

### J1 — `judge.rule` expression engine (DF-32)

1. Support rule expressions over `contextData` JSON, minimum set:
   - Field equality: `context.field == "value"`
   - Numeric compare: `context.count >= 1`
   - Boolean literals (retain `always_true` / `always_false`)
2. Return structured `{ result, reason }` on success; `InputInvalid` on unsupported syntax.
3. Pure evaluation — no LLM, no network.

### J2 — `judge.llm` real execution (DF-33)

1. Build judge prompt from input `prompt` field.
2. Call `acp.prompt` via worker IPC with `tool_policy: deny_all`.
3. Parse response with existing GO/NOGO word lists; error on ambiguous.
4. Remove heuristic that matches words in the **input** prompt instead of LLM response.

### J3 — `context.summarize` real execution (DF-34)

1. Build summarization prompt from `content`, optional `trace`, optional `template`.
2. Call `acp.prompt` via worker IPC.
3. Return `{ summary, prompt_hash }` where `prompt_hash` = blake3 of prompt sent.
4. Remove truncation-only fallback in production path (tests may keep fixture hook behind `cfg(test)` if needed).

### J4 — DF-37 task stub fallback cleanup

1. Audit `InnerGraphNodeTask` / `AcpPromptTask` paths in `tasks/mod.rs`.
2. When worker handle present, always use IPC; stub placeholder only when explicitly documented standalone/test mode.
3. Add test proving worker path used from capability layer.
