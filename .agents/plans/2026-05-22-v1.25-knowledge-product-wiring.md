## 1. Problem Statement

The V1.25 compass is the third full audit of knowledge specs versus crate/product wiring after V1.24 completed the normative doc refresh. It found a small set of post-V1.24 hygiene issues where prose, schema descriptions, tracker rows, or CLI spec labels still imply product wiring that is not currently shipped.

This locked plan starts with **Theme A only**: low-risk doc/schema/help hygiene with no behavior change. Themes B/C/D remain future implementation work and will be planned separately if PM chooses to ship those product surfaces.

---

## 2. Goals

1. Align schema descriptions and CLI-facing prose with the V1.24 decision that daemon `POST /v1/local/context/assemble` is retired.
2. Clarify Stage-0 / TwoStage CLI context assembly versus the library-only four-domain Moment assembly path.
3. Correct deferred tracker rows around orchestration engine wiring without implementing new behavior.
4. Mark `daemon schedule` as shipped and `daemon orchestrate *` as deferred/stubbed in the CLI spec.
5. Keep all changes constrained to documentation, schema prose, and clap help text; no runtime logic or API behavior changes.

---

## 3. Non-goals

- No new `platform context assemble-moment` command or flag.
- No product wiring for `nexus-narrative`, `nexus-kb`, or `nexus-knowledge` beyond existing compile-time reachability.
- No `creator kb --scope world` implementation and no World/User KB Local API redesign.
- No orchestration CLI session-control implementation for `daemon orchestrate *`.
- No schema version bump from prose-only schema description changes.
- No `nexus-platform` audit or private-repo edits.
