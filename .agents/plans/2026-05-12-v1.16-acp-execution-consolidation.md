# Plan: V1.16 ACP-first Execution Path Consolidation

**Plan ID**: `2026-05-12-v1.16-acp-execution-consolidation`
**Theme**: B2 — Execution path consolidation
**Milestone**: M3
**Effort**: L
**Depends on**: `2026-05-12-v1.16-daemon-acp-topology`
**Created**: 2026-05-12
**Status**: InReview

## Purpose

Consolidate execution paths so that **all agent execution flows through ACP**. Currently:

1. `agent run` in `agent.rs` (hidden, ~250 lines) spawns subprocesses directly — **bypasses ACP session/permission model**
2. `acp` top-level has `agent use` (stub) but no `run` equivalent — users must use hidden `agent run`
3. `session` (hidden, ~245 lines) manages ACP sessions independently from `acp`
4. `policy` (hidden, ~154 lines) manages global permission defaults independently from `acp`
5. `permission` (hidden, ~607 lines) manages per-agent permission rules independently from `acp`
6. `explore` (hidden, ~243 lines) does daemon-mediated platform reads — should be under `platform`
7. `doctor` (hidden, ~464 lines) is a standalone diagnostic — overlaps with `daemon doctor` + `acp doctor`

**Goal**: Move execution-relevant commands under `acp`, move platform reads under `platform`, and reduce hidden top-level clutter.

## Non-targets

- **Not** implementing full ACP `run` with SDK prompt integration (that requires `LocalSet` + `AcpSdkAdapter` — deferred to follow-up)
- **Not** removing hidden commands (backward compat) — only making them thin shims that delegate to V2 paths
- **Not** changing `acp_worker` internals (daemon-managed subprocess, not user-facing)

## Tasks

### T1: Add `acp session` subcommands (migrate from `session`)

Move `session list/show/delete` under `acp session list/show/delete`.

- Add `SessionCommand` variants to `AcpCommand` as `Session { #[command(subcommand)] command: SessionSubcommand }`
- `SessionSubcommand`: List, Show, Delete (same CLI args as current `SessionCommand`)
- `acp::run()` delegates to existing `session::run()` internals
- Keep `session.rs` as hidden top-level shim that prints deprecation notice and delegates to `acp session`

### T2: Add `acp policy` subcommands (migrate from `policy`)

Move `policy grant/deny/list` under `acp policy grant/deny/list`.

- Add `PolicySubcommand` to `AcpCommand` as `Policy { #[command(subcommand)] command: PolicySubcommand }`
- `PolicySubcommand`: Grant, Deny, List (same CLI args as current `PolicyCommand`)
- `acp::run()` delegates to existing `policy::run()` internals
- Keep `policy.rs` as hidden top-level shim → delegates to `acp policy`

### T3: Add `acp permission` subcommands (migrate from `permission`)

Move `permission list/grant/deny/ask/revoke/reset` under `acp permission list/grant/deny/ask/revoke/reset`.

- Add `PermissionSubcommand` to `AcpCommand` as `Permission { #[command(subcommand)] command: PermissionSubcommand }`
- All 6 subcommands (same CLI args as current `PermissionCommand`)
- `acp::run()` delegates to existing `permission::run()` internals
- Keep `permission.rs` as hidden top-level shim → delegates to `acp permission`

### T4: Add `acp run` command (promote from hidden `agent run`)

Promote `agent run` functionality to `acp run` — the primary ACP-first execution entry.

- Add `Run { agent_ref, message, cwd }` to `AcpCommand`
- Move the `cmd_run` implementation from `agent.rs` into `acp.rs` (it already uses `nexus_acp_host::transport::AgentSpawner`)
- Keep `agent run` as hidden shim → delegates to `acp run` implementation

### T5: Move `explore` under `platform explore`

The `explore` command does daemon-mediated platform reads (browse/search) — it belongs under `platform`.

- Add `Explore { #[command(subcommand)] command: ExploreSubcommand }` to `PlatformCommand`
- `ExploreSubcommand`: Browse, Search (same args)
- `platform::run()` delegates to existing `explore::run()` internals
- Keep `explore.rs` as hidden top-level shim → delegates to `platform explore`

### T6: Consolidate `doctor` diagnostics

`doctor` (hidden, 464 lines) overlaps with `daemon doctor` and `acp doctor`. Create a unified `system doctor` that runs all diagnostic checks.

- Add `Doctor` to `SystemCommand` — runs a combined diagnostic:
  1. Daemon connectivity (from `daemon::daemon_doctor`)
  2. ACP registry reachability (from `acp::cmd_doctor`)
  3. Home directory health
- Keep `doctor.rs` as hidden top-level shim → prints deprecation, suggests `system doctor`
- `daemon doctor` and `acp doctor` remain as focused sub-diagnostics (no change)

### T7: Update contract tests

Add V2 target tests for the new subcommand positions:

- `v2_target_acp_session_subcommands` — `acp session --help` shows list/show/delete
- `v2_target_acp_policy_subcommands` — `acp policy --help` shows grant/deny/list
- `v2_target_acp_permission_subcommands` — `acp permission --help` shows 6 subcommands
- `v2_target_acp_run` — `acp --help` shows `run`
- `v2_target_platform_explore_subcommands` — `platform explore --help` shows browse/search
- `v2_target_system_doctor` — `system --help` shows `doctor`

### T8: Deprecation notices

All hidden shims must print a one-line deprecation notice to stderr before delegating:

```
Note: `nexus42 session` is deprecated. Use `nexus42 acp session` instead.
```

Format: `eprintln!("Note: `nexus42 {old}` is deprecated. Use `nexus42 {new}` instead.");`

## Verification

1. `cargo test -p nexus42 --test command_surface_contract` — all pass (including 6 new)
2. `cargo clippy --all -- -D warnings` — clean
3. Hidden commands still work (backward compat) with deprecation notices
4. New `acp session/policy/permission/run` paths work identically to legacy paths
5. `platform explore` works identically to legacy `explore`
6. `system doctor` runs combined diagnostics

## Risk

- **Low**: All changes are delegation/shim pattern (same as Plan 2's `agent.rs` → `acp.rs` migration)
- **No runtime behavior change**: hidden commands delegate to same underlying code
