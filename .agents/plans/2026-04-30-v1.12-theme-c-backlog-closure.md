# V1.12 Theme C — Remaining Backlog Closure

## Summary

Close the five remaining backlog residuals assigned to V1.12 Theme C. The work addresses Windows build compatibility around `nix`, safer permission reset TOML manipulation, schema drift path derivation, graceful lifecycle shutdown, and drift detection test scalability.

## Background

- **V1.12 delivery compass**: `.agents/iterations/v1.12-delivery-compass-v1.md`
- **Theme C scope**: 5 remaining backlog residuals originally outside near-term scope but now pulled into V1.12.
- **Residual target**: R5, R11, R-M1-W02, R-M1-W07, R-M1-W09.
- **Execution intent**: Close backlog debt without introducing new features or broad refactors.

## Design Decisions

1. **Platform compatibility first**: Gate Unix-specific `nix` usage with `#[cfg(target_family = "unix")]` and provide non-Unix stubs where required.
2. **Permission reset reuse**: Refactor `run_reset` to call existing policy helper functions rather than performing raw `toml_edit` document surgery.
3. **Schema path truth source**: Derive schema drift paths from `build_schema_map()` or a deterministic glob instead of maintaining a second manual list.
4. **Graceful shutdown over abort**: Replace `JoinHandle::abort()` where feasible with a graceful shutdown signal such as `CancellationToken` or a `tokio::select!` shutdown channel.
5. **Drift test scale is documented**: Add a benchmark or time threshold so drift detection remains observable as schema count grows.

## Out of Scope

- New testing frameworks.
- Major lifecycle subsystem redesign beyond graceful shutdown for the recorded abort risk.
- Permission policy behavior changes unrelated to replacing raw TOML manipulation.
- New schema contract semantics.
