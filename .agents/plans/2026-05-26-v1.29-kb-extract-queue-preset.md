## 1. Problem Statement

Authors can add work-scope KB files and world-scope KeyBlocks separately, but there is **no agent-driven path** to promote/extract structured World KB from work index entries. Grill locked:

- **CLI**: queue + status only (deterministic).
- **LLM + business logic**: preset capability toolcalls (`kb.extract_work` + `acp_prompt`).

DF-35/36 (`acp_prompt`, `acp_session_load`) must be **partially de-stubbed** for this plan.

---

## 2. Goals

### B1 — Extract job queue (SQLite)

1. Migration: `kb_extract_jobs` (or equivalent) in `state.db` with: `job_id`, `creator_id`, `workspace_id`, `work_entry_id`, `world_id`, `status` (queued|running|done|failed), timestamps, optional error text.
2. SSOT in `nexus-local-db`; no second in-memory queue.

### B2 — CLI commands (frozen names)

1. `creator kb queue-extract <work-entry-id> --world-id <id>` — idempotent enqueue.
2. `creator kb extract-status [--job-id]` — show job(s) for active creator.
3. **No** CLI flags that call LLM providers.

### B3 — Capability `kb.extract_work`

1. Input: `job_id` or (`work_entry_id`, `world_id`).
2. Load work file content from work KB index path.
3. Call **`acp_prompt`** (real worker IPC) with structured prompt template; parse JSON response schema (block type, canonical name, body fields).
4. Insert KeyBlock via `SqliteKbStore` / narrative gateway for target world.
5. Update job status.

### B4 — Preset

1. Embedded preset `kb-extract` (or node in `_system` preset) invoking `kb.extract_work` for queued jobs.
2. Document manual run: `nexus42 daemon orchestrate run …` (exact command in spec plan).

### B5 — Wire acp_prompt + acp_session_load

1. Replace stub strings with worker-manager IPC path (reuse patterns from `acp_prompt_via_worker` tests).
2. Mark DF-35/36 **Shipped (partial)** in tracker when done.
