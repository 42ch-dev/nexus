## Task Group: Safety & Validation

### Task 1: Add file size limit to summary generator

**Residual:** CTX-R2 (low)

**Constraint:** See knowledge SSOT §9.1.1 — File Size Limit

**Files:**
- Modify: `crates/nexus42/src/context/summary.rs`

**Acceptance Criteria:**
- [ ] `SummaryConfig` struct with `max_file_size` field (default 10MB)
- [ ] `SummaryConfig::from_env()` reads `NEXUS_CONTEXT_MAX_FILE_SIZE` env var override
- [ ] `generate_summary()` checks file size before processing; skips oversized files with `tracing::warn!`
- [ ] Unit tests: skip large file, process small file, custom size limit

**Steps:**
- [ ] Implement `SummaryConfig` struct with `max_file_size`, `max_depth`, `include_extensions` fields and `Default` impl
- [ ] Implement `SummaryConfig::from_env()` for env var override of `max_file_size`
- [ ] Add file size check at the top of `generate_summary()` — skip with warning if exceeded
- [ ] Add unit tests in `summary.rs` or `summary_test.rs`

**Verification:**
```bash
cargo test -p nexus42 summary
```

**Commit:**
```bash
git commit -m "feat(context): add configurable file size limit to summary generator"
```

### Task 2: Add path traversal validation

**Residual:** CTX-R3 (low)

**Constraint:** See knowledge SSOT §9.1.2 — Path Traversal Validation

**Files:**
- Modify: `crates/nexus42/src/context/summary.rs`

**Acceptance Criteria:**
- [ ] `validate_path_within_base()` canonicalizes both path and base, checks containment
- [ ] Symlink escape is detected (canonicalization resolves symlinks)
- [ ] `SummaryError::PathTraversal` error variant with path and base fields
- [ ] `scan_recursive()` calls `validate_path_within_base()` at each level
- [ ] Unit tests: reject `..` traversal, reject symlink escape, accept valid path

**Steps:**
- [ ] Implement `validate_path_within_base(path, base) -> Result<PathBuf, SummaryError>` with canonicalization
- [ ] Add `SummaryError::PathTraversal { path, base }` and `SummaryError::InvalidPath(String)` variants
- [ ] Update `scan_recursive()` to validate every resolved path before processing
- [ ] Add unit tests in `summary.rs` or `summary_test.rs`

**Verification:**
```bash
cargo test -p nexus42 path
```

**Commit:**
```bash
git commit -m "feat(context): add path traversal validation to scan_recursive"
```

### Task 3: UTF-8 truncation safety tests

**Residual:** CTX-R5 (high) — decision: accept, needs tests

**Constraint:** See knowledge SSOT §9.1.3 — UTF-8 Truncation Safety

**Files:**
- Modify: `crates/nexus42/src/context/summary.rs` (add tests)

**Acceptance Criteria:**
- [ ] `truncate_at_char_boundary(s, max_bytes) -> &str` implemented or verified correct
- [ ] Tests cover: ASCII, CJK (3-byte chars), emoji (4-byte chars), mixed content, empty string, exact boundary, beyond length
- [ ] All results pass `is_char_boundary()` check

**Steps:**
- [ ] Implement or verify `truncate_at_char_boundary()` — walk backwards from `max_bytes` to find last valid char boundary
- [ ] Add truncation test module with all edge cases from §9.1.3 acceptance criteria
- [ ] Ensure opening excerpt generation in `generate_basic_summary()` uses `truncate_at_char_boundary()` instead of raw byte slicing

**Verification:**
```bash
cargo test -p nexus42 truncation
```

**Commit:**
```bash
git commit -m "test(context): add comprehensive UTF-8 truncation safety tests"
```
