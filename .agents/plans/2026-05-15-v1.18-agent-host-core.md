## Objective

Create `crates/nexus-agent-host` implementing a Hybrid host core (ACP + Native CLI adapters) in Managed-only mode. This is the architecture defined in spec Section 10.

## Scope

### In Scope (Wave 1 per spec Section 10.17)

1. Create `crates/nexus-agent-host/` library crate
2. Implement `core/` layer: session state machine, worker lease, policy, admission control
3. Implement `capability/` layer: normalized operation model, feature negotiation
4. Implement `providers/acp_provider/` adapter
5. Implement one native CLI provider adapter (e.g., Claude CLI)
6. Implement `discovery/`: managed-only provider discovery
7. Implement `telemetry/`: structured events, correlation
8. Implement `policy/`: host-level admission and execution policy
9. Define public facade traits in `lib.rs`
10. Wire into `nexus-daemon-runtime` as subsystem
11. Expose host API routes under `/v1/local/agent-host/` namespace
12. Config contract under `~/.nexus42/`

### Out of Scope

- Remaining native adapters (Codex, Gemini, OpenCode, Cursor, Kimi)
- Production hardening (Wave 4)
- Unmanaged session attach mode
- Provider-specific UX optimization
