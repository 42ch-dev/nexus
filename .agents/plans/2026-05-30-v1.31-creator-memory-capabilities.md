## 1. Problem Statement

Orchestration built-in capabilities `creator.read_memory`, `creator.write_memory`, and `creator.inject_prompt` return synthetic output (`count: 0`, `stub-fragment-id`, `queued: true` without persistence). V1.29 shipped real SQLite-backed creator memory via `nexus-creator-memory`, but orchestration capabilities were not wired. V1.31 P1 closes **DF-30** so preset graphs can use real memory I/O.
