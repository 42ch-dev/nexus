- `T11` / D-006 is lowest priority and can be deferred to V1.20 without blocking V1.19 functional completeness if Batch 2 runs long.

## Risks / Decisions Required

1. **D-001 Claude CLI multi-turn feasibility**: Current default `--print` mode is single-turn. Persistent stdin may be infeasible; fallback is resume/session-id continuity across invocations.
2. **D-002 ACP permission surfacing**: `AcpStreamUpdate` exposes only text/stopped and `AcpSdkAdapter` currently uses a no-op permission handler. Implementation likely requires changing `nexus-acp-host` before `AcpProvider` can evaluate permissions.
3. **D-003 config option IDs**: `set_config_option()` exists, but `SetModel { model }` / `SetMode { mode }` need a deterministic mapping from model/mode values to ACP config option IDs.
4. **D-009 compass line drift**: The Unix-only `which` command is in `claude.rs:183-210`; `path_scan.rs:40-104` does manual PATH scanning but still needs Windows-aware executable resolution.
5. **D-010 facade limitation**: Current delete handler shuts down the whole host and only echoes the path string. Returning 404 for valid-but-missing session IDs may require facade support beyond pure path validation.
