**Tech Stack**: Rust, existing graph-flow + orchestration runtime, `handlebars`, `blake3` for prompt-template hashing, `ulid` for `ScheduleId`.

- [ ] **Step 2: Implement `admit()` + `RunningSet` + basic supervisor loop**

Per [knowledge/creator-schedule-and-core-context-v1.md](knowledge/creator-schedule-and-core-context-v1.md) §5.1:
- `RunningSet` holds currently-`Running` Schedules (for the scope in question — per-creator).
- `admit(candidate, running)` returns `bool` per the rules in §5.1.
