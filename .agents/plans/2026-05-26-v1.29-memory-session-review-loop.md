## 1. Problem Statement

Daemon routes for `memory_pending_review` exist and `session_capture` can POST pending rows, but the **CLI author loop is incomplete**:

1. No `creator memory pending list|show|dismiss` (or equivalent) for authors to inspect queue without raw HTTP.
2. Session → pending → `memory review` promotion path is not covered by an integration test.
3. `assemble-moment` does not yet prove that reviewed memory surfaces in Stage0 after the loop.

---

## 2. Goals

### A1.1 — CLI pending-review surface

1. Add `MemoryCommand` variants (frozen names):
   - `PendingList` — list pending rows for active creator (optional `--world-id` filter)
   - `PendingShow { pending_id }` — show digest/metadata
   - `PendingDismiss { pending_id }` — delete pending row without promotion
2. Implement via `DaemonClient` against existing Local API routes (no new daemon routes unless gap found).
3. Document commands in plan acceptance; full cli-spec update deferred to spec-tracker plan (merge last).

### A1.2 — Session capture hardening

1. Ensure `session_capture` writes consistent `task_kind`, `world_id`, `raw_digest` for testability.
2. Wire capture trigger from existing agent/session path if not already invoked on session end (audit `session_capture` call sites).

### A1.3 — Review loop integration

1. Extend or add test: capture → `pending list` → `memory review` → long-term file exists.
2. Optional: `assemble-moment` includes new memory slug in Stage0 output after review.
