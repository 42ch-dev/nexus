**Goal**: Introduce the orchestration runtime in a new crate `crates/nexus-orchestration`, with: `graph-flow = "=0.2.3"` adapted behind a trait; SQLite `SessionStorage` sharing `nexus-local-db`'s `Arc<SqlitePool>` (post-WS8); capability trait + the non-ACP built-in registry; per-creator Worker Manager + stdin/stdout JSON-RPC IPC; `_system.maintenance` hardcoded graph running end-to-end; and `/v1/local/orchestration/*` HTTP endpoints. ACP-touching capabilities + preset file loader land in WS3.

**Architecture**:
- New crate `crates/nexus-orchestration/` ŌĆö the engine library.
- `OrchestrationEngine` trait (adapter layer) wraps `graph_flow::FlowRunner`.
- `SqliteSessionStorage: graph_flow::SessionStorage` ŌĆö sqlx-backed, consumes `Arc<sqlx::SqlitePool>` from `nexus-local-db`; `orchestration_sessions` schema added as a new `.sql` migration under `crates/nexus-local-db/migrations/`.
- `Capability` trait + `CapabilityRegistry`; first built-ins: `sync.*`, `outbox.*`, `workspace.*`, `registry.refresh`, `creator.read_memory`, `creator.write_memory`, `creator.inject_prompt`, `judge.rule`. `acp.*` + `judge.llm` deferred to WS3.
- `WorkerManager` ŌĆö spawns/supervises `nexus42 acp-worker` children, parent-owned stdin/stdout JSON-RPC 2.0 pipes via `jsonrpsee-core` + `RpcTransport` trait.
- `nexus42d` wires the engine at startup (statig-free for now ŌĆö WS4 brings the HSM).
- HTTP: new handlers under `crates/nexus42d/src/api/handlers/orchestration/` for the engine-session endpoints.

**Tech Stack** (decisions closed per [knowledge/crate-selection-best-practices-v1.md](knowledge/crate-selection-best-practices-v1.md) ŌĆö no selection work in this plan):
- `graph-flow = "=0.2.3"` (no `rig` feature), `async-trait` (workspace)
- **DB**: `sqlx` workspace dep (`runtime-tokio`, `sqlite`, `macros`, `migrate`, `chrono`, `uuid`) ŌĆö Task 3 consumes `nexus_local_db::open_pool` / `run_migrations` per ┬¦3.2 of best-practices
- **IPC**: `jsonrpsee-core` + `RpcTransport` trait + NDJSON framing via `tokio_util::codec::LinesCodec` ŌĆö Task 5 is pure **implementation** (selection closed in best-practices ┬¦3.1)
- `tokio` multi-thread on the daemon side, `serde`, `thiserror`

**Hard prerequisite**: **Task 3 depends on WS8** (`2026-04-17-v1.4-ws8-local-db-sqlx-migration`) Tasks 1ŌĆō2 having landed ŌĆö the shared `SqlitePool` + unified `crates/nexus-local-db/migrations/` directory are delivered by WS8.
