> **For agentic workers:** Ordered checklist of bite-sized tasks. Commit after each task unless noted.

**Plan ID**: `2026-04-17-v1.4-ws2-orchestration-skeleton`
**Wave**: V1.4 WS2 — see [knowledge/v1.4-delivery-compass-v1.md](knowledge/v1.4-delivery-compass-v1.md) §4 WS2
**Wave-0 spec (authoritative)**: [knowledge/orchestration-engine-v1.md](knowledge/orchestration-engine-v1.md) (primary) + [knowledge/acp-client-tech-spec-v2.md](knowledge/acp-client-tech-spec-v2.md) §4.3, §6 (IPC) + [knowledge/schemas-boundary-v1.md](knowledge/schemas-boundary-v1.md) §3 (local type placement)
**Working branch**: `create feature/v1.4-ws2-orch-skeleton from feature/v1.4-ws1-acp-host` (stacks on WS1 once that merges; before then, treat WS1 branch as base).
**Owner candidate**: `fullstack-dev` primary; consider splitting off a parallel `fullstack-dev-2` track for the HTTP endpoint layer.

**Goal**: Introduce the orchestration runtime in a new crate `crates/nexus-orchestration`, with: `graph-flow = "=0.2.3"` adapted behind a trait; SQLite `SessionStorage`; capability trait + the non-ACP built-in registry; per-creator Worker Manager + stdin/stdout JSON-RPC IPC; `_system.maintenance` hardcoded graph running end-to-end; and `/v1/local/orchestration/*` HTTP endpoints. ACP-touching capabilities + preset file loader land in WS3.

**Architecture**:
- New crate `crates/nexus-orchestration/` — the engine library.
- `OrchestrationEngine` trait (adapter layer) wraps `graph_flow::FlowRunner`.
- `SqliteSessionStorage: graph_flow::SessionStorage` — schema added under `crates/nexus-local-db/migrations/`.
- `Capability` trait + `CapabilityRegistry`; first built-ins: `sync.*`, `outbox.*`, `workspace.*`, `registry.refresh`, `creator.read_memory`, `creator.write_memory`, `creator.inject_prompt`, `judge.rule`. `acp.*` + `judge.llm` deferred to WS3.
- `WorkerManager` — spawns/supervises `nexus42 acp-worker` children, parent-owned stdin/stdout JSON-RPC 2.0 pipes.
- `nexus42d` wires the engine at startup (statig-free for now — WS4 brings the HSM).
- HTTP: new handlers under `crates/nexus42d/src/api/handlers/orchestration/` for the engine-session endpoints.

**Tech Stack**: Rust workspace crate, `graph-flow = "=0.2.3"` (no `rig` feature), `sqlx` (workspace version, SQLite), `tokio` multi-thread on the daemon side, `serde`, `thiserror`, `jsonrpsee` or hand-rolled JSON-RPC framing (decide in Task 5).

Design notes:
- Line-delimited JSON over parent↔child stdin/stdout.

- Supervise loop: if `Child::wait()` returns unexpectedly, emit `WorkerEvent::Crashed { pid, exit_status }` to the manager's broadcast channel.
- 30-second default timeout per request.
