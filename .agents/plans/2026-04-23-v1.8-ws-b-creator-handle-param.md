## §1 Overview

Add optional `--handle` parameter to `creator register` command. The frozen spec v3 §7 defines handle as optional at registration, 4–15 chars matching `[a-z0-9-_.]`. This parameter passes through to the platform `POST /creators/register` request body.

**Effort (agent-oriented)**: S (~0.5–1 agent session)
**Dependencies**: WS-A and WS-C should complete first (stable CLI surface before adding new parameter).
**Platform dependency**: ~~Platform V1.8 Theme A must be complete (registration endpoint accepts handle)~~ ✅ **Resolved** (2026-04-23): Platform V1.8 Plan 64 (Creator Handle) is Done. `POST /creators/register` now accepts optional `handle` field with `handleSchema` validation (regex + reserved words).

## §2 Scope

### In scope

- Add `--handle <handle>` optional flag to `creator register` subcommand in `crates/nexus42/src/commands/creator.rs`.
- Client-side format validation: match frozen spec regex `^[a-z0-9][a-z0-9._-]{2,13}[a-z0-9]$` (4–15 chars).
- Pass handle to platform API `POST /creators/register` request body.
- Add unit tests for handle validation (valid, too short, invalid chars).

### Out of scope

- Handle change/update (immutable once set — not a CLI operation).
- Handle availability check via platform API before registration (platform handles uniqueness server-side).
