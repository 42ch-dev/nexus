## 1. Problem Statement

- `platform context assemble` exits with code 1 and obsolete V1.10 message — poor UX for a **deferred** platform feature.
- `assemble-moment` exists but is **`hide = true`** and uses in-memory demo stores only.
- cli-spec may still overclaim four-domain assembly on the default `assemble` path.

PM policy: **no new platform HTTP** in V1.26; all context productization is **local**.

---

## 2. Goals

1. **Promote** `assemble-moment` to a visible subcommand with help text stating local four-domain assembly.
2. Wire `assemble_moment` to **persistent** `NarrativeGateway`, `KbStore`, and knowledge slice (in-memory knowledge acceptable for V1.26 if no User knowledge SQLite — document limitation).
3. Replace `platform context assemble` behavior with **deferred** message (exit 0 with stderr guidance **or** exit 2 with clear error — pick one in C1 and document; default: exit 2 + message, no panic).
4. Update cli-spec + LCCA §3.6 + schema descriptions (Theme D / carry V1.25 Theme A items).
5. Integration test: after narrative/KB seed, `assemble-moment` output contains world/KB sections.

---

## 3. Non-goals

- Implement platform HTTP for `ContextCommand::Assemble`.
- Daemon `/v1/local/context/assemble` (remains retired).
- Remove `assemble-local` or change Stage0/TwoStage semantics.
- Cloud-stage platform enhancement path changes beyond existing `TwoStageAssembly` fallback.
