## 1. Problem Statement

V1.29 shipped session review loop plumbing, but QC found:

| ID | Issue |
| --- | --- |
| R14 | `SessionCapture` is created in `handle_agent_stop` only — duration and message/tool counts are always ~zero |
| R17 | Persistent native child `Drop` cleanup is best-effort and Unix-only (follow-up to R11/R12 fixes) |

Authors need meaningful session metrics in pending-review records. Operators need documented, reliable child teardown on shutdown paths.

---

## 2. Goals

### A1 — SessionCapture lifecycle (R14)

1. Create `SessionCapture` at session start (`initialize` / `agent_start` — store in session state).
2. Call `record_message()` / `record_tool_call()` (or equivalent) on relevant ACP IPC events during the session.
3. On `agent_stop`, call `capture_session_end()` and `spawn_submit_capture` with populated digest (not a fresh empty capture).
4. Unit or integration test: non-zero message/tool counts when events are simulated.

### A2 — Persistent child cleanup (R17)

1. Retain child handles where needed so `Drop` can SIGTERM → wait → SIGKILL on Unix.
2. Document Unix scope and non-Unix limitation in code comment + agent-host spec snippet if required.
3. Avoid blocking the runtime unbounded in `Drop` (keep best-effort; prefer explicit `shutdown()` path).

---

## 3. Non-goals

- New ACP providers or wire contract changes.
- Platform HTTP memory routes.
- Full cross-platform process tree kill (document limits only).
