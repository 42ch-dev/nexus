**Goal**: Replace hardcoded `_system.maintenance` with configurable `~/.nexus42/presets/_system/` directory.

**Tasks**:

- T1: Define system preset directory convention (`~/.nexus42/presets/_system/<name>/`).
- T2: Engine startup scans `_system.*` directories and registers all discovered presets.
- T3: Extract `_system.maintenance` from hardcoded Rust into directory bundle. First-start fallback auto-creates from embedded content.
- T4: Graceful handling: missing directory = no system presets; corrupted = log warning + skip.
- T5: Implement `nexus42 system preset list` command.

**Evidence**:
- Engine startup with `_system/maintenance/`: preset registered and functional.
- Engine startup with additional `_system/health-check/`: both presets registered.
- Engine startup with no `_system/`: no presets, no error.
- `nexus42 schedule start _system.maintenance` identical to pre-V1.6 behavior.
- `cargo test --workspace` green; clippy/fmt clean.
