## 1. Vision & Target User

### Problem Statement

Creative writers working with AI agents lack a **CLI-native, open-source platform** that combines world-building structure, manuscript lifecycle management, and seamless ACP agent integration. Current tools (Obsidian, Notion, Scrivener) are GUI-first, proprietary, and lack native AI agent protocol support. Nexus42 V1.0 built the foundation but left most user-facing commands as skeleton implementations — users hit "V1.0 skeleton" messages at every turn.

### Target User Segments


| Segment                                    | Characteristics                                                                | Primary Need                                                             |
| ------------------------------------------ | ------------------------------------------------------------------------------ | ------------------------------------------------------------------------ |
| **Primary**: ACP Developer                 | Builds writing tools on ACP, comfortable with Rust/CLI, early adopter          | Test ACP integration patterns, contribute to open-source                 |
| **Primary**: Technical Writer              | Writes documentation or technical content, uses CLI daily, wants AI assistance | Manage manuscript phases, sync with team, run agents for editing         |
| **Secondary**: Creative Writer (Technical) | Writes fiction/non-fiction, tolerant of CLI, curious about AI                  | Track world-building, manage draft→publish lifecycle, get agent feedback |


### Value Proposition

**For technical writers and ACP developers**, Nexus42 V1.1 is a **CLI-first creative writing platform** that **integrates 16+ ACP AI agents directly into your manuscript workflow**, unlike Obsidian/Notion/Scrivener which are GUI-only and lack native agent protocol support.

### One-Sentence Pitch

> "Nexus42 V1.1 is the first CLI-native writing platform with built-in AI agent integration — manage your manuscript lifecycle, sync with your team, and get AI feedback without leaving the terminal."

## 5. Architecture Approach

### 5.1 Goals & Non-Goals

#### Goals

1. **Eliminate all CRITICAL and HIGH architectural blockers** (CLI-DAEMON-1–5, DEBT-X1–X4, CC-ERR-1, CODEGEN-1/2, SYNC-ARCH-1)
2. **Make ACP agent interaction functional** — `nexus42 agent run <agent>` must send prompts and receive responses
3. **Deliver a complete user workflow**: init → create manuscript → manage phases → run agent → sync to platform
4. **Standardize error propagation** with structured error codes and proper HTTP status codes
5. **Unify SQLite schema management** — single source of truth, shared between CLI and daemon
6. **Enforce schema-first contract discipline** — all wire types from JSON Schema codegen

#### Non-Goals

1. Platform API integration (sync push/pull) — only CLI-side preparation in scope
2. GUI frontend — V1.1 remains CLI-only
3. Multi-agent orchestration — single-agent interaction target; multi-agent is V1.2+
4. Binary distribution (Homebrew/npm) — separate ops effort
5. End-user documentation — separate plan
6. `!Send` futures migration to `sacp` — staying on `agent-client-protocol` v0.10.4

### 5.2 Key Technical Decisions


| Decision                     | Options                                          | Recommendation                                  | Rationale                                                                            |
| ---------------------------- | ------------------------------------------------ | ----------------------------------------------- | ------------------------------------------------------------------------------------ |
| Connection pool library      | deadpool-sqlite, r2d2_sqlite, hand-rolled        | `**deadpool-sqlite`**                           | Most actively maintained; async-native; works with tokio                             |
| Error strategy               | Unified thiserror, hybrid, custom error codes    | **Unified `thiserror` + `NexusErrorCode` enum** | Domain and sync already use thiserror; add error code enum for API boundary          |
| Schema dedup location        | New crate, daemon-only, shared module            | **Shared module in existing crate**             | New crate for 3 table definitions is overkill; use `crates/nexus42/src/db/schema.rs` |
| Workspace validation         | Axum middleware, per-handler, tower layer        | **Axum middleware (tower layer)**               | Catches all routes uniformly; easy to compose                                        |
| LocalSet bridge architecture | Dedicated OS thread, spawn_local, spawn_blocking | **Dedicated `std::thread` with `LocalSet`**     | SDK futures are `!Send`; dedicated thread is standard pattern                        |
| Bridge channel type          | tokio mpsc, std mpsc, flume                      | **tokio mpsc (async) + std mpsc (sync)**        | Async side needs tokio; LocalSet thread can't await on tokio channels                |
| ACP error handling           | String wrapping, typed enum, anyhow              | **Typed `AcpError` enum + `From<SDK Error>`**   | Matches existing trait pattern; enables structured recovery                          |
| Auth state storage           | Shared crate, CLI-only, daemon-only              | **Daemon owns auth state**                      | Per architecture review §9.4.3: "Daemon owns all SQLite state"                       |
| OAuth device flow            | RFC 8628, custom                                 | **RFC 8628**                                    | Standard device authorization grant                                                  |
| Token refresh strategy       | Lazy on 401, proactive timer, hybrid             | **Lazy on 401 + proactive if expiry < 5min**    | Simple; avoids latency spikes                                                        |

## 7. Contract Alignment & Cross-Repo Coordination

### Contract Principles

1. **Wire types from `@42ch/nexus-contracts`**: Do not redefine existing types
2. **Enum values frozen**: Per v1-spec domain data model §7
3. **Schema-driven codegen**: Any schema change must update `schemas/` and run `pnpm run codegen`
4. **Backward compatibility**: V1.0 wire types must remain compatible

### Architecture Principles (from V1.0-phase1 Review §9.4)

1. Every new type must come from schemas — no hand-written DTOs bypassing codegen
2. Error types must be structured — no anyhow at API boundaries
3. Daemon owns all SQLite state — CLI accesses workspace data through daemon API
4. HTTP status codes must be meaningful — proper 4xx/5xx, not always 200

## 9. Risks & Mitigation


| Risk                                                                                                      | Severity | Mitigation                                                                                                                                                     |
| --------------------------------------------------------------------------------------------------------- | -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ACP SDK breaking changes** — `agent-client-protocol` v0.10.4 may be superseded by `sacp` v1.0           | HIGH     | Adapter pattern (`NexusAcpClient` trait) isolates SDK to `AcpSdkAdapter`; pin exact version `=0.10.4`; migration path: create `SacpSdkAdapter` with same trait |
| **SQLite concurrency under sync daemon loops** — concurrent requests may cause `SQLITE_BUSY`              | MEDIUM   | `deadpool-sqlite` connection pool (Plan A); WAL mode; configurable pool size (default 8); fallback to `r2d2_sqlite` if problematic                             |
| **Schema-first enforcement drift** — developers continue hand-writing DTOs                                | MEDIUM   | Plan E fixes codegen pipeline; CI `verify-codegen` catches drift; add grep for `serde_json::Value` in CI; document in CONTRIBUTING.md                          |
| **Error strategy inconsistency** — new features perpetuate existing mixed patterns                        | MEDIUM   | Plan A establishes canonical `NexusApiError`; add `#[warn(clippy::unwrap_used)]`; QC checklist for new handlers; rollback: scope to API boundary only          |
| **Context assembly types violating schema-first** — CLI hand-wrote types that may conflict with generated | MEDIUM   | Plan E Task 2 replaces hand-written with generated; compare shapes before migrating; rollback: `#[deprecated]` annotation                                      |
| **Platform API contract drift** — CLI expects daemon/platform routes that don't exist                     | HIGH     | Plan C registers missing route; API contract tests for each daemon route; not in scope: platform-side validation                                               |
