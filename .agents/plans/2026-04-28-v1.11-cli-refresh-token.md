# V1.11 — CLI refresh_token Support

## Summary

Extend CLI authentication to support OAuth2 refresh tokens. When `access_token` expires, automatically use `refresh_token` to obtain a new token pair without requiring user re-authorization. Develop in parallel with platform V1.11 Theme F which adds `refresh_token` to the `/auth/device/token` response.

## Background

- **Platform V1.11 Theme F**: Will add `refresh_token` and `refresh_expires_at` fields to `POST /api/v1/auth/device/token` response. Platform and CLI develop in parallel.
- **Current auth flow**: CLI polls `POST /auth/device/token` during device code flow. On success, only `access_token` is stored. When token expires, user must re-run `nexus42 auth login`.
- **Problem**: Token expiry (typically 1 hour) forces frequent re-authorization. Refresh tokens (longer-lived) eliminate this friction.
- **V1.10 shipped**: `AuthStore` with `user_token: Option<UserTokenState>` holding `access_token`, `token_type`, `expires_at`, `user_id`. `user_auth.rs` handles login/status/logout.
- **Design compass**: `.agents/knowledge/v1.11-delivery-compass-v1.md`

## Design Decisions

1. **AuthStore extension**: Add `refresh_token: Option<String>` and `refresh_expires_at: Option<DateTime<Utc>>` to `UserTokenState`. Use `#[serde(default)]` for backward compatibility — CLI ignores `refresh_token` field until platform delivers it.
2. **Polling loop save**: In `user_auth.rs`, after successful device code polling, persist `refresh_token` and `refresh_expires_at` from the token response alongside existing `access_token`.
3. **Proactive refresh**: Before any authenticated CLI request, check if `access_token` expires within 60 seconds. If so and `refresh_token` exists, call `POST /auth/device/token` with `grant_type=refresh_token` to obtain a new token pair.
4. **Graceful degradation**: No `refresh_token` → current behavior (prompt re-authorization). Refresh failed → clear tokens, prompt re-authorization.
5. **No new dependencies**: Uses existing HTTP client (`reqwest`) and auth infrastructure.
6. **Parallel with platform**: CLI implements storage and logic first. Serde default ensures CLI works whether or not platform has delivered `refresh_token` yet.
