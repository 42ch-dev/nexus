## 1. Objective

Deliver the **first full implementation** of local/cloud hard isolation and the crate split described in the long-term spec:

1. Remove platform sync and registration from daemon Local API.
2. Rename / split crates so names reflect product lines (`nexus-cloud-sync`, `nexus-cloud-domain`, `nexus-creator`, …).
3. Enforce **contracts-first** types in `nexus-creator` and `nexus-cloud-domain` (no duplicate platform DTOs).
4. Focus engineering on **local product** capability; cloud sync remains behind `legacy-sync`, archived until redesigned.

### 1.1 Non-goals (this iteration)

- Production-ready redesigned bundle sync (only archive + `legacy-sync` gate).
- Local web UI.
- New platform HTTP endpoints in nexus repo.
- Full `nexus-runtime-policy` crate (may defer; stub cloud degradation off daemon path only).
- Renaming daemon `/kb/entries` → `/knowledge/entries` unless trivially aligned (optional stretch).

## 3. Deliverables (this program)

### 3.1 Crate graph (target workspace members)

| Crate | Action |
| --- | --- |
| `nexus-cloud-sync` | Rename from `nexus-sync`; `legacy-sync` feature for outbox/push/pull |
| `nexus-cloud-domain` | **New name** for narrowed ex-`nexus-domain` (User, Pairing logic only) |
| `nexus-creator` | Extract Creator local logic; **wire types from `nexus-contracts` only** |
| `nexus-creator-memory` | Rename from `nexus-memory`; depend on `nexus-creator` |
| `nexus-kb` | Narrative KeyBlock + SourceAnchor |
| `nexus-knowledge` | `reference_source` + global KB; owns `GET /references` handler logic |
| `nexus-narrative` | World / timeline / story / manuscript / consistency |
| `nexus-moment-context-assembly` | Extract `context_assembly`; default build without `cloud-stage` |
| `nexus-domain` | **Remove** from workspace after split (no long-term retention of name) |
| `nexus-daemon-runtime` | Drop `nexus-cloud-sync` dependency; remove sync + registration routes |

### 3.2 Contracts discipline (acceptance theme)

- [ ] `nexus-creator` public API uses `nexus_contracts` types for Creator (and related wire/local contract types); grep for duplicate struct definitions vs `generated/` and `local/`.
- [ ] `nexus-cloud-domain` uses contract types for User/Pairing; no parallel platform DTOs.
- [ ] `nexus-cloud-sync` HTTP bodies use generated or documented contract types only.

### 3.3 Daemon Local API (this iteration)

**Remove routes:**

- `/v1/local/sync/*`
- `/v1/local/creators/registrations`, `/registrations/{code}:verify`

**Keep (minimum):** per long-term spec §5; verify against `api/mod.rs` after edits.

**Handler moves:**

- Registration → `nexus42` commands → `nexus-cloud-sync::platform_client`
- Sync → CLI → `nexus-cloud-sync` with `legacy-sync` (no daemon HTTP)

### 3.4 CLI

- [ ] `nexus42 creator register|verify` calls cloud-sync directly
- [ ] `nexus42 sync` does not call removed daemon routes (cloud-sync or deprecation message)
- [ ] `Cargo.toml` workspace deps updated (`nexus-sync` → `nexus-cloud-sync`)

### 3.5 Orchestration

- [ ] `sync.pull`, `sync.push`, `outbox.flush` capabilities: stub on default build
- [ ] `nexus-orchestration` does not depend on `nexus-cloud-sync` without explicit non-default feature

### 3.6 Documentation

- [ ] Long-term spec stays principle-only (no phase tables)
- [ ] Update [cli-spec.md](../knowledge/specs/cli-spec.md) registration path (same PR program or follow-up task in plan)

## 5. Acceptance criteria (program Done)

1. `cargo tree -p nexus-daemon-runtime` shows **no** `nexus-cloud-sync` and **no** `nexus-cloud-domain`.
2. `api/mod.rs` has **no** §3.3 forbidden route families.
3. Workspace has **`nexus-cloud-domain`**; **no** `nexus-domain`, **no** `nexus-sync`, **no** `nexus-memory`.
4. `nexus-creator-memory` → depends on `nexus-creator`; references live in `nexus-knowledge`.
5. `cargo test --workspace` and `cargo clippy --all -- -D warnings` pass.
6. QC: spot-check `nexus-creator` / `nexus-cloud-domain` for contracts-first (no duplicate platform structs).
