## 2. Architecture

### 2.1 Capability wiring (constructor injection)

Follow `KbExtractWork::with_pool` pattern — extend `CapabilityRegistry::with_builtins_and_pool` (or adjacent factory) to inject:

```text
CapabilityRegistry
  ├── CreatorReadMemory / WriteMemory / InjectPrompt
  │     └── nexus-creator-memory store (creator_id from input / schedule context)
  ├── JudgeRule
  │     └── pure evaluator (no LLM)
  ├── JudgeLlm / ContextSummarize
  │     └── delegate to AcpPrompt via shared worker-handle accessor (DF-37)
  └── existing real caps (kb.extract_work, acp.prompt, soul.experience.aggregate, …)
```

**Invariant**: Capabilities never import LLM provider crates directly; LLM paths use `acp.prompt` IPC only.

### 2.2 Reflection loop preset (pattern 1)

```text
state: draft
  enter: acp.prompt (generate draft)
  exit_when: llm_judge (judge.llm) — "quality sufficient?"
  next: revise

state: revise
  enter: acp.prompt (apply critique)
  exit_when: llm_judge OR max_iterations rule
  next: summarize

state: summarize
  enter: context.summarize
  next: done (terminal)
```

### 2.3 Memory-augmented preset (pattern 2)

```text
state: recall
  enter: creator.read_memory (keyword from preset.input)
  exit_when: rule (judge.rule — memory count > 0 OR allow_empty)
  next: generate

state: generate
  enter: acp.prompt (prompt includes memory context)
  next: persist

state: persist
  enter: creator.write_memory (store output fragment)
  next: done (terminal)
```
