## 0. Position

V1.25 closed the **Cargo vs knowledge doc** drift and added **thin** product wiring (in-memory narrative routes, experimental `assemble-moment`, world KB list against empty in-memory stores). V1.26 makes those surfaces **durable and honest**:

1. **Persistence** — narrative and World KB data survive daemon restarts and are shared across CLI/daemon in a workspace.
2. **Reference model** — SQLite holds registration metadata; MD files hold canonical body text under `~/.nexus42`.
3. **Local context** — promote four-domain Moment assembly as a first-class **local** command wired to persistent stores; keep platform context assembly **deferred** (no new platform client work).
4. **Platform pause** — preserve `nexus42 platform` command group and contracts for future use; do not implement or extend platform HTTP in this iteration.

When this compass conflicts with an active locked plan, the active plan wins for delivery batching. When it conflicts with outdated normative prose, update specs in Theme D (or the reference plan for layout).

## 2. Reference storage layout (normative draft)

Operational partition uses **active Creator** under User scope (see [entity-scope-model.md](../knowledge/specs/entity-scope-model.md)). Pre-release: layout may change without migration; wipe acceptable.

```text
~/.nexus42/
  creators/<creator_id>/
    SOUL.md
    references/                              # User reference corpus (creator partition)
      units/<reference_source_id>/
        body.md                              # Canonical text body (SSOT for content)
    workspaces/<workspace_slug>/
      state.db                               # SQLite: reference_sources + narrative + world_kb tables
      kb/                                    # Work-scope file index (unchanged — NOT nexus-kb World KB)
        index.json
        entries/
```

### 2.1 Registry row (`reference_sources`)

| Field | Role |
| --- | --- |
| `reference_source_id` | Primary key |
| `workspace_id` | Binding (default `local` or active workspace slug per policy) |
| `source_type` | `file`, `url`, `pdf`, `note`, … (contract enum) |
| `source_mutability` | **`static`** — body fixed after registration; **`refreshable`** — scan/refresh may update `body.md` and `content_hash` |
| `uri` | Logical locator (`nexus42://references/units/<id>` or original import URI) |
| `title`, `tags`, `scan_status`, `content_hash` | Index metadata |
| `content_path` | Relative path from creator root, e.g. `references/units/<id>/body.md` |
| `content` | **Deprecated** — NULL for new rows; migration clears inline blobs |

### 2.2 Minimal product API (Theme R)

| Surface | Behavior |
| --- | --- |
| `GET /v1/local/references` | List registry metadata (existing; extend columns) |
| `GET /v1/local/references/{id}` | Optional — single registry row |
| CLI `creator reference register \| list \| show` | TBD in reference plan — at minimum **list + show body** for V1.26 |

## 3. SQLite persistence (compass-level)

All narrative and World KB tables live in **workspace** [`state.db`](../../crates/nexus-home-layout/src/lib.rs) (`$HOME/.nexus42/creators/<creator_id>/workspaces/<slug>/state.db`). Owner module: **`nexus-local-db`** migrations; domain adapters in **`nexus-narrative`** / **`nexus-kb`**.

| Domain | Tables (draft names) | Crate adapter |
| --- | --- | --- |
| Narrative | `narrative_worlds`, `narrative_timeline_events`, … | `SqliteNarrativeGateway` or repository behind `NarrativeGateway` |
| World KB | `kb_key_blocks`, `kb_source_anchors`, … | `SqliteKbStore` implementing `KbStore` |

**Runtime rule**: `WorkspaceState` (or boot-injected handle) holds `Arc<dyn NarrativeGateway>` and `Arc<dyn KbStore>` opened once per active workspace pool — **never** construct empty in-memory gateways inside HTTP handlers.

Detailed DDL and migration IDs: [2026-05-23-v1.26-narrative-kb-persistence.md](../plans/2026-05-23-v1.26-narrative-kb-persistence.md).
