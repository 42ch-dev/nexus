# V1.4 WS1 — `nexus-acp-host` crate extraction

> **For agentic workers:** This plan is expressed as an ordered checklist of bite-sized tasks. Execute task-by-task; commit after each task unless noted.

**Plan ID**: `2026-04-17-v1.4-ws1-acp-host-extraction`
**Wave**: V1.4 WS1 (see [.mstar/iterations/v1.4-delivery-compass-v1.md](.mstar/iterations/v1.4-delivery-compass-v1.md) §4 WS1)
**Wave-0 spec (authoritative)**: [archived/archived/knowledge/acp-client-tech-spec.md](archived/archived/knowledge/acp-client-tech-spec.md) §11
**Working branch**: `create feature/v1.4-ws1-acp-host from main`
**Branch policy**: feature branch with standard QC三审 + QA (PM policy).
**Owner candidate**: `fullstack-dev` (Rust crate extraction; no UI).

**Goal**: Extract the ACP client implementation out of `crates/nexus42/src/acp/` into a new `crates/nexus-acp-host` crate (worker-only consumer + interactive CLI consumer). Enforce via CI that `nexus42d` never links it, to prevent future LocalSet contagion into the daemon's axum runtime.

**Architecture**: `nexus-acp-host` is a "host library" for ACP (registry fetch + cache, subprocess transport with `tokio::task::LocalSet`, `NexusAcpClient` adapter). Two binaries consume it: the `nexus42` CLI (existing `agent list/show/run/probe`) and a new hidden `nexus42 acp-worker` subcommand (Phase-1 stub; WS2 fills it in). The crate guarantees `nexus42d` never holds an `AcpSession` Rust value.

**Tech Stack**: Rust workspace crate, `agent-client-protocol = "=0.10.4"` pinned in the new crate, `tokio` with `LocalSet`, `reqwest` for registry fetch, `serde` for cache.
