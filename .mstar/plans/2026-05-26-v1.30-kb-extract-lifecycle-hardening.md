## 1. Problem Statement

V1.29 shipped `kb_extract_jobs`, CLI `queue-extract` / `extract-status`, and a registered `kb.extract_work` capability with embedded `kb-extract` preset. QC left four findings:

| ID | Issue |
| --- | --- |
| R15 | `next_queued` + `mark_running` are not atomic — concurrent workers can double-claim |
| R16 | `kb.extract_work` returns `prompt_prepared` placeholder; no job lifecycle or KeyBlock insert |
| R18 | Job IDs use timestamp-derived hex, not UUIDv4-style |
| R20 | `list_by_creator` / `extract-status` list is unbounded |

Grill locked **R16 full e2e**: claim → mark_running → acp.prompt → parse → KeyBlock insert → mark_done/failed.

---

## 2. Goals

### K1 — Atomic claim (R15)

1. Add `claim_job(pool, creator_id) -> Option<KbExtractJob>` in one SQLite transaction (SELECT oldest `queued` + UPDATE `running` + set `started_at`).
2. Deprecate or wrap separate `next_queued` + `mark_running` for worker paths.
3. Concurrent tokio test: two claimers, only one succeeds per job.

### K2 — UUID job IDs (R18)

1. Replace `generate_job_id()` with `uuid` crate (already in workspace) + preserve `xj_` prefix.
2. Handle PRIMARY KEY collision with domain error or single retry.

### K3 — Bounded listing (R20)

1. `list_by_creator(pool, creator_id, limit)` with default cap (e.g. 100).
2. CLI `extract-status` without `--job-id` uses bounded list; document limit in help.

### K4 — Full e2e `kb.extract_work` (R16)

1. When `job_id` present: load job row; reject wrong status.
2. `mark_running` → load work content → invoke **acp.prompt** path (not placeholder) → parse JSON → insert KeyBlock via `SqliteKbStore` → `mark_done` or `mark_failed`.
3. When `job_id` omitted: call `claim_job` for `creator_id` then same pipeline.
4. Update `kb-extract` preset and/or the shipped schedule orchestration entry path so queued jobs can run without manual `preset.input.*` for every field.

### K5 — Verification

1. Integration or manual: `queue-extract` → orchestrate preset → `extract-status` shows `done` → `creator kb list --scope world` shows block.
