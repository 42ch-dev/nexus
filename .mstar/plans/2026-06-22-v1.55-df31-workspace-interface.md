## Goal

Close or materially narrow DF-31 by introducing a local workspace-session interface skeleton for `workspace.open` and `workspace.commit` that future capabilities can build on without redesigning the entire Local API.

## Scope In

- Define skeleton contracts for `workspace.open` and `workspace.commit`.
- Implement local bounds/session scaffolding if needed by current crate boundaries.
- Document conflict behavior (session id, snapshot, commit precondition, stale/conflict response).
- Add tests for path bounds, session lifecycle, and commit conflict rejection.

- DF-42 full Local API redesign.
- MCP server or external protocol exposure.
- Multi-branch graph runtime / DF-56 implementation.
- Cloud sync or platform publish integration.
