## 1. Problem Statement

Starting a novel Work today jumps from brief intake into `novel-writing` without a dedicated **project bootstrap** phase: `work_ref`, planned chapter count, directory scaffold, and high-level creative direction are implicit or demo-grade. Reference systems use a grill-me init flow; Nexus needs an orchestration preset, not a CLI wizard.

Round 3 expansion (2026-06-07): the spec now **enumerates** the full scaffold protocol (§5.4) and the **gate enforcement** for downstream presets (§5.3). The init preset must (a) actually create the documented files, (b) seed `work_chapters` rows, and (c) declare its own permissive gates.

---

## 2. Goals

1. Add embedded preset `novel-project-init` with `run_intents: [work_init]` and `gates:` per [novel-writing/workflow-profile.md §5.3.1](../knowledge/specs/novel-writing/workflow-profile.md).
2. ACP multi-turn grill-me collects: working title, genre/tone hooks, `total_planned_chapters`, `work_ref` slug, **World binding question** (existing world_id / create new World / worldless, per §3.5).
3. On success: **enumerated scaffold** per [novel-writing/workflow-profile.md §5.4](../knowledge/specs/novel-writing/workflow-profile.md) (mkdir dirs + copy template stubs + seed `work_chapters` rows 1..N + PATCH `works` table). All atomic; idempotent on re-init.
4. Persist `work_profile: novel`, `work_ref`, `total_planned_chapters`, `world_id` on Work via Local API.
5. CLI: `creator run start` may offer `--init-preset novel-project-init` or auto-suggest when `novel-writing` gate (§5.3.2) fails on `filesystem` check.
6. No auto-chain into `novel-writing`; downstream gate enforces the manual step.

---

## 3. Non-goals

- Auto-chain into `novel-writing` (gate enforces manual step)
- Platform publish IDs (DF-59)
- Automatic switch to next novel after completion (DF-59)
- `creator world create` CLI command (V1.37+; for V1.36, "create new World" grill-me option may prompt for world metadata inline and persist via a future API; document the gap)

### 4.2 Scaffold capability — file enumeration (T2)

The init preset's scaffold capability creates the canonical tree per [novel-writing/workflow-profile.md §5.4](../knowledge/specs/novel-writing/workflow-profile.md). Each sub-task lists the source template path and the destination path. All writes go through `creator.workspace.transaction` (or equivalent) for atomicity.

- [ ] T2a: `mkdir Works/<work_ref>/` (root)
- [ ] T2b: copy `embedded-presets/novel-project-init/templates/README.md` → `Works/<work_ref>/README.md`; render with `{{work_ref}}`, `{{title}}`, `{{world_id_or_null}}`; for worldless Works include 1–2 paragraph inline world setting note (collected from grill-me); for World-bound Works include one-line `world_id: <uuid>` + links
- [ ] T2c: `mkdir Works/<work_ref>/Outlines/`
- [ ] T2d: `mkdir Works/<work_ref>/Outlines/chapters/` (empty dir; first outline created on first outline pass)
- [ ] T2e: copy `embedded-presets/novel-project-init/templates/volume-outline.md` → `Works/<work_ref>/Outlines/volume-outline.md`; V1.36 single-volume placeholder (still created)
- [ ] T2f: copy `embedded-presets/novel-project-init/templates/foreshadowing.md` → `Works/<work_ref>/Outlines/foreshadowing.md` (F### table header; empty stub)
- [ ] T2g: copy `embedded-presets/novel-project-init/templates/event-index.md` → `Works/<work_ref>/Outlines/event-index.md` (E### table header; empty stub)
- [ ] T2h: `mkdir Works/<work_ref>/Stories/`
- [ ] T2i: `mkdir Works/<work_ref>/Logs/` (empty; structure OUT V1.36 per DF-66)

**Explicitly NOT created** (per spec §5.4.1): `work-status.md` (replaced by `work_chapters` table §4.1), `Worldbuilding/` subtree (content lives in World KB §3.5).

### 4.3 `work_chapters` row seeding (T3)

- [ ] T3: For `chapter IN 1..total_planned_chapters`, insert one `work_chapters` row per [novel-writing/workflow-profile.md §5.4.3](../knowledge/specs/novel-writing/workflow-profile.md):
  - `work_id`, `chapter`, `volume = NULL`, `slug` (user-provided or auto `ch<NN>`), `planned_word_count` (default `4000`, user may override per chapter in grill-me), `actual_word_count = NULL`, `status = 'not_started'`, `outline_path = "Works/<work_ref>/Outlines/chapters/ch<NN>-outline.md"`, `body_path = "Works/<work_ref>/Stories/ch<NN>-<slug>.md"`, `created_at`/`updated_at` = now()
  - On PK conflict (re-init case), preserve existing row (do not overwrite)
  - All inserts in a single DB transaction; rollback on any failure

### 4.4 `works` table PATCH (T4)

- [ ] T4: PATCH `works` table per [novel-writing/workflow-profile.md §5.4.4](../knowledge/specs/novel-writing/workflow-profile.md):
  - `work_profile = 'novel'`, `work_ref = <chosen>`, `total_planned_chapters = <N>`, `world_id = <from grill-me or NULL>`, `current_chapter = 0` (reset on fresh init), `updated_at = now()`
  - PATCH fields user did NOT change in this grill-me session are no-ops (idempotent)

### 4.5 CLI wiring (T5)

- [ ] T5: CLI wiring in `creator/run.rs`:
  - `creator run start --init-preset novel-project-init` (explicit)
  - `creator run start --init-preset auto` → if `novel-writing` gate would fail on `filesystem` check, auto-suggest `novel-project-init` (UX-only, not behavior change)
  - `--force-gates --reason "<text>"` override flag (V1.36 Master; passed through to engine)

### 4.6 Idempotency + re-init (T6)

- [ ] T6: Idempotency per [novel-writing/workflow-profile.md §5.4.5](../knowledge/specs/novel-writing/workflow-profile.md):

  - Existing files NOT overwritten unless grill-me answers "overwrite templates = yes" (default: preserve)
  - `work_chapters` rows NOT duplicated (PK conflict preserves existing)
  - `works` PATCH is a no-op for unchanged fields

## 5. Acceptance

1. `creator run start --init-preset novel-project-init --idea "..." --world-id <existing|new|none>` creates the full `Works/<work_ref>/` tree per [novel-writing/workflow-profile.md §5.4.1](../knowledge/specs/novel-writing/workflow-profile.md) — including `Outlines/foreshadowing.md`, `Outlines/event-index.md`, `Outlines/volume-outline.md` (T2).
2. `work_chapters` table contains exactly `total_planned_chapters` rows for the Work, all with `status='not_started'` and correct `outline_path`/`body_path` (T3).
3. `works` table PATCHed with `work_profile='novel'`, `work_ref=<chosen>`, `world_id` matches grill-me (T4).
4. Init preset does **not** enqueue `novel-writing` automatically; subsequent `creator run stage advance --stage produce` (or explicit `novel-writing` schedule) **passes** the §5.3.2 gate (T7e).
5. Re-running init on the same Work does **not** overwrite existing files (T6, T7b).
6. Init preset declares `gates:` per §5.3.1 in `preset.yaml`; load-time validation passes (T1).
7. `cargo test` on touched crates passes (T7).
