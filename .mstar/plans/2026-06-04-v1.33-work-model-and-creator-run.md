## 1. Problem Statement

Users cannot run a complete narrative cycle without knowing `daemon schedule`, preset IDs, and seed strings. There is no **Work** container for long-term goals, structured briefs, or inspiration supplements. `system preset list` only shows `_system.*` presets, hiding `novel-writing` and other embedded assets.

### 7.1 DDL â€” `works` table (idempotent migration)

**Migration file**: `crates/nexus-local-db/migrations/20260604_v133_works_table.sql`

**DDL** (matches `pending_review.rs` style; uses compile-time-checked sqlx for DML, raw for DDL):

```sql
-- V1.33: Work entity (per work-experience-model Â§3.2)
CREATE TABLE IF NOT EXISTS works (
    work_id          TEXT PRIMARY KEY,
    creator_id       TEXT NOT NULL,
    workspace_slug   TEXT NOT NULL,
    status           TEXT NOT NULL
                      CHECK (status IN ('draft','active','paused','completed','archived')),
    title            TEXT NOT NULL,
    long_term_goal   TEXT NOT NULL,
    initial_idea     TEXT NOT NULL,
    creative_brief   TEXT,                  -- JSON text, nullable until intake complete
    intake_status    TEXT NOT NULL
                      CHECK (intake_status IN ('pending','in_progress','complete','skipped')),
    world_id         TEXT,                  -- nullable
    story_ref        TEXT,                  -- nullable
    inspiration_log  TEXT NOT NULL DEFAULT '[]',  -- JSON text, append-only
    primary_preset_id TEXT NOT NULL DEFAULT 'novel-writing',
    schedule_ids     TEXT NOT NULL DEFAULT '[]',  -- JSON text
    created_at       TEXT NOT NULL,         -- ISO 8601
    updated_at       TEXT NOT NULL          -- ISO 8601
);

CREATE INDEX IF NOT EXISTS idx_works_creator_status
    ON works (creator_id, workspace_slug, status);

CREATE INDEX IF NOT EXISTS idx_works_creator_intake
    ON works (creator_id, workspace_slug, intake_status);

CREATE INDEX IF NOT EXISTS idx_works_creator_updated
    ON works (creator_id, workspace_slug, updated_at DESC);
```

| # | Method | Path | Purpose |
|---|--------|------|---------|
| 1 | `POST` | `/v1/local/works` | Create Work (idempotent on `client_request_id`) |
| 2 | `GET`  | `/v1/local/works` | List Works (filters: `status`, `intake_status`, `limit`, `offset`) |
| 3 | `GET`  | `/v1/local/works/{work_id}` | Get one Work |
| 4 | `PATCH`| `/v1/local/works/{work_id}` | Partial update (brief/intake/inspiration/title/goal/etc.) |
| 5 | `POST` | `/v1/local/works/{work_id}/inspiration` | Append-only `inspiration_log` entry (atomic) |
| 6 | `GET`  | `/v1/local/presets` | **Existing** â€” extend response to include `run_intents: Vec<String>` per preset |
| 7 | `POST` | `/v1/local/presets:validate` | **Existing** â€” extend validation to enforce `run_intents` rules (see Â§7.4) |

**Request/response shape** (Rust structs in handler; mirrors nexus-contracts style
â€” **no handwritten DTOs duplicate generated types**; local handler types live
in `nexus-daemon-runtime`, not `nexus-contracts`):

```rust
// POST /v1/local/works
#[derive(Deserialize)]
pub struct CreateWorkRequest {
    pub title: String,
    pub long_term_goal: String,
    pub initial_idea: String,
    pub world_id: Option<String>,
    pub story_ref: Option<String>,
    pub primary_preset_id: Option<String>,  // default "novel-writing"
    /// If provided and a Work with the same creator + client_request_id exists,
    /// return the existing work_id (idempotent).
    pub client_request_id: Option<String>,
}

#[derive(Serialize)]
pub struct WorkRecord {
    pub work_id: String,            // "wrk_<uuid>"
    pub creator_id: String,
    pub workspace_slug: String,
    pub status: String,
    pub title: String,
    pub long_term_goal: String,
    pub initial_idea: String,
    pub creative_brief: Option<serde_json::Value>,
    pub intake_status: String,
    pub world_id: Option<String>,
    pub story_ref: Option<String>,
    pub inspiration_log: Vec<InspirationLogEntry>,
    pub primary_preset_id: String,
    pub schedule_ids: Vec<String>,
    pub created_at: String,
    pub updated_at: String,
}

#[derive(Serialize, Deserialize, Clone)]
pub struct InspirationLogEntry {
    pub at: String,    // ISO 8601
    pub note: String,
}
```

**Status codes**:

- `201 Created` for successful create (with `Location: /v1/local/works/{work_id}` header)
- `200 OK` for successful get/list/patch/inspire/validate
- `400 Bad Request` for schema violation
- `401 Unauthorized` when no active creator in context
- `404 Not Found` for unknown `work_id`
- `409 Conflict` for `client_request_id` collision (use existing record's id)

**Idempotency**: if `client_request_id` is provided and a Work already exists
for `(creator_id, client_request_id)`, return that Work with `200 OK` (not `201`).
Storage approach: add `client_request_id TEXT` column to `works` table (nullable,
UNIQUE per `(creator_id, client_request_id)`); if `NULL`, generated server-side.

> **Refinement**: keep the DDL simple for V1.33 â€” add a separate
> `works_idempotency` table `(creator_id, client_request_id, work_id, created_at)`
> with `UNIQUE(creator_id, client_request_id)` to avoid polluting `works`.
> Add migration `20260604_v133_works_idempotency.sql`.

### 7.4 Preset manifest `run_intents` extension

**Closed enum** (5 values, defined in work-experience-model Â§5.1):

| Value | Meaning |
|-------|---------|
| `work_init` | Allowed to start a new Work (creator preset) |
| `work_continue` | Allowed on an existing Work; may append context |
| `knowledge_ingest` | Reference / KB pipeline presets |
| `work_maintenance` | Work-adjacent non-narrative upkeep (e.g. `soul-experience-refresh`) |
| `system_maintenance` | `_system.*` only |

**Hand-written types** (NOT codegen â€” see `crates/nexus-orchestration/src/preset/manifest.rs`):

The preset manifest types live in `crates/nexus-contracts/src/local/orchestration/preset.rs`
(re-exported via `nexus_orchestration::preset::manifest`). This is a
**hand-maintained** local type, not generated from `schemas/`. The
`generated/` directory in `nexus-contracts` is reserved for cloud-sync /
platform types only. Adding `run_intents` to `PresetHeader` is a direct
struct edit â€” no `pnpm run codegen` step.

**Concrete change** to `crates/nexus-contracts/src/local/orchestration/preset.rs`:

1. Add new enum (next to `PresetKind`):

   ```rust
   /// Preset run-intent classification (V1.33 work-experience-model Â§5.1).
   /// Closed enum â€” unknown strings cause loader validation errors.
   #[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, Serialize, Deserialize)]
   #[serde(rename_all = "snake_case")]
   pub enum RunIntent {
       /// May start a new Work (creator preset).
       WorkInit,
       /// May run on an existing Work; may append inspiration / context.
       WorkContinue,
       /// Reference / KB pipeline presets.
       KnowledgeIngest,
       /// Work-adjacent non-narrative upkeep (e.g. soul-experience-refresh).
       WorkMaintenance,
       /// `_system.*` only.
       SystemMaintenance,
   }
   ```

2. Add `pub run_intents: Vec<RunIntent>` to `PresetHeader` (default `#[serde(default)]`).
   Place it after `requires_capabilities` for YAML readability.

**Loader validation** (extend `crates/nexus-orchestration/src/preset/validation.rs`):

- Add a new `DiagnosticCategory::RunIntentsInvalid` for unknown intent strings (ERROR).
- Add a new check: every `kind: creator` preset must have **at least one** of
  `work_init` or `work_continue` (otherwise ERROR â€” the preset is unreachable
  via `creator run`).
- `kind: system` presets may have `run_intents = []` (or `system_maintenance`).
- This is wired into both `load_preset_from_str` (loader) and
  `POST /v1/local/presets:validate` (daemon) via the existing
  `validation::validate_preset` facade from V1.32 P1.
