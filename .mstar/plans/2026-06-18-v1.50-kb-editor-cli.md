## 1. Problem Statement

V1.40 shipped cross-Work World KB but author surface is read-only (`creator world show/list`). Authors cannot inspect/edit/delete KB rows without direct DB access.

---

## 2. Goals

1. CLI `creator world kb list <world_ref>` shows all KeyBlocks with id / canonical_name / block_type / status.
2. CLI `creator world kb show <world_ref> <block_id>` shows full body + provenance + status.
3. CLI `creator world kb edit <world_ref> <block_id> --body ...` edits body in place.
4. CLI `creator world kb delete <world_ref> <block_id>` deletes with confirmation prompt (or `--yes`).
5. Author identity check: edits/deletes require `works.creator_id` match (no cross-author).

---

## 3. Non-goals

- KB promotion (T-B P1).
- Refreshable scan (T-B P2).

---

## 4. Acceptance Criteria

1. CLI commands render correctly; `--json` output for `list` and `show`.
2. Cross-author edit returns `403` with stable error code.
3. Delete with confirmation prompt; `--yes` skips.
4. Hermetic test: list/show/edit/delete round-trip with fresh `handler_state()`.
5. Validation re-runs on edit (`ValidationMode::Novel`); invalid bodies return `ValidationError`.

`ValidationMode::Novel` validation on edit.

### Design decisions (documented in code)

- **Author identity**: `KeyBlock`s are World-scoped; the only ownership field is
  `narrative_worlds.owner_creator_id` (no direct `works.creator_id` linkage on
  `kb_key_blocks`). Edit/delete therefore gate on the **world owner** matching the
  active creator. The assignment/spec phrasing "works.creator_id" was resolved to
  the world owner as the only coherent ownership path. Stable error code
  `WORLD_KB_FORBIDDEN` embedded in the `403` message.
- **Novel validation on edit**: `kb_edit` constructs
  `SqliteKbStore::with_validation_mode(pool, ValidationMode::Novel)` so
  `update_key_block` re-runs the V1.40 P1 novel-profile validation.
- **Hermetic testability**: logic functions are `pub` and take an explicit pool so
  the integration tests use `crate::db::Schema::init(temp)` + public seed helpers
  (`nexus_local_db::kb_store::seed`) ÔÇö no `$HOME` resolution, no daemon. The
  author-check query uses runtime `sqlx::query_scalar` with a SAFETY comment,
  matching the existing `world.rs` pattern.
