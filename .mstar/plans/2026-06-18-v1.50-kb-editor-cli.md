## 1. Problem Statement

V1.40 shipped cross-Work World KB but author surface is read-only (`creator world show/list`). Authors cannot inspect/edit/delete KB rows without direct DB access.

---

## 2. Goals

1. CLI `creator world kb list <world_ref>` shows all KeyBlocks with id / canonical_name / block_type / status.
2. CLI `creator world kb show <world_ref> <block_id>` shows full body + provenance + status.
3. CLI `creator world kb edit <world_ref> <block_id> --body ...` edits body in place.
4. CLI `creator world kb delete <world_ref> <block_id>` deletes with confirmation prompt (or `--yes`).
5. Author identity check: edits/deletes require `works.creator_id` match (no cross-author).

---

## 3. Non-goals

- KB promotion (T-B P1).
- Refreshable scan (T-B P2).

---

## 4. Acceptance Criteria

1. CLI commands render correctly; `--json` output for `list` and `show`.
2. Cross-author edit returns `403` with stable error code.
3. Delete with confirmation prompt; `--yes` skips.
4. Hermetic test: list/show/edit/delete round-trip with fresh `handler_state()`.
5. Validation re-runs on edit (`ValidationMode::Novel`); invalid bodies return `ValidationError`.
