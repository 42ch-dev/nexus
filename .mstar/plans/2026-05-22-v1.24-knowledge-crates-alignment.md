## 1. Problem Statement

V1.23 closed Cargo-level wiring and introduced [entity-scope-model.md](../knowledge/specs/entity-scope-model.md). A **2026-05-22 re-audit** shows normative knowledge and several product paths still describe **pre-V1.23 reality**:

- [local-cloud-crate-architecture.md](../knowledge/specs/local-cloud-crate-architecture.md) §4 claims compile-only islands and missing `cloud-domain` edges that **already exist** in `Cargo.toml`.
- [local-runtime-boundary.md](../knowledge/specs/local-runtime-boundary.md) lists Local API routes as Active that are **not registered** in `nexus-daemon-runtime/src/api/mod.rs` (e.g. `POST /v1/local/context/assemble`).
- Daemon `/v1/local/kb/*` and CLI `creator kb` implement the **work file index**, not **World KB** (`nexus-kb`) — route naming contradicts [entity-scope-model.md §5.3](../knowledge/specs/entity-scope-model.md#53-cli-creator-kb--local-work-scope-file-index).

[docs/ARCHITECTURE.md](../../docs/ARCHITECTURE.md) was refreshed in the audit pass to separate **Cargo wiring** vs **product integration**; this plan drives normative spec parity and closes the highest-severity product gaps (KCA-002, KCA-003).

---

## 2. Goals

1. Refresh normative OSS specs so §4 “current Cargo” matches verified `cargo tree` (2026-05-22).
2. **KCA-002 (locked B2):** Retire `POST /v1/local/context/assemble` from the Local API contract; platform/creator context assembly runs **in-process** via `nexus-moment-context-assembly` on the CLI path (no daemon proxy).
3. **KCA-003 (locked C2):** Keep `/v1/local/kb/*` and `creator kb` paths for now but document and implement them explicitly as **`scope=work`** (CLI local work KB index only). Extract/refactor handler and CLI code so the work-scope implementation is isolated and labeled; **do not** imply `nexus-kb` (World KB). Full KB route redesign is deferred to a future iteration.
4. Register remaining audit items in [deferred-features-cross-version-tracker.md](../knowledge/deferred-features-cross-version-tracker.md) where not closed in this plan.
5. Pass QA + QC tri-review with evidence commands from the audit compass §6.

## 3. Non-goals

- Orchestration daemon lifecycle stubs (DF-38–DF-40) — tracker only, not in scope.
- Platform / `schemas/` changes.
