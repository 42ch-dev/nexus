## 1. Problem Statement

V1.50 P0 introduced `set_schedule_json_tx` as the first CAS pattern (atomic `SELECT-then-UPDATE-with-version` for `works.schedule_json`). T-B P1 generalises this pattern to additional mutating tables (`kb_extract_jobs`, `novel_pool_entries`) and surfaces explicit `E_VERSION` / `E_LOCK` stable CLI codes. Cron-side retry-on-conflict for transient CAS failures is added.

This complements T-B P0 advisory lock: advisory lock serialises *intra-Work* mutating paths at the file level; OCC detects *logical* state divergence when cron-side preimage becomes stale (e.g. daemon cron evaluator moves `kb_extract_jobs.status` between cron-fire attempts).

---

## 2. Goals

1. `version INTEGER NOT NULL DEFAULT 0` column added to `kb_extract_jobs` + `novel_pool_entries` (additive migration; defaults preserve existing rows).
2. CAS update pattern: `UPDATE ... WHERE id = ? AND version = ?`; returns `rows_affected = 0` on mismatch → caller maps to `E_VERSION`.
3. `E_VERSION` stable CLI code (in `nexus42::errors`) with exit code 76; surfaces "row was modified by another writer (holder=<id>); retry" to user.
4. Cron-side retry-on-conflict: `with_retry_cas(max_attempts=3, backoff_ms=100)` wrapper for cron-fire enqueue paths; logs warning on each retry.
5. CAS pattern documented in `knowledge/specs/concurrency.md` §7 (extension to T-B P0).
6. `kb_extract_jobs` writes from T-A P1 (cross-chapter upsert) and T-A P2 (missing-KB detection) use the CAS path.

---

## 3. Non-goals

- Advisory lock infra (T-B P0).
- All-write-paths audit (only high-risk cron-side + KB + pool paths).
- Distributed consensus (out of scope; local-only).
- New wire JSON schemas (V1.51 §0.1 #9).

## 4. Acceptance Criteria

1. `kb_extract_jobs.version` + `novel_pool_entries.version` columns added via additive DB migration; existing rows get `version=0` default.
2. CAS update pattern helper in `nexus-local-db`: `cas_update(table, id, expected_version, new_state) -> Result<(), E_VERSION>`; hermetic test asserts mismatch returns error.
3. `creator world kb adopt` on a stale preimage returns `E_VERSION` exit 76 with retry hint; hermetic test.
4. Cron-fire enqueue uses `with_retry_cas(3, 100)`; hermetic test asserts retry count + final success.
5. `world-kb-runtime-architecture.md` §6 OCC subsection body authored.
6. `concurrency.md` Master §7 OCC extension body authored.
