## 1. Problem Statement

V1.50 T-B P1 surfaces KB candidates only via `novel-review-master` review-time extraction. If an author finalizes a chapter that references an entity not yet in the World KB (and not picked up by review), the gap is silent. Authors discover the gap only when they manually read the chapter against `creator world kb list`.

T-A P2 introduces finalize-time **missing-KB detection**: when a chapter transitions to `finalized` state, the chapter text is scanned for entity references; referenced entities not in the World KB are surfaced as **missing** (separate from `pending` — these are *negative* signals).

---

## 2. Goals

1. Finalize-time entity reference detection: when `novel-writing` transitions a chapter from `draft` to `finalized`, scan the chapter text via the same LLM extraction pathway as T-A P0.
2. Diff against existing World KB: candidates not matching any `confirmed` KB row (canonical_name + world_id) are classified as `missing`.
3. New CLI flag `--missing-only` on `creator world kb pending` shows only `missing` candidates (separate UX surface from `pending`).
4. `missing` candidates are **advisory only** — they do not write to `kb_extract_jobs` (avoids spam) but appear in CLI output with a clear "missing" label.
5. Spec extension: `novel-writing/quality-loop.md` §5 quality loop + missing-KB detection paragraph.

---

## 3. Non-goals

- LLM extraction capability itself (T-A P0).
- Cross-chapter rescan (T-A P1).
- Auto-promotion of `missing` candidates (deliberate: human-in-loop).
- Write-time extraction (V1.51 §1.2 O12 — keep OUT).
- New wire JSON schemas (V1.51 §0.1 #9).

---

## 4. Acceptance Criteria

1. When a chapter transitions to `finalized`, the orchestration supervisor triggers missing-KB detection (reuses T-A P0 `LlmExtractTask`); hermetic test asserts detection on a known-finalized chapter.
2. `creator world kb pending` lists `pending` candidates (V1.50 behavior preserved).
3. `creator world kb pending --missing-only` lists `missing` candidates only; hermetic test seeds a chapter with 2 entities neither in KB nor pending; finalize surfaces 2 missing candidates.
4. `missing` candidates are **not** written to `kb_extract_jobs` (deliberate design choice; verified by hermetic test).
5. Spec overlay body authored:
   - `novel-writing/quality-loop.md` §5 missing-KB detection extension body.
   - `cli-spec.md` §6.2G `--missing-only` flag body.
