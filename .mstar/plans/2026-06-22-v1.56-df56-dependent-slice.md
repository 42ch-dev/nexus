## Goal

Complete the two remaining DF-56 sub-items that depend on P0 and P1: (1) `registry.refresh` conditional edges — routing decisions driven by registry capability freshness or availability; (2) `workspace.open`/`workspace.commit` branch inputs — routing decisions that inspect workspace session state, changes[] manifest, or OCC status — wiring the full conditional routing engine to real data sources.

## Scope In

- **`registry.refresh` conditional edges**: a stage node can declare a conditional `next` whose expression queries the output of `registry.refresh` (e.g. "if registry capability X is available, branch to Y; otherwise branch to Z"). Uses P1 synthetic output or `--cdn-url` real fetch.
- **`workspace.open`/`workspace.commit` branch inputs**: a stage node can declare a conditional `next` whose expression inspects workspace session state (e.g. "if changes[] contains a file matching pattern X, branch to review; otherwise skip"). Uses P0 persistent sessions and changes[] payload.
- Wire the expression language (P2) to the real context objects exposed by P0 (workspace sessions) and P1 (registry output).
- Update `entity-scope-model.md` for workspace-scoped branch input visibility and `cli-spec.md` if branch inputs need CLI surface.
- Tests: registry-driven edges with synthetic and mocked-CDN output; workspace-driven edges with mock sessions and changes[] payloads; integration tests with all three waves' subsystems combined.

## Scope Out

- New branching primitives beyond the P2 engine — P3 only wires existing P2 engine to P0/P1 data sources.
- Cross-daemon or cloud-sync workspace branch inputs.
- Platform-level routing or multi-user branch coordination.
- New Work profiles or preset authoring for conditional routing.
