## Goal

Coordinate the single-wave evidence gate across P0 (Track A) + P1 (Track B) parallel dispatch. Verify both tracks merged cleanly to `iteration/v1.60` before tri-review coordination. Track fix-wave cycles for each plan. No product code in P-mid itself.

---

## Scope

- **T1**: After P0 + P1 both report implement Done on their topic branches, verify clean merge to `iteration/v1.60`. Resolve any cross-track conflict (expected only at `preset_version_for_id` if both tracks amend it — see compass §4).
- **T2**: Dispatch tri-review for P0 (Track A) — three parallel QC specialists:
  - `@qc-specialist` (architecture/maintainability)
  - `@qc-specialist-2` (security/correctness)
  - `@qc-specialist-3` (performance/reliability)
  Each reviews at `iteration/v1.60` HEAD (post-P0-merge). Assignment uses `Working branch: iteration/v1.60`.
- **T3**: Dispatch tri-review for P1 (Track B) — same three parallel QC specialists, same `Working branch: iteration/v1.60` HEAD.
- **T4**: Consolidate tri-review verdicts per plan. For each plan: if all 3 Approve → mark plan QC Pass; if any Request Changes → trigger targeted fix-wave on that plan's topic branch.
- **T5** (conditional, per plan with Request Changes): Dispatch targeted fix-wave to plan owner (`@fullstack-dev` for P0; `@fullstack-dev-2` for P1). Fix-wave addresses only the cited findings (surgical, no piggyback). Re-merge to `iteration/v1.60`. Re-review only the citing QC specialist (targeted re-review).
- **T6**: Mid-QA gate — dispatch `@qa-engineer` to verify integrated `iteration/v1.60` HEAD: `cargo test --all`, `cargo clippy --all -- -D warnings`, `cargo +nightly fmt --all -- --check`. Verify Profile B compaction invariants hold. Report any regressions.
- **T7**: Once both plans QC Pass + mid-QA Pass, register new residuals from QC findings into `status.json.residual_findings` (severity per QC report). Update `status.json.plans[]` statuses to `QCPass` for P0 + P1.
- **T8**: PM signoff to proceed to P-last. Update `status.json` — P-mid status `Done`; P-last status `Todo` → `InProgress` when dispatched.

## Risks

- **Fix-wave scope creep**: Implementer may piggyback unrelated refactors into a fix-wave. Mitigation: fix-wave Assignment explicitly limits scope to cited QC findings; PM reviews fix-wave diff before re-merge.
- **Cross-track QC confusion**: Both P0 and P1 review at same `iteration/v1.60` HEAD — QC specialist may conflate findings across plans. Mitigation: Assignment explicitly states which plan's deliverables are in scope for that QC report; QC specialist scopes review to that plan's diff vs integration base.
- **Mid-QA regression discovery**: Integration of two parallel tracks may expose regression not caught in plan-level tests. Mitigation: mid-QA runs full workspace suite; any regression blocks P-last dispatch until resolved (either via additional fix-wave or accepted as residual with justification).
