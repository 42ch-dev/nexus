## 1. Problem Statement

V1.50 T-B P1 shipped auto-promotion with **heuristic** entity extraction (capitalized noun phrases → `block_type_guess='character'`). Authors must manually correct block_type on adopt for every non-character entity (location, organization, event). Heuristic cannot distinguish categories. R-V150KBED-01 was deferred to V1.51 LLM-driven extraction.

V1.51 introduces a new `nexus.llm.extract` orchestration capability that mirrors the existing `nexus.llm.judge` pattern but emits `Vec<KbCandidate>` instead of `GoNogo`. This unlocks accurate block_type classification at review-time extraction.

---

## 2. Goals

1. New `nexus.llm.extract` capability registered in `nexus-orchestration::capability` registry.
2. New `LlmExtractTask` task mirroring `LlmJudgeTask` lifecycle (input gather → LLM call → result parse → contract dispatch).
3. `kb_extract_jobs.proposed_payload` JSON extended with `block_type` (LLM-judged), `canonical_name` (LLM-extracted), `confidence` (0.0-1.0), `source_quote` (verbatim chapter excerpt).
4. `embedded-presets/novel-review-master/` heuristic extraction replaced with `LlmExtractTask` invocation.
5. `creator world kb adopt <id>` surfaces `confidence` + `source_quote` for human confirmation.
6. Closes R-V150KBED-01.

---

## 3. Non-goals

- Cross-chapter reconciliation (T-A P1 scope).
- Missing-KB detection (T-A P2 scope).
- New LLM worker pool (reuse existing V1.32 worker pool + V1.39 routing).
- Write-time extraction (V1.51 §1.2 O12).
- New wire JSON schemas (V1.51 §0.1 #9 — local-only Rust types).
