## 1. Scope

Extend worker `agent_tool_request` IPC from the V1.42 P3 narrow slice (1 ID: `nexus.orchestration.schedule_status`) to all 35 shipped catalog IDs. Build a hermetic cross-caller E2E test harness covering each of the 35 IDs via all 3 caller paths (CLI `host-call`, worker IPC, HTTP path) = 105 invocation cases. Verify `nexus.profile.{minimal,writer,publisher}` registration completeness.

### In scope

- **Worker IPC extension**: Register all 35 shipped `nexus.*` IDs in the worker tool-allowlist. The worker entry point (`agent_tool_request` IPC) can now dispatch any of the 35 IDs through `capability::Registry::dispatch`.
- **Cross-caller E2E test harness**: One parametrized test (or test suite) that exercises each of the 35 IDs via:
  - CLI `host-call` subcommand (P1).
  - Worker `agent_tool_request` IPC (orchestration → daemon).
  - HTTP `ToolExecuteRequest` (daemon HTTP endpoint).
  - Total: 35 × 3 = 105 invocation cases.
- **Profile-set registration completeness**: Verify that `nexus.profile.minimal`, `nexus.profile.writer`, and `nexus.profile.publisher` are correctly registered as §3.3 metadata profiles (not action IDs), and that the roster in `acp-capability-set.md` §4 tags them correctly.
- **Spec amendment**: `orchestration-engine.md` §6.4 updated to reflect that all 35 IDs (not just 1) are callable via worker IPC.

### Out of scope

- New capability implementations (all 35 already shipped; P3 only extends the caller path).
- Registry consolidation (P0).
- God-file refactor (P1).
- Schema rename (P2).
- Production caller performance testing (gated by mid-QA, not P3 scope).
