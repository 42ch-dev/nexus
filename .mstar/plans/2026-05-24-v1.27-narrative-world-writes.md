## 1. Problem Statement

V1.26 persisted narrative and World KB **reads** (`SqliteNarrativeGateway`, `SqliteKbStore`, daemon GET, `creator kb --scope world` list/search/show). There is **no product write path**: worlds and events are only created via test `seed::*` helpers; `creator kb add/remove` for `--scope world` prints a deferred message.

Authors cannot build local state for `assemble-moment` without internal seeds.

---

## 2. Goals

### Theme N — Narrative writes

1. `nexus-local-db` **write repository** (`narrative_write` or equivalent): `create_world`, `append_timeline_event` (promote logic from `seed_shared` / `narrative_gateway::seed`).
2. CLI subcommand tree `creator world`:
   - `create` — allocate `wld_*`, insert `narrative_worlds` + default `root_fork_branch_id`
   - `event add` — append row to `narrative_timeline_events` with monotonic `sequence_no` per `(world_id, branch_id)`
   - `list` — list worlds in active workspace DB (may share read path with gateway)
3. **`NarrativeGateway` remains read-only** — writes go through local-db repo, not trait extension.
4. Integration test: create world + event → `SqliteNarrativeGateway::list_worlds` / daemon GET after restart.

### Theme K — World KB writes

1. Wire `creator kb add --scope world --world-id <id>` to `SqliteKbStore::insert_key_block` (and file/content inputs as today for work scope).
2. Wire `creator kb remove --scope world` to persistent delete/soft-delete per `KbStore` semantics.
3. Remove `world_scope_write_deferred()` from production paths.
4. Update clap help on `CreatorCommand::Kb` — world scope **read and write** shipped (full help fix may land in hygiene plan; minimum: no false “coming soon” on write).

---

## 3. Non-goals

- `NarrativeGateway` write methods or daemon POST `/v1/local/narrative/*`.
- Fork branch table, local `world fork`, or `creator clone`.
- Manuscript / story manifest product APIs.
- Cloud sync of narrative/KB payloads.
- DB-level pagination for KbQuery (R9 remains deferred).
