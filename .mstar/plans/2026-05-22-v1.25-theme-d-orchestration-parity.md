# V1.25 Theme D Orchestration Parity Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove the legacy stubbed `daemon orchestrate *` CLI surface and document that `daemon schedule` owns orchestration session control.

**Architecture:** The CLI keeps a single shipped orchestration control path: `nexus42 daemon schedule ...`. Runtime HTTP session routes remain registered in `nexus-daemon-runtime`; they are reached through schedule state and supervisor cascades rather than a direct CLI command group.

**Tech Stack:** Rust (`clap`, `assert_cmd`), Nexus daemon Local API, Morning Star harness docs.
