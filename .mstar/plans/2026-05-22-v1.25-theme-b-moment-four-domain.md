## 1. Problem Statement

V1.25 KCA-25-003 found that the product-facing context assembly path in `nexus42` still assembles Stage-0 / TwoStage creator-memory context only, while the library-level `assemble_moment` function already composes the four-domain snapshot from narrative, World KB, user knowledge, and Stage-0 context.

This plan adds a deliberately **demo/experimental** CLI entry for `assemble_moment` using existing in-memory store implementations only. It does not claim production persistence or daemon-backed domain wiring.

---

## 2. Goals

1. Add a CLI-accessible `platform context assemble-moment` path that calls `assemble_moment` with `InMemoryNarrativeGateway`, `InMemoryKbStore`, and `InMemoryKnowledgeStore`.
2. Keep the command explicitly scoped as a demo/experimental path until SQLite/file-backed stores exist.
3. Update the CLI spec to distinguish shipped Stage-0 / TwoStage assembly from Moment four-domain assembly.
4. Add a CLI-level integration test proving the four-domain snapshot path is reachable from `nexus42`.

---

## 3. Non-goals / Out of Scope

- No SQLite-backed narrative, KB, or knowledge stores.
- No production persistence for Moment assembly inputs or outputs.
- No daemon HTTP surface for four-domain Moment assembly.
- No cloud sync integration.
- No `nexus-platform` private-repo work.
- No retirement or rename of existing Stage-0 / TwoStage commands beyond docs/help wording needed to avoid ambiguity.
