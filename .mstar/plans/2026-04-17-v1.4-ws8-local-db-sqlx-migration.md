# V1.4 WS8 — `nexus-local-db` sqlx migration (unified DB engine)

> **For agentic workers:** Ordered checklist of bite-sized tasks. Commit after each task unless noted. This plan touches ~34 files / ~193 references and lives on a dedicated branch; work sequentially.

**Plan ID**: `2026-04-17-v1.4-ws8-local-db-sqlx-migration`
**Wave**: V1.4 WS8 — see [.mstar/iterations/v1.4-delivery-compass-v1.md](.mstar/iterations/v1.4-delivery-compass-v1.md) §4 WS8
**Wave-0 spec (authoritative)**: [knowledge/crate-selection-best-practices.md](knowledge/crate-selection-best-practices.md) §2.3, §3.2, §3.3 (decision SSOT) + [archived/knowledge/local-db-refactor.md](archived/knowledge/local-db-refactor.md) (API shape + table classification; revised to v2 at WS8 T9)
**Working branch**: `feature/v1.4-ws8-local-db-sqlx` from `feature/v1.4`.
**Owner candidate**: `fullstack-dev` primary; regression surface is large — QC triple-review mandatory at T10.

**Goal**: Migrate **the entire Rust workspace** from `rusqlite` + `deadpool-sqlite` + bespoke sequential migrations to `**sqlx` (sqlite + runtime-tokio + macros + migrate)**. Every callsite that currently imports `rusqlite` or `deadpool_sqlite` becomes an async `sqlx` caller. After WS8, the workspace has **one DB engine, one pool implementation, one migration runner** — unblocking WS2 Task 3 (`SqliteSessionStorage`) to share the same pool on the same `state.db` file.

**Architecture**:

- `nexus-local-db` exposes `Arc<sqlx::SqlitePool>` + typed helpers, replacing today's `rusqlite::Connection` + `deadpool_sqlite::Pool` surface.
- Migrations live under `crates/nexus-local-db/migrations/` as timestamped `.sql` files; loaded via `sqlx::migrate!()`. The four existing code-form migrations (`v2_local_identities`, `v3_soul_meta`, `v4_memory_pipeline`) are ported verbatim to `.sql`.
- `workspace_meta.db_schema_version` key is **preserved** (external readers like `nexus42 doctor` still consume it). The key is written by an explicit migration step that reads from sqlx's `_sqlx_migrations` table, keeping both in sync.
- All CRUD modules (`identity`, `soul_meta`, `pending_review`, `memory_fragment`) become `async` using `sqlx::query!` / `query_as!` macros.
- Downstream crates (`nexus42`, `nexus42d`, `nexus-sync`) get async traits through their existing `#[tokio::main]` / Tokio runtime entry points; CLI callsites become `.await`-chained.
- `rusqlite` and `deadpool-sqlite` are removed from `[workspace.dependencies]` at T9.

**Tech Stack**: `sqlx = "0.8"` with features `["runtime-tokio", "sqlite", "macros", "migrate", "chrono", "uuid"]` in `[workspace.dependencies]` per [knowledge/crate-selection-best-practices.md](knowledge/crate-selection-best-practices.md) §3.2.

```rust
pub async fn open_pool(db_path: &std::path::Path) -> Result<sqlx::SqlitePool, LocalDbError> {
    let url = format!("sqlite://{}?mode=rwc", db_path.display());
    let pool = sqlx::sqlite::SqlitePoolOptions::new()
        .max_connections(8)
        .connect(&url)
        .await
        .map_err(LocalDbError::from)?;
    sqlx::query("PRAGMA journal_mode = WAL").execute(&pool).await?;
    sqlx::query("PRAGMA foreign_keys = ON").execute(&pool).await?;
    Ok(pool)
}

pub async fn run_migrations(pool: &sqlx::SqlitePool) -> Result<(), LocalDbError> {
    sqlx::migrate!("./migrations").run(pool).await.map_err(LocalDbError::from)?;
    Ok(())
}
```
