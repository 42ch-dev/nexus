## 1. Problem Statement

DF-62 (multi-volume PK + seed) shipped V1.42 but volume advancement is manual. Authors want auto-advance when current volume completes.

---

## 2. Goals

1. `works.auto_chronology BOOLEAN` column (default `false`).
2. CLI `creator works chronology set/show/advance`.
3. Daemon `auto_chronology_tick` task (5-min interval; env override).
4. Finish detection (all chapters finalized + intake complete).
5. Atomic outline creation + chapter seed + log entry.
6. Manual override CLI `creator works chronology advance --volume N+1`.

---

## 3. Non-goals

- Multi-volume auto-publication (V1.51+).
- World KB auto-promotion from new volume (T-B P1 separate concern).

---

## 4. Acceptance Criteria

1. Migration adds column with default `false`.
2. Daemon tick hermetic test: positive (all finalized + intake complete → advance), 4 negative edge cases (incomplete chapter / intake not complete / runtime lock / already-advanced).
3. Atomic outline creation + chapter seed; crash mid-advance rolls back cleanly.
4. Manual `creator works chronology advance --volume 3` works regardless of `auto_chronology` flag.
5. Log entry written to `Logs/chronology/<YYYY-MM-DD>-advance-vol<N+1>.md`.
