## 1. Problem Statement

`novel-project-init` grill-me offers "create new World" but `creator world create` is not implemented. `world_id` PATCH and `world_refs` validation are spec-only. World-bound users hit silent failures or advisory-only behavior.

---

## 2. Goals

1. Implement `nexus42 creator world create --name --kind narrative [--description]` returning `world_id`.
2. Wire init grill-me "create new" path to world create (atomic with scaffold transaction).
3. Validate `world_id` on Work create/PATCH; structured `preset_gates_failed` remediation.
4. Implement `world_refs` validation: warnings at outline, errors at finalize (world-bound Works).
5. Preserve worldless Work behavior unchanged.

---

## 3. Non-goals

- Full taxonomy schema (P1), prompt context block (P2), kb-extract binding (P3).
- Multi-volume PK, platform publish, multi-work switch.
- Tightening `world_binding` to `required`.

---

## 4. Acceptance Criteria

1. `creator world create` returns `wld_<uuid>` and persists World row visible to active creator/workspace.
2. `creator world list` and `creator world show <world_id>` work for created Worlds.
3. Init "create new World" binds `world_id` or fails closed (no partial scaffold).
4. PATCH with invalid `world_id` returns remediation pointing to `creator world create` or `list`.
5. World-bound outline with invalid `world_refs` emits warnings; finalize blocks with error unless `--force`.
6. Worldless Works: no `world_id` gate failure; `world_refs` warn-only if present.
7. Hermetic tests cover valid/invalid `world_id` and `world_refs` paths.
