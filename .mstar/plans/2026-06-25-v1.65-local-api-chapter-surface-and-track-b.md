## Overview

Build the Local API chapter-content surface (the data layer the P2 outline+structure editor consumes) and close 3 of the 4 Track B V1.64 carry-forwards (`work_profile`, preset full CRUD, `items`+cursor on new endpoints). The 4th (apps/web test baseline) is frontend, closed in P1.

Chapter content is **file-backed** (`outline_path`/`body_path` markdown); the DB `work_chapters` table holds metadata. New endpoints read/write outline + read body read-only + manage structure, following V1.64 cursor + `ErrorResponse` conventions (`local-api-surface-conventions.md`).
