## Scope (β depth — `wire_contracts_changed: TRUE`)

> Architect lock note: P0 mirrors the compass technical locks so implementers do not reopen storage, route, validation, token, or contract-version decisions.

- **A1 — Schemas + codegen**: New Local API DTOs in `schemas/local-api/canvas/outline/`:
  - `OutlinePatchStructureRequest` / `OutlinePatchChapterRequest` / `TimelinePatchEventRequest`
  - `OutlinePatchResponse`
  - `OutlineConflictError` (extends existing error envelope; mirrors V1.71 `StrategyConflictError`)
  - `OutlineValidationError` (mirrors V1.71 `StrategyValidationError`)
  - Generate TS (`apps/web/src/api-types/canvas/outline/`) and Rust (`crates/nexus-contracts/src/generated/canvas/outline/`); commit regenerated output.
- **A2 — outlineRevision tracking + persistence (LOCKED)**: store outline revision as `outline_revision:` key in outline markdown frontmatter (V1.65 atomic-save file pattern). Existing outline without `outline_revision:` reads as `0`; first successful patch writes `1`. On every accepted patch, increment revision exactly once in the same committed outline markdown image; read endpoints expose the current revision; reject stale `base_revision` with `OutlineConflictError`. Do **not** add a V1.72 sidecar `.outline_revision` file or `outline_revisions` DB table; a future DB migration must backfill from frontmatter if audit history / multi-device sync / collaborative edits require it.
- **A3 — Local API routes (LOCKED)**: New `POST` routes following V1.65 + V1.71 patch-route conventions:
  - `POST /v1/local/works/{work_id}/outline/patch` with `OutlinePatchStructureRequest` → `OutlinePatchResponse` (operations: `move_chapter`, `link_event`, `attach_to_volume`)
  - `POST /v1/local/works/{work_id}/chapters/{chapter_id}/patch` with `OutlinePatchChapterRequest` → `OutlinePatchResponse` (chapter structure fields: title, slug, planned/actual word counts where supported, volume binding, status using existing V1.65 wire enum `not_started → outlined → draft → finalized`; `published` remains read/protected in V1.72)
  - `POST /v1/local/works/{work_id}/timeline/patch` with `TimelinePatchEventRequest` → `OutlinePatchResponse` (operations: `add_event`, `remove_event`, `attach_event_to_chapter`, `link_foreshadow`)
  - Path identifiers are authoritative; duplicated body identifiers must match or return the canonical 400/path-mismatch error envelope.
  - Error envelope: 409 `OutlineConflictError`, 422 `OutlineValidationError`, standard 400/404/500 envelope codes.
- **A4 — Daemon validation + atomic persistence** (`crates/nexus-daemon-runtime/src/handlers/outline.rs`):
  - Validate **ID existence**: work_id, chapter_id, volume_id, event_id, foreshadow_id, timeline_event_id; path ids match body ids when duplicated.
  - Validate **structural integrity**: no orphan chapters or multiple parents; valid Work/Volume containment (`belongs_to_volume` invariants); `ordered_after` edges remain acyclic; `foreshadows` edges reference existing event/chapter ids.
  - Validate **status transition**: chapter status moves only along the existing V1.65 lifecycle (`not_started → outlined → draft → finalized`); reverse transitions, forward skips, and `published` edits are rejected unless a future explicit action/reason contract authorizes them.
  - Validate **timeline reference resolution**: timeline events reference existing chapter ids; `realizes_event` references existing event ids; `foreshadows` targets events at or after the source event.
  - Validate **outline revision precondition**: stale `base_revision` short-circuits as 409 before validation/persistence side effects.
  - Atomic outline markdown write (temp + rename + fsync, V1.65 durability pattern) with the incremented `outline_revision:`; failed validation/conflict must not increment revision.
- **A5 — NexusClient methods**: `outlinePatchStructure`, `outlinePatchChapter`, `timelinePatchEvent`; track `lastKnownRevision`; refetch canonical outline after patch.
- **A6 — Canvas UI write affordances**:
  - **Outline graph projection**: Work → Volume → Chapter → Scene/Beat as nested graph nodes; `belongs_to_volume` and `ordered_after` edges; volume lane is a sub-flow with `parentId`+`extent:parent` children; chapter cards show `wc`/`slug`/`status` from outline; TipTap fragment preview is read-only on canvas.
  - **Timeline lane**: separate React Flow lane showing events across chapters; events positioned by chapter realization point; `foreshadow` edges link events resolving later; `realizes_event` edges link chapter nodes to events.
  - **Chapter inspector**: edit title/slug/wc/volume/status; submit `outline.patch_chapter`; show validation errors inline; show conflict modal on 409.
  - **Event inspector**: edit event metadata (title/description/realizes_chapter); submit `timeline.patch_event`; same conflict UX.
  - **Outline structure inspector**: drag-chapter-to-volume gesture triggers `outline.patch_structure` with `move_chapter`; link-event gesture triggers `link_event`; same conflict UX.
  - **Conflict modal UX / copy** (reuse V1.71 modal component with outline copy):
    - On 409 `OutlineConflictError`, keep the draft patch in memory, refetch canonical outline, and show a modal.
    - **Headline**: "This outline changed while you were editing."
    - **Body**: "Nexus updated **{node_label}** to revision **{current_revision}** while you were editing **{field}**. Your change is still in the inspector." (PM: `{node_label}` adapts to domain entity per compass A6 note.)
    - Show **What changed** (canonical value) and **What you were about to do** (draft value).
    - **Actions**: **Use current** (primary/default), **Reapply my edit**, **Review side-by-side** (enabled only when draft and canonical changes touch non-overlapping fields; disabled for same-field/path or timeline-event content).
    - **Cancel** returns focus to the originating inspector.
  - **outlineRevision freshness indicator**: "Outline · revision {revision} · updated {relative time} — refresh now" (reuse V1.71 Strategy revision copy pattern; explicit refresh action added).
  - **Non-spatial alternate view** (Draft §4.4 accessibility): sortable chapter list + sortable timeline event list, toggle from canvas toolbar. List views are default for keyboard-only / screen-reader users.
  - **Acceptance (A6)**: conflict modal copy is clear and actionable; default action is **Use current**; **Review side-by-side** is enabled only when safe; draft not lost on cancel; non-spatial alternate views are reachable and equivalent to canvas (chapter list columns to verify: title / status / wc / volume / updated; timeline event list columns: event / realizes_chapter / foreshadows / updated).
- **A7 — Tauri WKWebView parity smoke**: outline drag-to-volume, timeline event pin, inspector modal, conflict modal work in Tauri shell.
- **A8 — Accessibility**: conflict modal uses ARIA live region to announce conflict + current-vs-draft difference; focus moves into modal and is trapped; closing returns focus to originating node/inspector; keyboard paths for **Use current / Reapply my edit / Review side-by-side / Cancel**; `prefers-reduced-motion` honored for modal + timeline lane animation; non-spatial alternate views reachable.
- **A9 — Tests**: unit (validation + revision + conflict + structural integrity), integration (3 routes), e2e (canvas edit → save → refetch), regression (concurrent daemon write causes 409 — proves outlineRevision catches stale writes from BOTH canvas and daemon/orchestration-side writers, mirroring V1.71 `R-V152TA-S006` promise gap closure). The concurrent-writer regression must cover canvas read at `N`, daemon/orchestration write to `N+1`, stale canvas patch rejected as 409 before validation/persistence side effects, and UI draft-preserving recovery after canonical refetch.

## Acceptance

- All 3 outline/timeline patch routes return correct `OutlinePatchResponse` and increment revision atomically
- outlineRevision storage uses outline markdown frontmatter `outline_revision:`; existing missing revisions migrate lazily from `0` to `1` on first successful patch; no separate V1.72 DB table is introduced
- Stale `base_revision` returns `OutlineConflictError` (409) with current revision, target id, and recovery hint
- Validation rejects missing ids, broken structural integrity, invalid status transitions using the V1.65 status enum, and unresolved timeline references before persistence
- Canvas UI can edit chapter fields, event metadata, and outline structure; conflict modal offers **Use current**, **Reapply my edit**, **Review side-by-side**; conflict modal copy is clear and actionable; default action and disabled side-by-side state implemented
- Non-spatial alternate views reachable + equivalent; outlineRevision freshness indicator copy clear and actionable

- Tests cover validation, conflict detection, concurrent daemon write, and e2e canvas edit→save→refetch
