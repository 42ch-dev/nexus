## 1. Problem Statement

`novel-writing/author-experience.md` §3.4 and quality-loop specs describe `reflection-loop` as producing findings, but the shipped preset does not bind work/chapter or call `from-review`. V1.39 plan T3 was never completed.

---

## 2. Goals

1. Transform FL-E `review` preset (rename allowed) to novel/work/chapter-aware prompts and inputs.
2. On successful review terminal (auto-chain **and** on-demand), persist one or more findings via existing `from-review` path.
3. Populate optional `rule_suggestion` metadata on findings (no `AGENTS.md` file write).
4. Hermetic E2E: supervisor or API-level test proving finding row creation.
5. Update FL-E preset id references in specs/README if preset renamed.

---

## 3. Non-goals

- New wire schemas in `nexus-contracts`.
- Writing `Works/<work_ref>/AGENTS.md` or `novel-rules.md`.
- Findings → draft prompt enrichment (§5.5.2).
- `read_rules_layers` path migration to `AGENTS.md`.
- Keeping a parallel generic topic demo preset.

---

## 4. Acceptance Criteria

1. Auto-chain review stage creates ≥1 finding row for a novel Work with chapter context.
2. `creator run <review_preset_id> <work_id>` creates findings on the same code path.
3. Findings include `kind`, `severity`, `target_executor`, optional `rule_suggestion` in stored metadata/body contract per spec §8.
4. Auto-chain driver invariant preserved (no fork/cancel of active FL-E driver).
5. ≥1 hermetic integration test for review → finding insert.
