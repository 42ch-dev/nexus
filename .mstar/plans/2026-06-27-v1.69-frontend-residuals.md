## Goal

Close the 4 V1.67-deferred frontend refactor residuals on the `apps/web/src/` surface. All are pure TS refactors — no wire touch, no new feature. Closes the slate of V1.67 frontend QC suggestions ahead of the V1.70 canvas implement (which will consume the SSOT modules and query-key structure produced here).

## Scope (in)

- [ ] **C1 — `R-V167P1-QC1-S1`**: narrow `useState<string>` to a `work_profile` literal union in the Create-Work dialog. The wire already carries `work_profile: Option<String>`; the runtime-admitted values are a closed set (`novel` / `essay` / `game_bible` / `script`). Narrow the state type + the selector to the literal union; reject invalid values at the type boundary.
- [ ] **C2 — `R-V167P1-QC1-S2`**: extract `WORK_PROFILES` to a SSOT module (e.g. `apps/web/src/lib/work-profiles.ts` exporting the literal-union type + the values array + the labels). Consumed by the Create-Work dialog (C1) + future V1.70 canvas Strategy surface. Keeps a single source for the profile list.
- [ ] **C3 — `R-V167P1-QC3-S1`**: `adapter-contract.test.ts` 21→24 method coverage (matches the V1.67 G2 TS-client promotion: `getPreset`/`updatePreset`/`deletePreset`) + a preset-method parity guard so the contract test fails if the `NexusClient` interface and the adapter drift on preset methods.
- [ ] **C4 — `R-V167P1-QC3-S2`**: preset query keys + invalidation structure prepared for the V1.70 canvas UI. Define the query-key factory (`presetKeys.all`, `presetKeys.detail(id)`, etc.) + the invalidation helpers; the actual `invalidateQueries` wiring in canvas components lands in V1.70 when the canvas UI consumes them. V1.69 ships the structure so V1.70 doesn't redesign the key shape mid-implement.
- [ ] **Optional cherry-pick**: `R-V165-QC-SUGG-DX` items that are pure-token or copy fixes and non-conflicting with P0's DESIGN.md work. Skip any item that touches token values (P0 owns those). PM discretion per item; document any pick/skip in the completion report.
