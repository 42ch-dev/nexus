## 1. Problem Statement

`works.schedule_json` is now writable (T-A P0), but the daemon does not yet fire cron-triggered schedules for `novel-brainstorm` / `novel-write`. Authors can configure schedule but nothing happens.

---

## 2. Goals

1. Daemon tick evaluates per-Work cron; fires `novel-brainstorm` and `novel-write` schedules per role enabled + cron match.
2. Per-Work gating: skip when intake != complete / runtime_lock != NULL / completion_locked.
3. Idempotency: skip when prior same-role same-work schedule still active.
4. Cron fire enqueues a `Schedule` with `preset_id = <role-preset>` and `work_ref` from source Work.

---

## 3. Non-goals

- Review cron (T-A P2).
- Auto-chronology (T-A P3).

---

## 4. Acceptance Criteria

1. `cargo test -p nexus-orchestration --test cron_supervisor` covers: fire on cron match, skip on intake/lock/completion, idempotent re-fire skip.
2. `creator run status` after cron fire shows the auto-chain step advanced.
3. Test: synthetic tick advances `now` to a cron match minute; expect enqueue; advance to non-match → no enqueue.
4. Hermetic test: two cron fires within same minute for same role+work → second skipped.
