## 1. Problem Statement

Current `novel-writing` preset produces demo-grade `outline.md` + `ch01-introduction.md` + `ch02-body.md` under legacy paths. It does not implement per-chapter outline, draftвҶ’final status transitions, or `work-status.md` chapter table updates aligned with reference-system quality bars (distilled, not copied).

---

## 2. Goals

1. Rewrite `novel-writing` prompts to use `Works/{{work_ref}}/Outlines/chapters/ch<nn>-outline.md` and `Works/{{work_ref}}/Stories/ch<nn>-<slug>.md`.
2. Pipeline for **one chapter** (ch01 MVP): outline state вҶ’ draft state вҶ’ finalize state.
3. Update `work-status.md` chapter row on each transition (`outlined` вҶ’ `draft` вҶ’ `finalized`).
4. Integrate with FL-E `produce` stage (`creator run stage advance --stage produce` or chain default).
5. Optional light `reflection-loop` hook documented; not required for V1.36 MVP acceptance.

---

## 3. Non-goals

- Multi-chapter batch automation in one schedule tick
- Long-form serial cron (brainstorm/review loops)
- Platform publish

## 5. Acceptance

1. After produce stage on demo Work, `Works/<work_ref>/Stories/ch01-*.md` has `status: finalized` **and** the `finalize` state's `judge.llm` returned GO (T7).
2. `Outlines/chapters/ch01-outline.md` exists and is non-empty, with at least one F### entry if any foreshadowing is touched (T9).
3. `work-status.md` shows ch01 as `finalized`, `е®һйҷ…еӯ—ж•°` = chapter frontmatter `word_count`, `е®ҢжҲҗеәҰ` = 100%.
4. No files written under workspace-root `Stories/`.
5. `cargo test -p nexus-orchestration` preset regression passes.
6. **Negative path**: a known-bad-quality draft triggers NOGO from `judge.llm`; chapter frontmatter `status` stays `draft`; `work-status.md` row does not flip to `finalized`. `creator run continue --note "..."` followed by re-run flips to `finalized` on next GO.
7. **Override path**: `creator run stage advance --force` on a NOGO chapter flips to `finalized`; audit log records `forced_finalize_on_nogo: <reason>`.
