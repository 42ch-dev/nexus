### Root-cause verification (C1)

The canonical backend-accepted set was **verified by direct grep, not inferred from the QC report**. Three independent authoritative sources agree:

- **DB CHECK constraint (latest cumulative, V1.60 P1)** — `crates/nexus-local-db/migrations/202606230001_work_profile_script.sql:27`:
  `CHECK (work_profile IS NULL OR work_profile IN ('novel', 'essay', 'game_bible', 'script'))`
- **Rust helpers** — `crates/nexus-local-db/src/works.rs:28-60`: `is_novel_profile==Some("novel")`, `is_game_bible_profile==Some("game_bible")`, `is_script_profile==Some("script")`, `is_essay_profile==Some("essay")`. Mutual-exclusion unit test at `works.rs:2320-2340` confirms the 4-value set.
- **Daemon handlers** — `crates/nexus-daemon-runtime/src/api/handlers/works.rs:576,623,678,733` all use the underscore form `Some("game_bible")`. The create handler at `works.rs:387` stores `req.work_profile` **verbatim** (no normalization).

**Canonical backend-accepted set** = `["novel", "essay", "game_bible", "script"]` (NULL also accepted). `game_bible` uses **underscore**.

Note: the CLI bootstrap (`crates/nexus42/src/commands/creator/bootstrap.rs:140-143`) accepts **both** `game-bible` and `game_bible` and normalizes — but that path is **not** used by the Web UI, which goes through the daemon HTTP API (no normalization). The UI must therefore emit the canonical form directly.

### Fixes applied

**C1** (`apps/web/src/pages/dialogs/create-work-dialog.tsx`): changed `WORK_PROFILES` Game Bible option value `'game-bible'` → `'game_bible'`. The other three values (`novel`, `essay`, `script`) already matched the canonical set.

**W1** (same file): added a `workProfileTouched` state flag (default `false`, set `true` on selector `onChange`, reset on dialog open). The submit payload now omits `work_profile` when the form is untouched, preserving V1.66 wire shape (daemon stores NULL). The selector still displays `novel` as the default choice — it just isn't sent unless the author explicitly selects. Updated the component doc comment to document the new semantics.

**Export**: `WORK_PROFILES` is now `export const` so the wire-contract test can assert canonical-set membership directly. Extraction to a SSOT module remains tracked as `R-V167P1-QC1-S2` (deferred).
