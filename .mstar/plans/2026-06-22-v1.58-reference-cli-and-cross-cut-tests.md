## Scope

- **T1**: Implement `nexus42 reference refresh` CLI subcommand. Add `ReferenceCommand::Refresh` variant to the CLI command enum. Subcommand syntax: `nexus42 reference refresh [work_ref|all] [--policy <on_change|scheduled>] [--dry-run]`. `work_ref` scopes refresh to references belonging to a specific Work; `all` refreshes every non-offline reference source; `--dry-run` prints what would be refreshed without mutating.
- **T2**: Wire CLI subcommand to daemon IPC. `nexus42 reference refresh` → `DaemonClient::post` → daemon → `capability::Registry::dispatch("nexus.reference.refresh", input)`. The CLI entry normalizes its input into the internal registry shape.
- **T3**: Write cross-reference E2E test. Create an integration test that: (a) registers a reference source with `refresh_policy = on_change`, (b) invokes `nexus42 reference refresh <work_ref>`, (c) asserts the reference body is updated, (d) asserts `last_refreshed_at` is set, (e) asserts `refresh_status = fresh`. Exercise the `--dry-run` path (no mutation) and the `all` path (multiple sources).
- **T4**: Add `cli-spec.md` §6.2N — document the `reference refresh` subcommand: syntax, flags, behavior, exit codes, daemon IPC wiring. Follow the existing §6.2M (`host-call`) format.
- **T5**: Amend `daemon-runtime.md` — add refresh-scheduler hook section documenting the periodic task: interval configurability, query logic, dispatch path, error handling, tracing contract.
- **T6**: Amend `local-runtime-boundary.md` — update topology diagram/notes to include the `nexus42 reference refresh` CLI → daemon IPC → `capability::Registry::dispatch` caller path.
- **T7**: Run schema codegen if P1 added capability enum variants. `pnpm run codegen` and commit regenerated output under `crates/nexus-contracts/src/generated/`.
- **T8**: Add `--reference-policy` flag documentation. If the CLI needs to set/update refresh policy per reference source, add `nexus42 reference policy set <source_id> --policy <on_change|scheduled|offline>` subcommand. (Scope: only if needed for P3 E2E test; otherwise deferred to future iteration.)
- **T9**: Verify. `cargo test -p nexus42 --test reference_refresh_cli` (CLI integration tests) + `cargo test -p nexus-orchestration --test cross_reference_refresh_e2e` (E2E test) + `cargo clippy --all -- -D warnings` + `cargo +nightly fmt --all -- --check`.

## Risks

- **CLI subcommand name collision**: `nexus42 reference refresh` may conflict with existing `reference` noun patterns. Mitigation: `reference` is a new CLI noun group (V1.58); verify no existing `ReferenceCommand` variant in the CLI enum before adding.
- **Daemon IPC wiring may expose timeout/latency issues**: The refresh path (CLI → daemon IPC → registry dispatch → network fetch) may take longer than the default daemon IPC timeout. Mitigation: set a reasonable IPC timeout (30s) for the refresh path; document in cli-spec.md.
