## 1. Problem Statement

Users can pass a one-line `--seed` to `novel-writing`, but there is no **grill-me** phase that forces structured clarification before production. Gathering-state `llm_judge` may not evaluate quality until P3 lands.

---

## 2. Goals

### A1 — `creative-brief-intake` embedded preset

1. New bundle under `embedded-presets/creative-brief-intake/`.
2. `run_intents: [work_init]` only.
3. ACP multi-turn prompts to elicit genre, tone, audience, constraints, themes, non-goals.
4. Terminal state writes structured `creative_brief` JSON to Work (via Local API from capability hook or schedule callback — architect picks one).

### A2 — Brief validation

1. Validator for §4 schema (required fields).
2. Work `intake_status=complete` only when valid.

### A3 — `novel-writing` wiring

1. Seed / `preset.input` populated from `creative_brief` + `initial_idea`.
2. `creator run start` chains intake → novel-writing (auto or explicit flag documented).
3. Document interim path in embedded-presets README.

### A4 — `memory-augmented` persist fix (if in scope)

1. `persist` state writes generation output fragment, not only `preset.input.topic`.
