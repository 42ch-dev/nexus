### Scope Grouping for This Plan

**In scope (Batch 1)** — Grouped by thematic affinity for surgical resolution:

**Group A: Test Isolation** (R-CREATOR-001, R-DAEMON-ACP-001, R-V113-007)
- All three share the same root cause: `HOME` environment variable manipulation causes race conditions under parallel test execution on macOS (`dirs::home_dir()` does not respect `$HOME` on all platforms).
- Fix: Introduce a `testutil` module that provides reliable home directory isolation using `tempfile` + explicit env guard.

**Group B: Data Integrity Hardening** (R-KB-001, R-KB-002, R-CREATOR-002, R-CREATOR-003)
- Index corruption resilience, ID collision prevention, file locking, and input validation.
- Fix: Add structured validation, advisory file locking, and collision detection.

**Group C: Trace I/O Hardening** (R-TRACE-001, R-TRACE-002, R-TRACE-003, R-TRACE-004)
- All trace residuals are in the same file/module and share the same architectural fix: replace per-event open/close with a buffered writer, add file locking, replace `eprintln!` with `tracing`, and add size limits.
- Fix: Refactor trace writer to use a singleton `BufWriter` with advisory lock and size guard.

**Out of scope** (deferred to future plans):
- R-V113-005: UpstreamTimeout e2e test variance (low severity, accept — OS/proxy dependent, not fixable in code).
- DEBT-RAND-073: rand 0.7.3 upgrade blocked by wiremock upstream (cross-cutting, accept).
