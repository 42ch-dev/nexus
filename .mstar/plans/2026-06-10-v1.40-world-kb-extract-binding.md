## 1. Problem Statement

FL-E `persist` already maps to `kb-extract` (`preset_for_stage`), and `build_preset_input` passes `work_id` + chapter paths for produce — but persist path lacks `world_id`, jobs still use `work_entry_id`, and capability does not read `preset.input.body_path` for chapter ingest.

---

## 2. Goals

1. Extend `kb_extract_jobs` with artifact locator fields (`source_kind`, `source_locator`, `profile_hint`); V1.40 ships `work_chapter` + `novel` only.
2. Extend `WorkFields` + `build_preset_input` with `world_id` from Work row.
3. Extend **`kb.extract_work`** to accept `body_path` + `world_id` + `work_id` (auto-enqueue or inline job); delegate parse/validate/insert to `nexus-kb` helpers.
4. Auto-chain / persist: World-bound Works pass chapter locator into kb-extract schedule input.
5. Worldless Works: skip World promotion.
6. **Tail (grill-me #14):** `novel-review-master` `sync_world_kb` state + **`schedule.enqueue_child`** capability → child `kb-extract` with `depends_on`; `refresh_kb` gate (default true when World-bound).

---

## 3. Non-goals

- Renaming `kb.extract_work` → `kb.ingest_artifact`.
- Full multi-profile extract prompts (screenplay/essay) — schema reserved.
- Merging review-master and kb-extract presets.

---

## 4. Acceptance Criteria

1. World-bound Work after chapter finalize: extract uses `body_path` + `work.world_id` artifact locator.
2. Hermetic test: extract → KeyBlock + SourceAnchor; wire `BlockType` + `novel_category` per P1.
3. Worldless Work: no World KeyBlocks from persist/extract.
4. `creator kb queue-extract` extended for artifact locator (`--chapter N` sugar for novel).
5. Review-master with `refresh_kb=true` waits for child kb-extract (`schedule.enqueue_child`).
6. DF-63 closable after P3 ships.
