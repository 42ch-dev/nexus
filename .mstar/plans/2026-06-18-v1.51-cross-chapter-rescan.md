## 1. Problem Statement

V1.50 T-B P2 shipped `creator kb rescan <chapter>` as **chapter-scoped** only. An entity referenced across multiple chapters (e.g. a recurring character) generates N pending candidates — one per chapter — that authors must adopt independently. R-V150KBED-08 was deferred to V1.51 for cross-chapter scope.

T-A P1 extends rescan to `creator kb rescan --work <ref>` which reconciles entities across all chapters in the work, surfacing cross-chapter reuse and unifying pending candidates.

---

## 2. Goals

1. New CLI flag `--work <work_ref>` for `creator kb rescan` (mutually exclusive with positional `<chapter>`).
2. Work-scoped rescan iterates all chapters in `Works/<work_ref>/Stories/` and aggregates extraction candidates by canonical_name (LLM-extracted from T-A P0).
3. Cross-chapter reuse surfaced in `--dry-run` output (e.g. "Entity 'Aelin' referenced in chapters 3, 5, 7; existing KB row found → no new candidate").
4. Non-dry DB upsert reconciles: if a candidate matches an existing KB row (canonical_name + world_id), update source_quote to `ch3+ch5+ch7`; if no match, write new pending candidate with cross-chapter source_quote list.
5. Closes R-V150KBED-08.

---
