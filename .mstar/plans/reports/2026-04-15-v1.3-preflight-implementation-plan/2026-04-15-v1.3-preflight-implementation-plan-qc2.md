## T1: Review Classification Quality Signal

### Critical — Memory Leak in `quality_signal()`
- **File:** `crates/nexus-domain/src/review_quality.rs`  
- **Line:** 43  
- **Code:**
  ```rust
  let entry = seen.entry(Box::leak(lower.into_boxed_str())).or_insert(0);
  ```
- **Issue:** `Box::leak` converts every lowercased token into a `'static str`, permanently leaking that allocation. Because `quality_signal` is a pure helper intended to be called frequently during review classification, this introduces unbounded memory growth with no recovery path.
- **Impact:** High — every call to `classify_pending_review` on creative tasks now leaks `O(token_count)` bytes.
- **Fix:** Change `seen: HashMap<&str, usize>` to `HashMap<String, usize>` and use `seen.entry(lower).or_insert(0)`.

### Warning — Magic-Number Thresholds
- **File:** `crates/nexus-domain/src/review_quality.rs`  
- **Lines:** 78  
- **Issue:** Thresholds `0.35`, `0.65`, `0.45` are inline literals without named constants. This makes tuning harder and increases risk of drift if the values are ever copied elsewhere.
- **Fix:** Extract to `const QUALITY_UNIQUE_RATIO_THRESHOLD: f32 = 0.35;` etc.

### Positive
- Tests cover empty input, repetitive noise, rich text, symbol-heavy text, and mixed content with stop words.
- Integration into `classify_pending_review` correctly gates `PromoteToLongTerm` behind both length (`>= 80`) and signal quality.

## T2: Memory CLI — HTTP Error Handling, Serde Correctness, Graceful Degradation

### Warning — `list_memory_fragments` Missing `creator_id` Propagation
- **File:** `crates/nexus42/src/api/daemon_client.rs`  
- **Lines:** 265–288  
- **Issue:** The CLI `fragments` command enforces `active_creator_id` (in `memory.rs` line 57), but `list_memory_fragments` calls `GET /v1/local/memory/fragments` with **no query parameter** for the creator. If fragments are scoped per-creator (consistent with the rest of the memory domain), the daemon cannot filter correctly.
- **Impact:** Medium — API contract gap; may return global fragments or silently return an unfiltered superset.
- **Fix:** Add `creator_id` as a query parameter to the daemon endpoint and pass it from the CLI.

### Positive
- `review_pending_memories` and `list_memory_fragments` both translate connection errors into a friendly `CliError::DaemonNotReachable`.
- `parse_error_response` attempts structured JSON parsing and falls back to raw body text, preserving backward compatibility.
- `models.rs` has serde round-trip tests for `ReviewResponse`, `FragmentRow`, and `RuntimeStatus` (with default handling).

## T3: Runtime Mode — Schema Validity, Codegen Pipeline, Type Consistency

### Warning — Weak Typing in `RuntimeStatus`
- **File:** `crates/nexus42/src/api/models.rs`  
- **Lines:** 17–19  
- **Code:**
  ```rust
  #[serde(default)]
  pub runtime_mode: String,
  ```
- **Issue:** The field uses `String` instead of the strongly-typed `RuntimeMode` enum that was just added to the generated contracts. This undermines the contract alignment goal and allows invalid runtime modes to deserialize without error at this boundary.
- **Fix:** Change type to `RuntimeMode` and add a custom default (e.g., `RuntimeMode::LocalOnly`) or keep `#[serde(default)]` with a helper.

### Positive
- `runtime-mode.schema.json` is a valid Draft-07 schema with correct `enum` values and `default`.
- Codegen outputs are present and consistent in both Rust (`crates/nexus-contracts/src/generated/runtime_mode.rs`) and TypeScript (`packages/nexus-contracts/src/generated/RuntimeMode.ts`).
- `enum_conversions.rs` successfully migrated the hand-maintained `RuntimeMode` to the generated type, adding `Display`, `as_str`, and `FromStr` without breaking callers.
- `SCHEMA_VERSIONS` and `LATEST_SCHEMA_VERSION` were updated.
