#### S-002: Pagination cursor format varies across endpoints

**Issue**: Cursor formats differ:
- `workspaces.rs`: `"{creator_id}/{workspace_slug}"`
- `agent_host.rs`: session UUID string

While both work correctly, a standardized format (e.g., always UUID or always opaque base64-encoded token) would simplify client cursor handling across the API.

**Source**:
- `crates/nexus-daemon-runtime/src/api/handlers/workspaces.rs:446-449`
- `crates/nexus-daemon-runtime/src/api/handlers/agent_host.rs:307-312`

**Fix**: No change required for V1.20. Document cursor semantics per endpoint in API contract. Consider unification in V1.21 if feedback indicates client friction.

**Confidence**: Medium

---

#### S-003: `is_loopback_request` treats missing address as loopback

**Issue**: The `is_loopback_request()` function returns `true` when `ConnectInfo` is not present (test environments). This is intentional and documented, but could be misinterpreted as a security bypass in audits.

**Source**:
- `crates/nexus-daemon-runtime/src/api/auth_middleware.rs:236-248`

**Fix**: Consider adding a `#[cfg(test)]` guard or a named constant like `IN_PROCESS_LOOPBACK_ASSUMPTION` to make the intent clearer in security audits. No functional change needed.

**Confidence**: Low
