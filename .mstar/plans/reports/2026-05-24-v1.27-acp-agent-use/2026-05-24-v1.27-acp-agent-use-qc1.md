### 🟢 Suggestion

**S-001: Use `toml` crate for config serialization instead of manual string formatting**

- **Location**: `crates/nexus42/src/commands/acp/mod.rs`, `cmd_agent_use` (lines 707–713)
- **Issue**: The TOML config file is written via `format!` with manual escape logic (`trimmed.replace('\\', "\\\\").replace('"', "\\"")`). This approach only handles two characters and may produce malformed TOML if future agent-ref formats introduce other special characters (newlines, tabs, backspace, etc.).
- **Fix**: Serialize via the existing `toml` crate dependency (already in `Cargo.toml`) using a small struct:
  ```rust
  #[derive(Serialize)]
  struct DefaultAgentConfig {
      schema_version: u32,
      default_agent_ref: String,
  }
  ```
  Then write with `toml::to_string_pretty(&config)?`. This eliminates the escape-risk surface entirely and is more maintainable.
- **Priority**: Low — current escape logic covers the common agent-ref character set, and the crate is pre-release.

**S-002: Add integration test coverage for `acp agent list` default-agent banner**

- **Location**: `crates/nexus42/tests/cli_agent.rs`
- **Issue**: `default_agent_ref()` is integrated into the `AgentSubcommand::List` handler (line 217) to print a banner, but no integration test verifies this banner appears when a default agent is configured.
- **Fix**: Add a test that sets a default agent via `acp agent use`, then runs `acp agent list` and asserts stdout contains `Default agent:`.
- **Priority**: Low — the read path is exercised indirectly by existing code, but explicit coverage would prevent regressions.

**S-003: Extract shared creator/workspace resolution logic**

- **Location**: `crates/nexus42/src/commands/acp/mod.rs`, `cmd_agent_use` and `default_agent_ref`
- **Issue**: Both functions duplicate the same ~6 lines for resolving `creator_id` + `workspace_slug` + path validation + `dirs::home_dir()`.
- **Fix**: Extract a private helper such as:
  ```rust
  fn active_workspace_config_path(config: &CliConfig, file_name: &str) -> Option<PathBuf>
  ```
  This reduces duplication and makes future workspace-scoped config commands easier to add.
- **Priority**: Low — only two call sites today.
