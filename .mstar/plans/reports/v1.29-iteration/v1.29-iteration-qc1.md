### 🟡 Warning

**W-001: Creator command module approaching maintainability threshold**
- **Location:** `crates/nexus42/src/commands/creator/mod.rs` (2772+ lines)
- **Issue:** The module mixes workspace lifecycle, creator authentication/registration, KB file index operations, and the new KB extract queue logic. The new `kb_queue_extract` and `kb_extract_status` functions (lines 1415–1534) add ~130 lines to an already overloaded module.
- **Impact:** High cognitive load; changes to one subsystem risk affecting others; review surface area grows quadratically.
- **Fix:** Extract KB operations (list, search, show, add, remove, queue-extract, extract-status) to a dedicated `commands/creator/kb.rs` module. `mod.rs` should delegate, not implement.
- **Evidence:** File is 2772 lines — 2.5× the recommended 1000-line threshold for CLI command modules.

**W-002: `kb.extract_work` capability is a placeholder**
- **Location:** `crates/nexus-orchestration/src/capability/builtins/kb_extract_work.rs`
- **Issue:** The `run()` method returns a placeholder JSON (`"status": "ready_for_acp_prompt"`) instead of performing actual LLM-based extraction. While documented as intentional (design doc §B3), the capability is registered in the production registry and may be called by presets expecting real behavior.
- **Impact:** Silent degradation — callers get a success response with no actual extraction performed.
- **Fix:** Either (a) mark the capability as `#[doc(hidden)]` / unregistered until fully implemented, or (b) return `CapabilityError::NotImplemented` with a clear message indicating the capability is not yet functional.
- **Evidence:** Lines 131–140 return placeholder without error.
