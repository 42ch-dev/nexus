### 🟡 Warning

**W-001: API `shutdown_session` handler shuts down entire host, not one session**
- **Issue**: `DELETE /v1/local/agent-host/sessions/{id}` calls `host.shutdown().await`, which terminates **all** sessions and the entire host subsystem, ignoring the `{id}` path parameter.
- **Source**: `crates/nexus-daemon-runtime/src/api/handlers/agent_host.rs:178-193`
- **Impact**: A client requesting shutdown of a single session inadvertently kills all active sessions. This is a correctness/UX issue that could lead to unexpected data loss.
- **Evidence**:
  ```rust
  pub async fn shutdown_session(...) {
      let host = get_host(&state)?;
      // Wave 1: shutdown the entire host — per-session shutdown
      // requires session-to-provider routing exposed through the facade.
      host.shutdown().await.map_err(|e| map_host_error(&e))?;
      ...
  }
  ```
- **Fix**: Return HTTP 501 Not Implemented for Wave 1, or implement per-session shutdown via `HostFacade`. At minimum, document this behavior in the API spec so clients are not surprised.

**W-002: `create_session` handler does not validate `cwd` against path traversal**
- **Issue**: The `cwd` field from the HTTP request is converted directly to `PathBuf` without validating it is absolute or within the workspace root.
- **Source**: `crates/nexus-daemon-runtime/src/api/handlers/agent_host.rs:134-139`
- **Impact**: A malicious or buggy client could pass `cwd: "../../../etc"` which would be passed to the provider subprocess as its working directory. While the admission policy checks `workspace_root` for absolute path, it does not check per-session `cwd`.
- **Evidence**:
  ```rust
  cwd: req.cwd.map_or_else(
      || std::path::PathBuf::from("/tmp"),
      std::path::PathBuf::from,
  ),
  ```
- **Fix**: Validate `cwd` is absolute and optionally enforce it is within the configured workspace root. Reject with 400 Bad Request if validation fails.

**W-003: Native CLI provider `cancel()` is a no-op — no force-kill on hung processes**
- **Issue**: `ClaudeCliProvider::cancel()` returns `Ok(())` without terminating the child process. If a native CLI process hangs (e.g., waiting on stdin), there is no host-level mechanism to force-kill it.
- **Source**: `crates/nexus-agent-host/src/providers/native_cli/claude.rs:288-306`
- **Impact**: A hung native CLI process could consume resources indefinitely. The `shutdown()` method does call `child.kill()`, but cancellation mid-operation is unsupported.
- **Evidence**:
  ```rust
  async fn cancel(&self, _session: &ManagedSessionHandle, _op_id: HostOperationId) -> HostResult<()> {
      // Native CLI cancellation in Wave 1 is a no-op ...
      Ok(())
  }
  ```
- **Fix**: Store the child process handle per-operation and implement `cancel()` by sending `SIGTERM`/`kill` with a configurable grace timeout, matching the compass §Shutdown specification.

**W-004: `probe()` uses `which` command — not portable to Windows**
- **Issue**: `ClaudeCliProvider::probe()` spawns `which` to check command existence. The `which` command does not exist on Windows, causing the provider to always report unavailable.
- **Source**: `crates/nexus-agent-host/src/providers/native_cli/claude.rs:147-173`
- **Impact**: Native CLI provider is non-functional on Windows in Wave 1.
- **Evidence**:
  ```rust
  let which_result = tokio::process::Command::new("which")
      .arg(&self.command)
      .output()
      .await;
  ```
- **Fix**: Use a cross-platform command lookup (e.g., the `which` crate or the same PATH-scan logic used in `discovery/path_scan.rs`).

**W-005: PATH scan `find_command()` does not check executable permissions**
- **Issue**: `find_command()` checks `candidate.is_file()` but does not verify the file is executable. On Unix, a non-executable file named `claude` would be discovered and later fail at spawn time.
- **Source**: `crates/nexus-agent-host/src/discovery/path_scan.rs:96-104`
- **Impact**: Non-executable files are reported as valid providers, leading to a later `LaunchFailed` error that could have been avoided at discovery time.
- **Evidence**:
  ```rust
  fn find_command(path_dirs: &[PathBuf], command: &str) -> Option<PathBuf> {
      for dir in path_dirs {
          let candidate = dir.join(command);
          if candidate.is_file() {
              return Some(candidate);
          }
      }
      None
  }
  ```
- **Fix**: Add `is_executable()` check (via `std::os::unix::fs::PermissionsExt` on Unix, or check `.exe` extension on Windows).

**W-006: `AcpSdkAdapter::Drop` uses fire-and-forget cleanup — may leak child processes**
- **Issue**: The `Drop` implementation spawns an async task to join the setup task. If the tokio runtime is shutting down, this spawned task may not execute, leaving the ACP child process and SDK connection alive.
- **Source**: `crates/nexus-acp-host/src/client.rs:858-873`
- **Impact**: During daemon subsystem restart (not process exit), the ACP child process may become orphaned.
- **Evidence**:
  ```rust
  impl Drop for AcpSdkAdapter {
      fn drop(&mut self) {
          if let Some(setup_task) = self.setup_task.take() {
              if let Ok(handle) = tokio::runtime::Handle::try_current() {
                  handle.spawn(async move { let _ = setup_task.await; });
              }
          }
      }
  }
  ```
- **Fix**: Add an explicit `async fn close()` or `shutdown()` method to `AcpSdkAdapter` that synchronously terminates the child process before dropping. Ensure `AgentHostSubsystem::shutdown()` calls this.

### 🟢 Suggestion

**S-001: Clippy dead_code warning in `path_scan.rs`**
- **Issue**: `cargo test -p nexus-agent-host` emits: `warning: function 'default_config' is never used`.
- **Source**: `crates/nexus-agent-host/src/discovery/path_scan.rs:155`
- **Fix**: Remove the unused `default_config()` function or add `#[allow(dead_code)]` with a comment explaining why it is retained.

**S-002: `list_sessions` API returns dummy data with empty session IDs**
- **Issue**: `GET /v1/local/agent-host/sessions` returns `SessionResponse` items with empty `session_id` and `provider_id` strings.
- **Source**: `crates/nexus-daemon-runtime/src/api/handlers/agent_host.rs:158-176`
- **Impact**: API consumers cannot distinguish sessions or link them to providers.
- **Fix**: Either implement full session listing (preferred) or return a 501/placeholder response that clearly indicates the data is incomplete.

**S-003: Admission policy `check_workspace_root` only checks absolute, not containment**
- **Issue**: `check_workspace_root` verifies the path is absolute but does not ensure it is within an allowed directory tree.
- **Source**: `crates/nexus-agent-host/src/policy/admission.rs:110-118`
- **Fix**: For defense-in-depth, consider adding a containment check (e.g., `workspace_root.starts_with(allowed_base)`) if the host ever supports multi-tenant or sandboxed execution.
