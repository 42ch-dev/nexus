## Performance & Reliability Assessment

### Smoke Test (`all_embedded_presets_pass_strict_validation_gate`)
- **Fast**: Yes. Pure in-memory operations only. Uses `EMBEDDED_PRESETS` (compiled via `include_dir!`), so no filesystem I/O, no network calls, no LLM invocations.
- **Deterministic**: Yes. Validates against a fixed `CapabilityRegistry::with_builtins()` and static embedded YAML.
- **Bounded**: Yes. Iterates over `list_embedded_presets()` (currently 6 presets), running O(n) validation per preset where n = number of states + enter actions.
- **Execution time**: <50ms in practice (sub-test timing not isolated, but full 353-test suite completes in ~1.85s).

### kb-extract Inner Graph (`extraction_graph`)
- **Runtime overhead**: Minimal. The new inner graph is a single-node graph (`extract`, kind `acp_prompt`). At runtime, `InnerGraphTask` spawns one child session and polls it to completion. This is proportional to the graph size (1 node) and is the intended functional behavior (B3: repair inner graph wiring).
- **No unbounded recursion**: The loader already enforces cycle-free inner graphs (`build_inner_graphs` in `loader.rs`).
- **No resource leak**: Child sessions are scoped to the parent state execution and cleaned up by the engine.

### TD-V131-08 Test (`memory_augmented_rule_exit_is_explicit_always_true`)

- **Fast**: Yes. Sub-millisecond execution.
- **Correct**: The test locks the contract that `ExitWhen::Rule` (unit variant) means "transition as soon as enter action completes."
