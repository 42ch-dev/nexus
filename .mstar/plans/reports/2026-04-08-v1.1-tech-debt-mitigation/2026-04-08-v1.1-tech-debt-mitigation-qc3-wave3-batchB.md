#### S-QC3-B-W1: CI job column extraction uses fragile grep/sed patterns

- **Location**: `.github/workflows/ci.yml:134-151`
- **Issue**: The `reference_sources` column extraction uses `sed` and `grep` to parse SQL column definitions. This works for current schema but could break if schema format changes (e.g., inline comments, multi-line constraints).
- **Current approach**:
  ```bash
  CLI_COLS=$(sed -n '/CREATE TABLE IF NOT EXISTS reference_sources/,/);/p' crates/nexus42/src/db/mod.rs \
    | grep -v '^//' \
    | grep -oP '^\s+\K\w+' \
    | sort \
    | tr '\n' ' ')
  ```
- **Suggestion**: Consider adding a comment in CI explaining the assumptions (single-line column definitions, no inline comments in column list) so future maintainers understand limitations.
- **Severity**: Suggestion (low priority, current approach is functional)
- **Confidence**: Medium
