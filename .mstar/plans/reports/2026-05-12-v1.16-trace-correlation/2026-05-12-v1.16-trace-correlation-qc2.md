## Security & Correctness Review Notes
- `validate_run_id_safe()` rejects empty strings, `/`, `\`, `..`, and control characters before a user-supplied run ID can be used for trace-path construction. This blocks the relevant path traversal forms for `~/.nexus42/acp/runs/<run_id>/trace.jsonl`.
- Generated IDs use `run_` / `cap_` plus UUID simple hex output, so generated correlation IDs are path-safe and environment-safe under the same validator.
- Trace events are serialized with `serde_json::to_string()` and then newline-delimited. User-controlled fields such as `cwd`, `error`, and `run_id` are JSON escaped, so embedded quotes or newline characters cannot create additional JSONL events.
- `NEXUS_RUN_ID` propagation is limited to the validated/generated run ID. It is not a credential and does not grant permissions by itself.
- `spawn_with_env()` only adds caller-provided key/value pairs on top of the inherited subprocess environment. The new production call site sets only `NEXUS_RUN_ID`; it does not introduce a capability token or broaden agent permissions beyond the existing inherited environment behavior.
- Trace writes are best-effort and non-fatal after run ID validation. This is acceptable for correlation telemetry because failures do not change run execution semantics.
