### 🟡 Warning

#### W-001: Fallible `From` implementations with `.unwrap()` can panic
- **Location**: `crates/nexus-cloud-domain/src/pairing.rs` (lines 151-158), `crates/nexus-narrative/src/story_manifest.rs`, `crates/nexus-narrative/src/world_membership.rs`
- **Issue**: The `From<Pairing> for nexus_contracts::Pairing`, `From<StoryManifest>`, and `From<WorldMembership>` implementations use `.unwrap()` on `FromStr::from_str()` for enum conversions. This can panic if the internal string representation doesn't match a valid enum variant.
- **Evidence**: 
  ```rust
  // pairing.rs
  #[allow(clippy::fallible_impl_from)]
  impl From<Pairing> for nexus_contracts::Pairing {
      fn from(d: Pairing) -> Self {
          Self {
              // ...
              pairing_source: nexus_contracts::PairingSource::from_str(&d.pairing_source).unwrap(),
              status: nexus_contracts::PairingStatus::from_str(&d.status).unwrap(),
              // ...
          }
      }
  }
  ```
- **Risk**: Medium. Domain logic controls state transitions, so values are set internally (not from untrusted input). However, inconsistent with `User` which correctly uses `TryFrom`. Future callers could receive panics if contracts contain invalid data.
- **Fix**: Consider converting to `TryFrom` pattern (like `User`) or ensure invariant enforcement that guarantees valid enum strings. Alternatively, document the invariant clearly.

#### W-002: Test-only `.unwrap()` usage is acceptable but should be confined to `#[cfg(test)]`
- **Location**: Multiple files in new scaffold crates
- **Issue**: Many `.unwrap()` calls exist in test code (acceptable), but some domain logic also contains `.unwrap()` in tests without `#[cfg(test)]` gating. Verified: all production paths use proper error handling (Result types, CloudDomainError, etc.)
- **Evidence**: grep output shows `.unwrap()` primarily in test modules and `#[test]` functions
- **Risk**: Low. No production-path unwraps found after manual review of error handling patterns.
- **Fix**: Ensure all `.unwrap()` in test helpers are marked with `#[cfg(test)]` or moved to test modules.
