## Findings

### 🟡 Warning

#### W-01: `tempfile` declared in `[dependencies]` instead of `[dev-dependencies]` in `nexus-acp-host/Cargo.toml`
- **Location**: `crates/nexus-acp-host/Cargo.toml`, line 25
- **Evidence**: `tempfile = "3"` is in `[dependencies]` section. Grep confirms `tempfile` is only referenced inside `#[cfg(test)]` blocks across all `crates/nexus-acp-host/src/` files (`session_manager.rs` tests, `policy.rs` tests, `registry.rs` tests).
- **Impact**: Production builds of `nexus-acp-host` (and downstream consumers like `nexus42`) pull in `tempfile` unnecessarily, bloating compile times and dependency tree.
- **Fix**: Move `tempfile = "3"` from `[dependencies]` to `[dev-dependencies]`.
- **Confidence**: High

#### W-02: `session_capture` re-exports removed from compat shim without explicit API deprecation
- **Location**: `crates/nexus42/src/acp/mod.rs` (new)
- **Evidence**: Old `acp/mod.rs` re-exported `session_capture::{spawn_submit_capture, SessionCapture, SessionDigest}` at the `crate::acp::*` level. New shim does `pub use nexus_acp_host::*;` + `pub mod session_capture;`, so these types are now only reachable via `crate::acp::session_capture::SessionCapture`.
- **Impact**: Low immediate impact (grep shows no external consumers of `crate::acp::SessionCapture` in `nexus42/src/`). However, this is a subtle API surface change. If any external crate or doc example referenced `nexus42::acp::SessionCapture`, it would break silently.
- **Fix**: Either (a) add back `pub use session_capture::{spawn_submit_capture, SessionCapture, SessionDigest};` to the compat shim, or (b) add a doc comment in `acp/mod.rs` explicitly noting the removal and the new path.
- **Confidence**: Medium

### 🟢 Suggestion

#### S-01: CI gate regex does not check transitive linkage
- **Location**: `.github/workflows/ci.yml`, `verify-no-acp-in-daemon` job
- **Evidence**: The gate checks `crates/nexus42d/Cargo.toml` directly with `rg -n '^\s*(nexus-acp-host|agent-client-protocol)\s*='`. It does not verify that none of `nexus42d`'s dependencies transitively pull in `nexus-acp-host`.
- **Impact**: Low for current workspace (none of `nexus42d`'s deps currently depend on `nexus-acp-host`). Medium if a future refactor introduces an intermediate crate that links ACP.
- **Fix**: Add a transitive check to the CI gate, e.g.:
  ```bash
  cargo tree -p nexus42d | grep -E 'nexus-acp-host|agent-client-protocol' && exit 1 || true
  ```
  This provides defense-in-depth against transitive leakage.
- **Confidence**: Medium

#### S-02: `AcpWorkerArgs.creator` accepts empty string without validation
- **Location**: `crates/nexus42/src/commands/acp_worker.rs`, line 7–10
- **Evidence**: `#[arg(long)] pub creator: String` has no `value_parser` or custom validation.
- **Impact**: Low (Phase-1 stub; WS2 replaces with full JSON-RPC loop).
- **Fix**: Add `#[arg(long, value_parser = non_empty_string)]` or at minimum a runtime check with a clear error message.
- **Confidence**: High

#### S-03: `acp_worker` stub JSON-RPC is not fully conformant
- **Location**: `crates/nexus42/src/commands/acp_worker.rs`, lines 15–20
- **Evidence**: The stub outputs `{"jsonrpc":"2.0","method":"worker/initialize","result":{...}}`. A proper JSON-RPC 2.0 response requires an `id` field matching the request's `id`. The stub also doesn't read stdin (the request) at all.
- **Impact**: Very low (intentional stub).
- **Fix**: Document in a `TODO(WS2)` comment that the current output is placeholder and the real implementation must parse the request `id` and echo it back.
- **Confidence**: High

#### S-04: `nix` dependency in `nexus-acp-host` may be unused in production code
- **Location**: `crates/nexus-acp-host/Cargo.toml`, line 29
- **Evidence**: `nix` is declared for Unix signal handling. Need to verify it's actually used in production code (not just tests). A quick grep for `nix::` in `crates/nexus-acp-host/src/` would confirm. If unused, it should be removed to reduce platform-specific dep complexity.
- **Impact**: Low (small dep, but adds conditional compilation surface).
- **Fix**: Verify usage; remove if dead.
- **Confidence**: Low (needs verification — not confirmed dead).
