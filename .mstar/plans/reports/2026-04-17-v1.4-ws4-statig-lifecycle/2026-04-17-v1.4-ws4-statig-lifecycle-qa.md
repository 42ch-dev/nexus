### Medium

**R1 (medium): WS2 engine dependency — `Running.entry` engine session start is stubbed**

- **Source**: QC2 W8
- **Scope**: `crates/nexus42d/src/lifecycle/actions.rs`, `Running.entry`
- **Description**: The `Running.entry` action logs "would start _system.maintenance session (engine not yet available)" but does not actually create the engine session. This is a WS2 orchestration dependency.
- **Decision**: Defer to WS2 integration
- **Owner**: @fullstack-dev-2 (WS2)
- **Target**: V1.5 or WS2 delivery

### Low

**R2 (low): dispatch() spawns independent tokio tasks per event**

- **Source**: QC1 C2 (downgraded by QC3 + PM consolidation)
- **Scope**: `crates/nexus42d/src/lifecycle/state.rs`, `dispatch()` method
- **Description**: Each `dispatch(event)` call spawns a new `tokio::spawn` task. Documented as FIFO-safe via `Mutex` serialization on the statig machine, but a single event loop with mpsc channel would be cleaner and more predictable.
- **Decision**: Accept current implementation; refactor in WS2 scope
- **Owner**: @fullstack-dev-2
- **Target**: WS2

**R3 (low): `started_at` field not populated in v2 response**

- **Source**: QC2 W7
- **Scope**: `crates/nexus42d/src/api/handlers/runtime.rs`, `daemon_status` handler
- **Description**: `started_at` is always `None`. The HSM does not track when `Running` was entered.
- **Decision**: Defer
- **Owner**: @fullstack-dev-2
- **Target**: V1.5

### Nit

**R4 (nit): Hardcoded sleep values in tests (100ms–200ms)**

- **Source**: QC2 W6, QC3 Q3-4
- **Scope**: All 5 lifecycle test files
- **Description**: Tests use `tokio::time::sleep(Duration::from_millis(50..200))` to wait for async dispatch propagation. Fragile on slow CI runners.
- **Decision**: Accept for now; refactor when tests show flakiness
- **Owner**: @qa-engineer / @fullstack-dev-2
- **Target**: Test quality pass
