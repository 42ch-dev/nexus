### [Nit]

1. **N-QC3-1: `creator_register_twice_does_not_double_write_credentials` assertion documents, but does not enforce, idempotency**
   - **Scope**: `crates/nexus42/tests/creator_register_e2e.rs` lines 278–289
   - **Finding**: The test comment correctly states "the CLI currently has no idempotency guard". The test verifies the credential is not corrupted after a second run, but it does not enforce a meaningful idempotency contract (e.g., second run should either be a no-op or return a stable error). The current assertion only checks `!api_key.is_empty()` which passes regardless of whether the CLI re-registers, re-verifies, or short-circuits.
   - **Evidence**: The test's own comment at line 281: `"the CLI currently has no idempotency guard"`.
   - **Risk**: If idempotency behavior changes (e.g., CLI starts returning errors on re-registration), this test would not catch the regression.
   - **Recommendation**: Either (a) add a comment noting this test is a *regression guard for data-integrity*, not idempotency semantics, or (b) expand the assertion to also verify a specific exit-code or stdout message that reflects the current observed behavior (no error, same result).
   - **Severity**: nit — test still provides value as a data-integrity guard.
