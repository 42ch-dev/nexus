### 🟡 Warning

#### F-002: Overlapping `doctor` commands in daemon and acp may confuse users

**Location**: `crates/nexus42/src/commands/daemon.rs` (daemon_doctor) and `crates/nexus42/src/commands/acp.rs` (cmd_doctor)

**Issue**: Both `nexus42 daemon doctor` and `nexus42 acp doctor` check daemon connectivity as their first diagnostic step. This overlap could confuse users about which command to use for troubleshooting:

- `daemon doctor`: checks daemon connectivity, PID file, home directory (3 checks)
- `acp doctor`: checks daemon connectivity, ACP registry reachability, configuration (3 checks)

Both print "Daemon connectivity..." as their first check with similar output format.

**Impact**: Users may run one when the other is more appropriate. The semantic distinction (daemon diagnostics vs ACP capability diagnostics) should be clearer.

**Recommendation**: 
1. Add explicit scope differentiation in help text: "daemon doctor" focuses on daemon lifecycle diagnostics; "acp doctor" focuses on ACP connectivity diagnostics
2. Consider having `acp doctor` skip the daemon connectivity check (assume daemon is prerequisite) or reference daemon doctor if daemon issues are detected

**Evidence**: Both functions have `print!("  [1/3] Daemon connectivity... ")` pattern.

**Confidence**: High

**Location**: `crates/nexus42/src/commands/agent.rs` lines 865-1077 (tests module)

**Issue**: The `agent.rs` module contains comprehensive unit tests (lines 865-1077) that test internal functions like `print_list_table`, `print_show_details`, `describe_distribution`, etc. While integration tests in `cli_agent.rs` were migrated to the `acp` namespace, the internal unit tests remain in `agent.rs`.

Since `agent.rs` is marked deprecated with `#![allow(dead_code)]`, these tests test functionality that is:
1. Hidden from users (no CLI route visible)
2. Duplicated in `acp.rs` (which has its own similar tests)

**Impact**: Test duplication and confusion about which module is authoritative. When `agent.rs` is eventually removed, these tests will need migration or deletion.

**Recommendation**: Document a timeline for `agent.rs` removal. If the module will be removed in V1.17+, remove the tests now since `acp.rs` covers the functionality. If backward compat is required longer, mark tests as `#[ignore]` or add comments explaining retention rationale.

**Evidence**: `git diff feature/v1.16..HEAD -- crates/nexus42/src/commands/agent.rs` shows only deprecation header added, tests unchanged.
