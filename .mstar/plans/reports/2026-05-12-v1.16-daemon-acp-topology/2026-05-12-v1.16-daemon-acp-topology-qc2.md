### 🟡 Warning
- **W-001 — `daemon restart` can silently leave the old daemon running when stop falls back to port-based termination.**
  - Evidence: `restart_daemon()` unconditionally treats `stop_daemon(port).await?` as a successful stop before calling `start_daemon()` (`crates/nexus42/src/commands/daemon.rs:500-504`). In the Unix no-PID-file fallback, `stop_daemon()` sends SIGTERM to PIDs returned by `lsof`, waits, prints `Daemon may still be running` if health check still succeeds, but still returns `Ok(())` (`crates/nexus42/src/commands/daemon.rs:405-437`). `start_daemon()` then sees the still-running daemon and returns `Ok(())` with "already running", so a restart may report success without replacing the process.
  - Impact: state consistency / correctness issue for the new restart command; a user may believe a restart applied new daemon code/config when it did not.
  - Fix: make the port-based stop fallback return an error if the daemon is still healthy after the wait, or have `restart_daemon()` explicitly verify the daemon is down before starting and fail if it is not.
- **W-002 — ACP registry table rendering can panic on UTF-8 descriptions from the public registry.**
  - Evidence: `print_list_table()` truncates external registry descriptions with byte indexing (`&d[..57]`) after checking `d.len() > 60` (`crates/nexus42/src/commands/acp.rs:497-503`). Rust string slices must land on UTF-8 character boundaries; a non-ASCII description whose byte 57 is not a boundary will panic.
  - Impact: correctness/input-validation issue in new visible `nexus42 acp registry list` / `nexus42 acp agent list`; registry metadata is external input.
  - Fix: truncate by characters or display width, e.g. `let prefix: String = d.chars().take(57).collect(); format!("{prefix}...")`, and add a regression test with a multibyte description longer than the display limit.

### 🟢 Suggestion
- **S-001 — Treat hidden commands as a compatibility surface, not a security boundary.** `#[command(hide = true)]` removes commands from help output but keeps them invokable. That is acceptable for backward compatibility if intentional, but docs/release notes should not imply these commands are disabled or protected.
- **S-002 — Prefer redacting user-home paths in daemon diagnostics where possible.** `daemon logs` / `daemon doctor` print `$HOME/.nexus42/...` paths. No credentials were observed, and this is a local CLI, but `~/.nexus42/...` would reduce incidental sensitive path exposure in copied logs.
- **S-003 — Orchestrate stubs have no side effects, but echo raw `schedule_id`.** Current implementation does not execute, persist, or shell out with `schedule_id`, so injection risk is low. If IDs may later come from external sources, add validation before wiring real behavior.
