  - Fix: either tighten validation (add alnum check post-prefix or regex) + update DB CHECK, or align docs/error to actual "prefix + non-empty" rule.

  - Impact: can create timeline events under arbitrary branch strings (including non-`fbk_root_*` or cross-world). No branch table yet (per plan non-goals), but violates "per-branch monotonic sequence" intent and future fork semantics. Correctness risk for narrative timeline integrity.

### 🟢 Suggestion
- **S1**: All new write queries use runtime `sqlx::query()` + SAFETY comments (per crate AGENTS.md exception for avoiding sqlx-prepare churn). Consider adding a CI step that runs `cargo sqlx prepare` for the write module or documents the exception scope. (narrative_write.rs:149,216,232,245 + tests)
- **S2**: No length or content validation on `title`/`summary`/`slug`/`visibility`/`time_policy`/`event_type` before DB insert. Large inputs (MB-scale) possible via CLI; could affect storage/perf. Add bounds (e.g. title <= 200 chars) + sanitize slug always (even when `--slug` supplied).
- **S3**: Sequence allocation (`MAX+1` then INSERT + 2067 catch) is racy under concurrent writers (multiple `nexus42` processes or daemon + CLI). Acceptable for local single-user authoring but document the assumption; consider advisory lock or retry loop for robustness.
- **S4**: In `run_create`, `--slug` bypasses `slug_from_title` sanitization (only derived path lowercases + alnum/-). Inconsistent slug hygiene. Always run sanitization or reject invalid `--slug`.
- **S5**: Add `#[arg(value_parser = ...)]` or custom validator on `world_id` / `--world-id` in `WorldCommand` and `KbCommand` (world scope) for early clap-level rejection with clean error (before pool open or DB roundtrip). (world.rs:35, mod.rs:640+)
- **S6**: CLI error paths wrap internal `NarrativeWriteError` / sqlx errors directly into `CliError::Other`. Consider a small mapper that redacts DB internals for user-facing messages while preserving detail in logs/tracing.
- **S7**: `branch_id` fallback query in `run_event_add` and root lookup assume `root_fork_branch_id` is always present/non-NULL for existing worlds (created by `create_world`). Add defensive handling or migration guard if older data can have NULL.
- **S8**: Test coverage good for happy + error paths (FK, prefix, sequence, roundtrips via gateway). Consider adding a test that exercises invalid branch_id formats and very long titles/summaries (boundary).
