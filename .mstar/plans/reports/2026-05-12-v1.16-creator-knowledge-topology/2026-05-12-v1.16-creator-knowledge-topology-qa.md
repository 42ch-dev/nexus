## Findings

### Low — Pre-existing Flaky Test

**F-001: `auth::tests::get_returns_none_for_unknown_creator` fails under parallel test execution**
- **Severity**: Low
- **Location**: `crates/nexus42/src/auth/mod.rs:330`
- **Root Cause**: The test uses `AuthStore::default()` with `creators: None`. The `get_creator_api_key` method falls through to `Self::load()` which reads `~/.nexus42/auth.json`. When other parallel tests modify this file concurrently, a race condition causes an EOF parse error.
- **Evidence**: Test passes when run in isolation (`cargo test -p nexus42 --lib auth::tests::get_returns_none_for_unknown_creator` → ok). Test also passes on base commit `13fd99d` when run alone.
- **Not introduced by this plan**: `auth/mod.rs` has zero changes in the diff range `13fd99d..HEAD`.
- **Already tracked**: This is registered as `R-V113-007` in `status.json` residual findings for plan `2026-05-06-v1.13-oss-forward-delivery`.
- **Recommendation**: No action for this plan. Track under existing residual.
