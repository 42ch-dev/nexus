## QC Fix Verification

### C1: `link_to_platform` returns distinct errors for "not found" vs "already linked"

**Status**: ✅ Fixed

| Scenario | Command | Output |
|----------|---------|--------|
| Linking non-existent identity | `nexus42 identity link ctr_nonexistent --platform-id ctr_abc...` | `Error: local database error: local identity 'ctr_nonexistent' not found` |
| Linking already-linked identity | `nexus42 identity link ctr_localf6516bb661fc --platform-id ctr_different...` | `Error: local database error: local identity '...' is already linked to a platform creator; cannot link again` |

Distinct error messages confirmed. Previously both returned the same "not found" error.

### W1: Unknown `identity_type` returns error instead of silent default

**Status**: ✅ Fixed

```
$ nexus42 identity create --kind nonexistent
error: invalid value 'nonexistent' for '--kind <KIND>'
  [possible values: anonymous, persistent]
```

Clap validation rejects unknown values before they reach domain logic. No silent default.

### W2: CLI `link` validates `platform_creator_id` format

**Status**: ✅ Fixed

```
$ nexus42 identity link ctr_anon... --platform-id "invalid-format"
Error: Domain error: invalid ID format: platform_id 'invalid-format' does not match CreatorId pattern (expected: ctr_ followed by alphanumeric characters)

$ nexus42 identity link ctr_anon... --platform-id "pc_abc123xyz"
Error: Domain error: invalid ID format: platform_id 'pc_abc123xyz' does not match CreatorId pattern (expected: ctr_ followed by alphanumeric characters)
```

Invalid formats rejected with descriptive error. Previously skipped validation.
