## 1. Problem Statement

Findings are visible on `creator works status` but do not influence `novel-writing` outline/draft prompts. §5.5.2 deferred consumer behavior blocks the quality loop value proposition.

---

## 2. Goals

1. Query open findings for active `work_id` + selected chapter (and work-level findings with `chapter IS NULL` if applicable).
2. Inject a bounded, prompt-safe summary block into `novel-writing` outline and draft template context.
3. Cap token footprint (max findings count + truncated body per overlay §2).
4. Hermetic test: seeded findings appear in rendered prompt context or snapshot hook.

---

## 3. Non-goals

- Mutating findings status from produce stage.
- `AGENTS.md` rules path (P2).
- Brainstorm/review-master preset changes (already consume findings).

---

## 4. Acceptance Criteria

1. `novel-writing` with open findings for chapter N includes findings block in outline/draft prompt assembly.
2. No findings → no block (no empty sentinel noise).
3. Overlay §2 limits enforced (count + char cap).
4. Auto-chain produce stage behavior unchanged aside from enriched prompts.
