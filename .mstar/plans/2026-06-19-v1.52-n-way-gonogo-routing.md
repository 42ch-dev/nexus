## 1. Problem Statement

V1.42 P2 shipped binary GO/NOGO routing (`GoNogoNext { go, nogo }`) for `llm_judge` states. This is a 2-way branch — the judge returns a boolean (`true` → GO, `false`/unavailable → NOGO). Real presets need N-way routing: a judge that returns one of N labels (e.g. `"outline"`, `"research_more"`, `"abandon"`), and edges keyed by those labels. T-B P0 generalizes the binary `GoNogoNext` struct into an N-way `LabeledNext` edge type while preserving backward compatibility with all existing binary GO/NOGO presets.

---

## 2. Goals

1. Generalize `GoNogoNext` → `LabeledNext` with `label: String` field (keyed by judge-returned label string).
2. Rename `NextTarget::GoNogo(GoNogoNext)` → `NextTarget::Labeled(LabeledNext)` — or keep `GoNogo` variant for backward compat and add new `Labeled` — agnostic to branch count.
3. Extend edge `kind` to carry a `label: <str>` field; matching uses the judge-returned label string at runtime.
4. Preserve **binary GO/NOGO behavior** — all existing embedded presets continue to work without changes (label defaults to `"go"` / `"nogo"`).
5. Extend reachability validator (`validate_reachability`) to handle N-way labels: all label values referenced in `next` edges must be producible by the judge node; all judge-returned label values must have at least one `next` edge; reject orphan labels.

4. **✅ SHIPPED (implementation differed from plan)** — Reachability validator: (a) each label target is validated as a valid state ID at load time, (b) duplicate labels within `Labeled` edges are rejected (implemented as `check_labeled_edge_duplicates`; full label-coverage/producibility checking is deferred to V1.52 P-last WL-A per residual R-V152-LABELCOV). Runtime coverage: `resolve_labeled_target` logs `tracing::warn!` when no label matches (observable, not silent). (c) all embedded presets pass strict validation gate.
5. **✅ SHIPPED** — Spec overlay body in `preset-conditional-routing.md` documents N-way labeled routing with binary→Labeled auto-conversion, substring matching semantics, no-match deterministic fail, and context key `_judge_label`.
6. **✅ SHIPPED** — All hermetic tests pass: `cargo test -p nexus-orchestration` — 705 passed, 0 failed. New test modules: `tasks::tests` (9 `resolve_labeled_target` unit tests) + `tests/labeled_routing.rs` (5 integration tests).

**Additional shipped scope (not in original plan):**
- Binary→Labeled auto-conversion: `resolve_labeled_target` handles `GoNogo` edges as labeled edges with labels `"go"` / `"nogo"` (same preset reachable via either routing API).
- No-match deterministic fail: returns `Err(TaskExecutionFailed)` not `WaitForInput` (W-QC3-3 fix).
- Descending-length label sort to mitigate substring matching issues.
