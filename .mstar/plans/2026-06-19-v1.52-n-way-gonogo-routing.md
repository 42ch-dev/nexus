## 1. Problem Statement

V1.42 P2 shipped binary GO/NOGO routing (`GoNogoNext { go, nogo }`) for `llm_judge` states. This is a 2-way branch ‚Äî the judge returns a boolean (`true` ‚Üí GO, `false`/unavailable ‚Üí NOGO). Real presets need N-way routing: a judge that returns one of N labels (e.g. `"outline"`, `"research_more"`, `"abandon"`), and edges keyed by those labels. T-B P0 generalizes the binary `GoNogoNext` struct into an N-way `LabeledNext` edge type while preserving backward compatibility with all existing binary GO/NOGO presets.

---

## 2. Goals

1. Generalize `GoNogoNext` ‚Üí `LabeledNext` with `label: String` field (keyed by judge-returned label string).
2. Rename `NextTarget::GoNogo(GoNogoNext)` ‚Üí `NextTarget::Labeled(LabeledNext)` ‚Äî or keep `GoNogo` variant for backward compat and add new `Labeled` ‚Äî agnostic to branch count.
3. Extend edge `kind` to carry a `label: <str>` field; matching uses the judge-returned label string at runtime.
4. Preserve **binary GO/NOGO behavior** ‚Äî all existing embedded presets continue to work without changes (label defaults to `"go"` / `"nogo"`).
5. Extend reachability validator (`validate_reachability`) to handle N-way labels: all label values referenced in `next` edges must be producible by the judge node; all judge-returned label values must have at least one `next` edge; reject orphan labels.

---

## 3. Non-goals

- T-B P1 merge point semantics (sibling plan ‚Äî adds explicit join/merge after branch fan-out).
- Auto-promotion (T-A P0).
- Expression-based `NextTarget::Conditional` activation (still deferred per preset-conditional-routing.md ¬ß3).
- New wire JSON schemas (local-only Rust types per compass ¬ß0.1 row 15).
- Any changes to `crates/nexus-orchestration` outside the files listed in ¬ß5.

---

## 4. Acceptance Criteria

1. `LabeledNext` struct exists with `label: String` + `target: String` (or `go`/`nogo` for backward compat) fields; backward-compat deserialization accepts YAML `{ go: <s>, nogo: <s> }` and maps it to `LabeledNext { label: "go", target: <s> }` + `LabeledNext { label: "nogo", target: <s> }`.
2. `NextTarget::Labeled(Vec<LabeledNext>)` variant carries 2+ labeled edges; backward-compat deserialization accepts the old `{ go, nogo }` shape.
3. Binary GO/NOGO presets (`novel-writing`, `novel-branching-test`, etc.) load and execute identically ‚Äî all 12 existing `judge_next_action` tests pass unchanged.
4. Reachability validator: (a) each label in a `Labeled` edge's target string must be a valid state ID, (b) orphan labels (labels defined in `next` edges but not producible by the judge) are rejected with a diagnostic, (c) all label values the judge can produce must have at least one `next` edge.
5. Spec overlay body in `preset-conditional-routing.md` documents N-way labeled routing with 2+ backward-compat examples.
6. All hermetic tests pass: `cargo test -p nexus-orchestration` ‚Äî with new test modules for `labeled_next` routing and `reachability_n_way`.
