## Goal

Close DF-31 and DF-42 simultaneously by delivering production-grade `workspace.open`/`workspace.commit` with file-level OCC (content hash), persistent DB-backed sessions, a changes[] payload manifest, and a full `/v1/local/{world,work,kb,schedule,workspace,findings}` Local API scope redesign — replacing the V1.55 skeleton with the complete production surface.

## Scope In

- File-level optimistic concurrency control: content hash per file on `workspace.open`, hash comparison on `workspace.commit`, conflict rejection on mismatch.
- Persistent workspace sessions: migrate from in-memory `WorkspaceSessionManager` (V1.55 P1 skeleton) to DB-backed session table(s) with durable session identity, TTL, and expiry.
- changes[] payload: manifest of paths, content hashes, and operations (create/modify/delete) in the commit request, validated against the session snapshot.
- Local API redesign: define and implement `/v1/local/{world,work,kb,schedule,workspace,findings}` scope with coherent resource naming, unified error model, and spec documentation.
- Amend normative specs: `local-runtime-boundary.md`, `daemon-runtime.md`, `local-db-schema.md`, `concurrency.md` for OCC semantics.

## Scope Out

- DF-29 `registry.refresh` — that is P1, Wave 1 parallel.
- DF-56 conditional routing — P2/P3; P0 provides the workspace API surface those plans consume.
- R-V155P2-F002 (game-bible section_status) — that is P-last fix-wave.
- Cloud sync or platform workspace sessions.
- Multi-daemon / cross-process session negotiation (single-daemon model).
- Git-backed workspace sessions (file-level OCC only, not git-managed).

## Acceptance Criteria

- [ ] `workspace.open` returns a session with content hashes for all tracked files in the workspace scope.
- [ ] `workspace.commit` validates the changes[] manifest against the session snapshot; rejects on hash mismatch (OCC conflict).
- [ ] Sessions are persisted in SQLite (DB-backed), survive daemon restart, and expire per TTL.
- [ ] changes[] payload includes manifest path, content hash, and operation type; invalid manifests are rejected with typed errors.
- [ ] Local API `/v1/local/*` scope is redesigned and documented: coherent resource naming, unified error model, no ad-hoc single-endpoint additions.
- [ ] V1.55 P1 skeleton code is fully replaced with production implementation; no dual in-memory/DB session path.
- [ ] All amended specs reflect the new OCC, session, and API scope behaviour.
- [ ] P0 topic branch is merged to `iteration/v1.56` before tri-review.
