## 1. Problem Statement

World-bound Works do not receive the spec'd World context block in `novel-writing` outline/draft prompts. LLM drafts drift from World KB character/location names and active rules.

---

## 2. Goals

1. **Refactor** existing `fetch_world_kb` in `nexus-moment-context-assembly` into a shared query+format module (grill-me #12); add `format_chapter_kb_block(world_id, world_refs, chapter)` for novel YAML shape.
2. Orchestration calls moment-context-assembly — **no** inline query in `nexus-orchestration`; optional thin capability wrapper only.
3. Build World context block YAML per novel-workflow-profile §3.5.1.3 before outline and draft states.
4. Populate `characters_in_chapter` / `locations_referenced` from `world_refs` + heuristics.
5. Include high-priority items (`novel_category` foundation/rules via body filter) in `active_rules`.
6. Omit block entirely for worldless Works.
7. **No new CLI** — debug via existing `creator kb --scope world list/search` or `platform context assemble-moment`.

---

## 3. Non-goals

- Full World KB graph retrieval in every prompt (compact slice only).
- kb-extract promotion (P3).
- i18n of context block labels.

---

## 4. Acceptance Criteria

1. World-bound Work: outline and draft prompts contain World context block with required fields.
2. Worldless Work: prompts unchanged from V1.39 behavior.
3. `world_refs` drive `characters_in_chapter` / `locations_referenced` when present.
4. Query API returns 404/remediation for missing `world_id`.
5. Hermetic tests: prompt rendering with/without World binding.
