## 1. Problem Statement

World-bound Works do not receive the spec'd World context block in `novel-writing` outline/draft prompts. LLM drafts drift from World KB character/location names and active rules.

---

## 2. Goals

1. Implement `creator kb query world <world_id>` or equivalent Local API for compact World KB slice.
2. Build World context block YAML per novel-workflow-profile §3.5.1.3 before outline and draft states.
3. Populate `characters_in_chapter` / `locations_referenced` from `world_refs` + heuristics.
4. Include high-priority `foundation` and `rules` items in `active_rules`.
5. Omit block entirely for worldless Works.

---

## 3. Non-goals

- Full World KB graph retrieval in every prompt (compact slice only).
- kb-extract promotion (P3).
- i18n of context block labels.

---

## 4. Acceptance Criteria

1. World-bound Work: outline and draft prompts contain World context block with required fields.
2. Worldless Work: prompts unchanged from V1.39 behavior.
3. `world_refs` drive `characters_in_chapter` / `locations_referenced` when present.
4. Query API returns 404/remediation for missing `world_id`.
5. Hermetic tests: prompt rendering with/without World binding.
