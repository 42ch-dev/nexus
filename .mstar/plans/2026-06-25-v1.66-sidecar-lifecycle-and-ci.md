## Goal

Make the V1.66 desktop app **self-contained** (bundled `nexus42` sidecar — auto-start/stop/restart-on-crash) and **shippable-as-dev-build** (macOS CI unsigned `.app`/`.dmg` build leg). The user double-clicks the `.app` and the daemon comes up transparently — no terminal. Builds on P0's shell scaffold.

## Tasks (stub — elaborated at implement)

- [ ] T1: Bundle `nexus42` as Tauri sidecar — `externalBin` config in `tauri.conf.json`; target-triple-renamed binaries (`nexus42-aarch64-apple-darwin` + `nexus42-x86_64-apple-darwin` for universal macOS); build pipeline produces both architectures (lipo or separate)
- [ ] T2: Sidecar lifecycle via `@tauri-apps/plugin-shell` Sidecar API — auto-start on app launch (`nexus42 daemon start --foreground --port <port> --cdn-url <if-any>`); stop on app quit (graceful SIGTERM → escalate); health probe (poll `GET /v1/local/health`); restart-on-crash with backoff
- [ ] T3: Port discovery (§5 #3) — `TauriClient` finds the sidecar port: lean fixed-default 8420 + `NEXUS_DAEMON_PORT` override + health probe; surface the resolved port to the SPA via capability-injection (boot wait until health probe succeeds, with timeout + user-visible error)
- [ ] T4: Daemon lifecycle UI — minimal desktop-only surface: daemon status indicator (starting/healthy/degraded/stopped), "Restart daemon" action; reuse V1.64 daemon-health-indicator patterns (desktop-augmented)
- [ ] T5: Asset serving confirmation (§5 #4) — Tauri webview serves `apps/web` dist directly (`frontendDist`); confirm the daemon's rust-embed path (V1.64) is unused in desktop mode but retained for the browser-tab flow
- [ ] T6: macOS CI `desktop-build` workflow job — `macos-latest` runner, `pnpm --filter desktop tauri build` (universal binary), unsigned `.app` + `.dmg` artifacts uploaded to the workflow run; path filter (`apps/**`, `crates/nexus42/**`, `schemas/**`) to avoid running on unrelated PRs; artifact retention (lean 90 days)
- [ ] T7: Dev + CI verification — `pnpm --filter desktop tauri build` produces a runnable unsigned `.app` on a clean macOS checkout; the `.app` launches, starts the bundled daemon, renders the SPA, Q5 actions work; CI job green

## Acceptance criteria

- The unsigned `.app` from `pnpm --filter desktop tauri build` launches on a clean macOS checkout **without** the user pre-starting the daemon — the sidecar autostarts; SPA renders; Q5 actions work.
- Sidecar lifecycle robust: quit app → daemon stops; crash sidecar → app restarts it; health-degraded → user-visible status.
- Port discovery works on default + override; conflict/diagnostics surfaced.
- `desktop-build` CI job green on `iteration/v1.66`; unsigned `.app` + `.dmg` artifacts uploaded.
- `cargo check` / `clippy` clean on any touched crates; `apps/web` + `apps/desktop` builds green.
