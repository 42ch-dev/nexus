## 1. Problem statement

The SOUL surface needs (A) a nullable `world_id` on `memory_fragments` to enable
the whole-vs-subset product model, and (B) an on-demand LLM synthesis path that
reflects the creator's accumulated fragments into a persisted narrative. Both are
backend; both must land before/independently of the frontend track.

## 2. Scope

### 2.A per-World data foundation (mechanical, additive)

- **Migration**: add nullable `world_id` to `memory_fragments` plus the
  narrative cache table in one migration under
  `crates/nexus-local-db/migrations/YYYYMMDD_memory_fragments_world_and_soul_narratives.sql`:

  ```sql
  ALTER TABLE memory_fragments ADD COLUMN world_id TEXT;

  CREATE INDEX IF NOT EXISTS idx_memory_fragments_creator_world_created
      ON memory_fragments (creator_id, world_id, created_at DESC);

  CREATE TABLE IF NOT EXISTS memory_soul_narratives (
      creator_id TEXT NOT NULL PRIMARY KEY,
      narrative TEXT NOT NULL,
      generated_at TEXT NOT NULL,
      fragment_count_at_generation INTEGER NOT NULL,
      max_fragment_created_at_at_generation TEXT,
      created_at TEXT NOT NULL,
      updated_at TEXT NOT NULL
  );
  ```

  Existing fragment rows default `NULL` and therefore represent Creator-core-only
  fragments. No foreign key is added in V1.81: fragments can outlive or precede
  richer World metadata, and `world_id` is a provenance tag, not ownership.
- **Record + insert**: add `pub world_id: Option<String>` to
  `nexus_local_db::memory_fragment::MemoryFragmentRecord`; update
  `create_fragment` to insert `(fragment_id, session_id, creator_id, keywords,
  summary, created_at, ttl, world_id)` with `sqlx::query!`. Update every row
  projection in `list_fragments`, `list_fragments_limited`,
  `list_fragments_by_session`, `list_fragments_filtered`, and tests to populate
  the field.
- **Threading**: propagate `world_id` from `PendingReviewInput` into the
  fragment row at every non-test `create_fragment` site. Grep-confirmed on
  2026-07-01 (excluding local-db tests) there are **4 direct call expressions in
  2 files**, representing the 3 production paths P0 must audit:
  1. `crates/nexus-daemon-runtime/src/api/handlers/memory.rs:887` — review
     pipeline `FragmentOnly` arm; pass `fragment.world_id` from
     `create_fragment_from_review(input)` into `MemoryFragmentRecord.world_id`.
  2. `crates/nexus-orchestration/src/capability/builtins/creator.rs:175` —
     standalone `CreatorCapabilityStore::write_memory`; no world context in the
     public signature today, so set `world_id: None` unless P0 explicitly extends
     this capability input in the same schema-free internal API.
  3. `crates/nexus-orchestration/src/capability/builtins/creator.rs:797/826/841`
     — integration-test seed paths; set `world_id: Some(...)` in the new
     world-propagation test and `None` in existing tests.
- **DAO filter**: update signatures exactly:

  ```rust
  pub async fn list_fragments_limited(
      pool: &SqlitePool,
      creator_id: &str,
      world_id: Option<&str>,
      limit: i64,
  ) -> Result<Vec<MemoryFragmentRecord>, LocalDbError>;

  pub async fn list_fragments_filtered(
      pool: &SqlitePool,
      creator_id: &str,
      keyword: Option<&str>,
      world_id: Option<&str>,
      limit: u32,
  ) -> Result<Vec<MemoryFragmentRecord>, LocalDbError>;
  ```

  Omitted `world_id` = no world predicate (Creator SOUL whole). Present
  `world_id` = `AND world_id = ?`. Do not implement a public core-only filter in
  V1.81. Dynamic SQL remains allowed here only with the existing `// SAFETY:`
  comment updated to mention optional keyword + optional world filters.
- **Wire DTO**: add `world_id` to `MemoryFragmentInfo`; populate it in
  `handlers/memory.rs::fragments`. Add `world_id` to `ListMemoryFragmentsQuery`
  and pass `params.world_id.as_deref()` into both DAO paths.
- **Codegen + drift**: **owned by P-1** (schema edits + `pnpm run codegen` +
  version bump + drift registration land in Prepare). P0 consumes the generated
  Rust types; P0 still runs `cargo sqlx prepare --workspace --all -- --all-targets`
  for its own migration/DAO query changes and verifies `check-wire-drift.sh`
  stays clean.
- **Tests**: regression — a world-scoped pending review produces a fragment
  carrying that `world_id`; a NULL-world pending review produces a
  Creator-core-only fragment; the DAO world filter returns the correct subset.

### 2.B Creator-SOUL narrative synthesis (new path)

- **Synthesis seam** (P-1 LOCKED §2.4#2): add a sibling trait in
  `crates/nexus-creator-memory/src/review.rs` (or `soul_narrative.rs` re-exported
  from `lib.rs` if P0 splits modules):

  ```rust
  #[allow(async_fn_in_trait)]
  pub trait SoulNarrativeSynthesizer: Send + Sync {
      async fn synthesize(
          &self,
          creator_id: &str,
          input: SoulNarrativeSynthesisInput,
      ) -> Result<SoulNarrativeDraft, MemoryError>;
  }
  ```

  `SoulNarrativeSynthesisInput` is an internal domain struct containing top 30
  keyword counts, 24 recent summaries (280 chars each), up to 8 temporal buckets
  with top 5 keywords each, and total counts/timestamps. `SoulNarrativeDraft`
  contains `narrative: String`. Grep-confirmed current state: the only production
  `SessionDigestSummarizer` impl is `PassthroughSummarizer` in
  `handlers/memory.rs`; real ACP prompt wiring lives in
  `ProductionWorkerProvider` + `CapabilityRegistry` (`acp.prompt`,
  `context.summarize`). Therefore P0 creates a new daemon adapter
  `AcpSoulNarrativeSynthesizer` in `nexus-daemon-runtime`, not in
  `nexus-creator-memory`.
- **Daemon DI path**: in the handler, construct `AcpSoulNarrativeSynthesizer` from
  `WorkspaceState::capability_registry()`. The adapter looks up
  `registry.get("acp.prompt")`, calls `run(json!({ "prompt": prompt,
  "tool_policy": "deny_all", "_creator_id": creator_id,
  "_session_id": "soul_narrative_reflect" }))`, and reads `full_text` from the
  result. Missing registry/capability or `WorkerUnavailable` maps to
  `NexusApiError::ServiceUnavailable` (or existing equivalent); malformed LLM
  output maps to `NexusApiError::Internal`/`BadRequest` as appropriate. The
  endpoint never constructs ad-hoc error JSON.
- **Endpoint**: add to `memory_routes()`:

  ```rust
  .route("/v1/local/memory/soul/reflect", post(handlers::memory::reflect_soul))
  ```

  Request/response shapes must match P-1 §2.1 exactly:
  `SoulNarrativeRequest { creator_id: String, force_regenerate: Option<bool> }`
  → `SoulNarrativeResponse { creator_id, state, narrative?, generated_at?,
  stale, fragment_count_at_generation?, max_fragment_created_at_at_generation?,
  current_fragment_count, current_distinct_keyword_count, min_fragment_count,
  min_distinct_keyword_count }`. The handler enforces active creator equality
  using the same `read_active_creator_id` pattern as `fragments`.
- **Persistence DAO**: add `nexus-local-db` functions for
  `get_soul_narrative(creator_id)`, `upsert_soul_narrative(record)`, and
  `soul_narrative_fragment_stats(creator_id)` returning
  `{ fragment_count: i64, distinct_keyword_count: usize, max_created_at: Option<String> }`.
  Static SQL must use `sqlx::query!` / `query_scalar!`; keyword distinct counting
  may decode JSON arrays in Rust for V1.81.
- **Stale-detection algorithm**:

  ```text
  stats = soul_narrative_fragment_stats(creator_id)
  cached = get_soul_narrative(creator_id)
  insufficient = stats.fragment_count < 10 || stats.distinct_keyword_count < 20

  if cached exists:
      stale = stats.fragment_count != cached.fragment_count_at_generation
           || stats.max_created_at != cached.max_fragment_created_at_at_generation
  else:
      stale = false

  if insufficient:
      return state="insufficient_data", cached narrative omitted, counts + thresholds
  if cached exists && !force_regenerate && !stale:
      return state="current", cached narrative + stale=false + snapshots + counts
  if cached exists && !force_regenerate && stale:
      return state="stale", cached narrative + stale=true + snapshots + counts
  if cached missing && !force_regenerate:
      return state="ungenerated", counts + thresholds

  input = build_capped_signal(stats, fragments)
  draft = synthesizer.synthesize(creator_id, input).await
  validate draft includes specificity + temporality + actionable-tone structure
  upsert cache with generated_at=now, fragment_count_at_generation=stats.fragment_count,
      max_fragment_created_at_at_generation=stats.max_created_at
  return state="current", narrative=draft.narrative, stale=false, snapshots + counts
  ```

  `force_regenerate=true` skips the cached current/stale/ungenerated return after
  the insufficient gate and overwrites the cache.
- **Insufficient-data gate**: hard constants in P0 (`MIN_SOUL_NARRATIVE_FRAGMENTS
  = 10`, `MIN_SOUL_NARRATIVE_DISTINCT_KEYWORDS = 20`) must be returned in the
  response. The gate is evaluated before any ACP call, so new creators never pay
  LLM latency for a guaranteed-thin result.
- **Crate boundaries**: `SoulNarrativeSynthesizer` trait and input/draft structs
  live in `nexus-creator-memory` (domain seam; no daemon dependency). SQLite DAO
  and migration live in `nexus-local-db`. Endpoint + ACP adapter live in
  `nexus-daemon-runtime`, which already depends on `nexus-creator-memory`,
  `nexus-local-db`, and `nexus-orchestration`; no new crate dependency edge is
  expected. If P0 moves the adapter to a new file, it remains inside
  `nexus-daemon-runtime/src/api/handlers/`.
- **Tests**: synthesis trait unit test (mock synthesizer); endpoint integration
  test (generate → persist → stale-after-new-fragment → re-generate); round-trip
  DTO regression extended.
