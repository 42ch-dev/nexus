## 1. Problem Statement

Quality loop requires dedicated presets aligned with novels-system brainstorm/review-master crons, mapped to Nexus FL-E stages without multi-cron staggering.

---

## 2. Goals

1. Embed `novel-brainstorm` preset: open findings → ideation prompts; no direct正文 write.
2. Embed `novel-review-master` preset: surfaces findings + rules for user master decision.
3. Register in preset validation stage mapping where appropriate (research/review analogs).
4. Auto-chain may invoke review stage with `reflection-loop`; brainstorm/master via routing from findings (P1).

---

## 3. Non-goals

- Three-cron staggering (novels-system pattern).
- Platform publish.
- World KB prompt blocks.

---

## 4. Acceptance Criteria

1. Both presets load via `load_embedded_preset` and pass validator.
2. `novel-brainstorm` consumes open findings in prompt context.
3. `novel-review-master` lists findings requiring approval; integrates with P1 API.
4. CLI hints: `creator run brainstorm <work_id>`, `creator run review-master <work_id>` (or stage advance equivalents documented).
