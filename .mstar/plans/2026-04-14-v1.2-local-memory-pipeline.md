## Scope

Complete the local truth pipeline by implementing long-term memory, session-based review, and Stage-0 context assembly:
1. Local long-term memory Markdown lifecycle (file I/O, frontmatter, CRUD)
2. Memory fragment + pending review DB models and lifecycle
3. ACP session-end hooks → pending review capture
4. Review algorithm (deterministic + LLM-assisted via local ACP)
5. Personality push-down (SOUL → local memory)
6. Experience aggregation (LLM-assisted via local ACP, with deterministic fallback)
7. Stage-0 context assembly (SOUL + long-term MD + fragment keywords → final context)
8. CLI commands for memory management, review, and local context assembly

This is the largest V1.2 implementation unit — it ties together the entire local source-of-truth chain.

1. **ACP exists in ALL modes** — `local_only` means "no Nexus platform HTTP", not "no ACP". ACP agents provide LLM capabilities in all modes.

3. **Full session hooks are in scope** — ACP session flow exists in local_only; session-end → pending review capture must be fully implemented.
4. **Review trigger: CLI first** — `nexus42 memory review` command; daemon scheduling deferred.

## Tasks

### Batch 1: Foundation — Long-term Memory MD I/O + DB Schema

**T5.1**: Domain model `LongTermMemory` — struct for parsed long-term memory MD with frontmatter (§3.3 fields: `nexus_memory_version`, `memory_id`, `memory_kind`, `updated_at`, `source_session_ids`) + markdown body.

**T5.2**: Long-term memory file I/O (`crates/nexus-domain/src/memory_io.rs`) — create, read, update, delete for `memory/long-term/<slug>.md`. Include slug validation (path-safe, unique within creator), frontmatter YAML parse/serialize, path traversal protection (reuse `assert_creator_id_safe`).

**T5.3**: DB migration v4 — add `memory_pending_review` table (pending_id PK, session_id, creator_id, world_id optional, task_kind, raw_digest, created_at) and `memory_fragments` table (fragment_id PK, session_id, keywords JSON, summary, created_at, ttl optional). Update `DB_SCHEMA_VERSION` to 4.

**T5.4**: DB CRUD modules — `pending_review.rs` and `memory_fragment.rs` in `crates/nexus-local-db/src/`.

### Batch 2: Session Lifecycle Hooks

**T5.5**: Session-end hook in `AcpSdkAdapter` — when a session ends (graceful or forced), serialize session context (session_id, creator_id, world_id optional, task_kind heuristic, raw_digest from message history) into a `PendingReviewRecord` and persist to `memory_pending_review` table via daemon API or direct DB write. Add `on_session_end` callback to the client trait.

**T5.6**: Session context capture — extract structured context from completed ACP session (tool call summary, output file paths, user/assistant message count) for the `raw_digest` field. This is the "collect serializable context" step from §6.2.

### Batch 3: Review + Promotion

**T5.7**: Review algorithm (`crates/nexus-domain/src/review.rs`) — implement the decision logic from §7.2: for each `PendingReviewRecord`, classify as `drop` | `fragment_only` | `promote_to_long_term` | `merge_into_existing` | `trigger_soul_experience_only`. Use deterministic heuristics (message count, task_kind, digest length) for initial classification.

**T5.8**: Fragment creation — when review decides `fragment_only`, create a `MemoryFragment` entry (keywords extracted from raw_digest, summary) and persist to DB.

**T5.9**: Long-term memory promotion — when review decides `promote_to_long_term`, generate a new long-term memory MD file (via ACP LLM-assisted summarization) and persist to `memory/long-term/<slug>.md`. Use ACP client to call local agent for summarization.

**T5.10**: Idempotency — same `session_id` must not produce duplicate long-term memories (content_hash dedup).

### Batch 4: SOUL Integration

**T5.11**: Personality push-down — when SOUL `## Personality` is saved, extract structured fragments and upsert local personality memory MD (`memory/long-term/personality-core.md` or similar). Hook into existing `soul edit-personality` command.

**T5.12**: Experience aggregation (LLM-assisted) — scan `memory/long-term/*.md` for experience-kind entries, call local ACP agent to generate aggregated experience text, replace `## Experience` section in SOUL.md. Fallback to deterministic concat if ACP unavailable.

### Batch 5: Stage-0 Context Assembly

**T5.13**: Context assembly module (`crates/nexus-domain/src/context_assembly.rs`) — implement Stage-0 merge from §9.2 ordering:
  1. System/policy prefix
  2. `## Personality` (from SOUL.md)
  3. Long-term memory MD (sorted by kind, recency)
  4. `### Fragment keywords (deduped)` (keywords union from all fragments)
  5. `## Experience` (aggregated)
  6. User task prompt

**T5.14**: Token budget / truncation — basic token estimation (chars / 4 heuristic) with configurable max. If over budget, truncate long-term MD entries from oldest first. Never truncate Personality.

### Batch 6: CLI + Integration

**T5.15**: Memory CLI commands (`crates/nexus42/src/commands/memory.rs`):
  - `nexus42 memory list` — list long-term memories for current creator
  - `nexus42 memory create [--kind <kind>] [--session <id>]` — create new memory MD
  - `nexus42 memory show <slug>` — display memory content
  - `nexus42 memory edit <slug>` — edit memory in editor
  - `nexus42 memory delete <slug>` — delete memory
  - `nexus42 memory review` — trigger review of pending queue
  - `nexus42 memory fragments` — list memory fragments
  - `nexus42 context assemble-local` — run Stage-0 assembly and output result

**T5.16**: CLI integration — register new commands in main.rs, add `memory` subcommand group.

**T5.17**: End-to-end tests — scenario: init SOUL → create memories → end ACP session → review → promote → aggregate experience → assemble context. Verify §9.2 ordering.
