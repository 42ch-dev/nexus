## 1. Problem Statement

**DF-18**: `NativeSession` exists in [`claude.rs`](../../crates/nexus-agent-host/src/providers/native_cli/claude.rs) but `ClaudeCliProvider::execute()` spawns a new process per operation. Multi-turn native CLI conversations cannot reuse context, blocking a basic local authoring + agent workflow.

---

## 2. Goals

1. Reuse a **persistent child process** per `HostSessionId` for native CLI provider across multiple `execute` calls within session lifetime.
2. Define lifecycle: create → execute (N) → shutdown (coordinate with DF-24 provider shutdown).
3. Tests proving two consecutive executes on the same session share state (stdout marker, session file, or mock — choose minimal reliable signal in implementation).
4. Mark **DF-18** Shipped in deferred tracker when Done.

---

## 3. Non-goals

- Full ACP provider multi-turn (separate from native CLI path unless shared abstraction falls out naturally).
- DF-19–28 except coordination with DF-24 shutdown from sibling plan.
- Replacing Claude CLI with other native providers.
