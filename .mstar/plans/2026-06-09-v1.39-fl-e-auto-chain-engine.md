## 1. Problem Statement

V1.38 requires manual `creator run continue` and `stage advance` between chapters and FL-E stages. Users expect: daemon online → chain runs; daemon restart → resume from checkpoint. DF-53 and DF-68 are open.

---

## 2. Goals

1. Implement on_complete / stage-driver hook to auto-advance `intake → research → produce → review → persist` when `--auto-chain` enabled (default true).
2. Chapter outer loop: after persist for chapter N, auto-enqueue produce for chapter N+1 via `next_chapter(work_id)` until Work complete.
3. Persist Work continuation checkpoint (`stage`, `chapter`, `driver_schedule_id`, `auto_chain_interrupted`).
4. Boot: conditional auto-resume of checkpointed driver schedules only (replace blanket pause-for-all for auto-chain drivers).
5. Side-input lane: `--note` / patch enriches context without forking or canceling driver; status UX for daemon/chain/pending inspiration.
6. CLI: `--no-auto-chain` opt-out; `creator run resume <work_id>` for manual recovery.

---

## 3. Non-goals

- Research KB consumption wiring (P0.5).
- Findings table, novel-brainstorm/review-master presets (P1/P2).
- Rules/logs/96h (P3/P4).
- World KB (DF-63), multi-volume PK, platform publish.

---

## 4. Acceptance Criteria

1. `creator run start` with default flags auto-chains through all stages for chapter 1 without manual `stage advance`.
2. After chapter N persist completes, produce for N+1 enqueues automatically when chapters remain.
3. Work completion stops all further auto-enqueue.
4. Daemon restart auto-resumes checkpointed auto-chain driver; non-checkpoint schedules remain paused.
5. `creator run continue --note` during active chain does not create second driver schedule; note visible in status.
6. `--no-auto-chain` disables automatic enqueue; checkpoint still written.
