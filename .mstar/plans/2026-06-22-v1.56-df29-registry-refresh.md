## Goal

Close DF-29 by implementing `registry.refresh` as a configurable capability: synthetic output by default (generated from an embedded registry snapshot), optional `--cdn-url <url>` daemon start flag for real network fetch with timeout/retry, and sandbox/air-gap compatibility so the capability works offline with no network dependency.

## Scope In

- Synthetic output generator: build a default response from an embedded registry snapshot (canned data shipped with the binary) for `registry.refresh` calls.
- `--cdn-url <url>` daemon start option: when provided, `registry.refresh` fetches from the CDN with configurable timeout (default e.g. 10s) and retry (default e.g. 3 attempts with exponential backoff).
- Sandbox/air-gap compatibility: default behaviour (no `--cdn-url`) returns synthetic output immediately; no network attempts when `--cdn-url` is omitted.
- Register `registry.refresh` as a new ACP capability ID in `capability-registry.md` and the runtime capability set.
- Tests: synthetic output determinism, `--cdn-url` fetch+retry+timeout, sandbox no-network path, and capability registration.
- Amend `acp-capability-set.md` and `cli-spec.md` for the new capability and `--cdn-url` flag.

## Scope Out

- Registry publish / signing / third-party registry (BL-03).
- Full DF-46 capability parity — this is one new capability, not the full set.
- DF-56 registry-driven conditional edges — those are P3, which consumes this capability's output.
- CDN infrastructure provisioning — only the client-side fetch logic is in scope.
- Real-time registry push or subscription model.

## Risks

| Risk | Mitigation |
| --- | --- |
| Synthetic snapshot goes stale between releases | Embedded snapshot is updated per release; `--cdn-url` provides real-time path for users who need freshness. |
| Network timeout/retry blocks daemon startup | `--cdn-url` fetch is lazy (first `registry.refresh` call triggers fetch); startup is not blocked. |
| Capability ID collides with existing IDs | Follow existing capability registration discipline; check against `capability-registry.md` before registering. |
| Embedded snapshot inflates binary size | Snapshot is minimal (capability IDs + version info only, not full registry metadata); measured and documented. |
