## 1. Problem Statement

DF-56 conditional routing was deferred since V1.33. V1.42 delivers a **minimal slice**: `llm_judge` stage evaluates GO/NOGO and selects one of two `next` edges. Full stage-level branching, `registry.refresh` linkage, and multi-branch graphs remain post-V1.42 (tracker §3.6 / DF-56 Deferral history).

---

## 2. Goals

1. Preset graph loader accepts conditional `next` on `llm_judge` nodes (GO branch / NOGO branch).
2. Minimal evaluator invokes existing `judge.llm` worker path; parses GO/NOGO.
3. Schedule executor follows selected edge.
4. Hermetic unit + integration tests (synthetic preset fixture).
5. Document **full DF-56 roadmap** in deferred tracker (not in this plan's AC).

---

## 3. Non-goals

- Arbitrary expression routing on all stage types
- `registry.refresh` conditional edges
- Multi-volume changes (P1 must merge first)

---

## 4. Acceptance Criteria

1. Embedded or test preset with `llm_judge` → two `next` targets; GO path taken when worker returns GO.
2. NOGO path taken on NOGO or worker unavailable per existing judge semantics.
3. Spec promoted from Exploration to Draft V1.42 with shipped minimal slice noted.
4. Tracker DF-56 row updated with Post-V1.42 scope in Deferral history.
