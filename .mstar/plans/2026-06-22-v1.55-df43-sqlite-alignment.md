## Goal

Close DF-43 by aligning SQLite persistence ownership with crate models: `nexus-local-db` remains the production persistence owner, while `nexus-knowledge` exposes durable model/adaptor seams without becoming a second database truth source.

## Scope In

- Audit current reference-source persistence and model boundaries.
- Define and implement the minimal adapter seam required for `nexus-knowledge::ReferenceSource` ↔ local DB persistence.
- Update specs if the ownership boundary changes.
- Add tests that prevent duplicate persistence truth or drift between crates.

## Scope Out

- Full Local API redesign (DF-42).
- Network refresh pipeline for DF-44.
- Third-party registry/publish flows.
- New public wire schemas unless a PM-approved blocker proves they are required.
