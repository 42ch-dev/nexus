## 5. Technical Design

### 5.1 Game-Bible Directory Layout

```text
<workspace>/
  Works/
    <work_ref>/
      README.md                  # Core pillars, genre, tone, target audience
      Design/
        overview.md              # North star section (required for completion)
        pillars.md               # Design pillars, constraints
        characters.md            # Character roles, archetypes
        factions.md              # Factions, politics, alignment
        species.md               # Sapient species, traits
        locations.md             # World geography, levels
        mechanics.md             # Core mechanics / gameplay loops
        magic_system.md          # Magic / superpower system rules
        technology.md            # Tech level, tools, artifacts
        economy.md               # Currency, trade, resources
        progression.md           # Leveling, skill trees, unlocks
        lore.md                  # History, mythology, cosmology
      Logs/
        design/
        review/
```

Each `Design/*.md` carries frontmatter: `section_status: draft|reviewed|accepted`, `section_weight: critical|important|nice_to_have`.

**Not scaffolded**: `Stories/`, `Outlines/`, `work_chapters` rows, `volume-outline.md`, `foreshadowing.md`, `event-index.md` â€” these are novel-specific.

### 5.2 Profile Registration

Game-bible uses `work_profile = "game_bible"` (TEXT column, no enum migration needed). Registration:
- `nexus42 creator bootstrap --profile game-bible` auto-selects `--init-preset game-bible-init`
- No auto-chain after init (unlike novel's `--auto-chain`)
- No primary production preset in V1.54; `design-writing` stub only

### 5.3 KB Taxonomy â€” New BlockType Variants

Add 7 new variants to `schemas/common/common.schema.json` `BlockType` enum:

| Wire name | UI label | `game_bible_category` | Design section |
|-----------|----------|----------------------|----------------|
| `species` | Species | `species` | `species.md` |
| `faction` | Faction | `faction` | `factions.md` |
| `magic_system` | Magic System | `magic_system` | `magic_system.md` |
| `technology` | Technology | `technology` | `technology.md` |
| `deity` | Deity | `deity` | `lore.md` |
| `level` | Level | `level` | `locations.md` |
| `economy_tier` | Economy Tier | `economy_tier` | `economy.md` |

Schema change requires `pnpm run codegen` â†’ regenerated `crates/nexus-contracts/src/generated/common_types.rs`.

**ValidationMode::GameBible**: new variant in `nexus-kb::validation`, mapping `game_bible_category` â†’ `BlockType`. Reuses existing `character`, `organization`, `scene`, `item`, `conflict` variants for cross-domain concepts.

### 5.4 Profile Differences from Novel and Essay

| Dimension | Novel | Essay | Game-Bible (V1.54) |
|-----------|-------|-------|-------------------|
| Init preset | `novel-project-init` | `essay-init` | `game-bible-init` |
| Auto-chain | Yes | No | No |
| Uses `work_chapters`? | Yes | No | No |
| Content layout | `Stories/` + `Outlines/` | `Drafts/` + `Outlines/` | `Design/` (12 sections) |
| World binding | Required | Optional | Optional (highly recommended) |
| KB ValidationMode | `Novel` | `Generic` fallthrough | `GameBible` (new) |
| Category field | `novel_category` | None | `game_bible_category` |
| Completion criteria | All chapters finalized | Draft status == finalized | All critical sections accepted |

### 5.5 Profile Gate Strategy

The key risk is profile abstraction leakage: novel-specific code paths must not execute for game-bible Works.

**Gate sites** (must add `if work_profile != "novel"` guards):

| File | Path | Gate |
|------|------|------|
| `nexus-local-db/src/works.rs` | `complete_work_if_done` | Early return `Ok(false)` for non-novel |
| `nexus-local-db/src/work_chapters.rs` | All functions | Non-novel â†’ clear error or no-op |
| `nexus42/src/commands/creator/works/mod.rs` | `handle_status` | Chapter table display gated to novel |
| `nexus-orchestration/` | Preset gate evaluation | Novel presets skip game-bible works |

Each gate site gets a `// SAFETY: game-bible profile gate` comment and a unit test asserting game-bible Works are not affected.

## 9. Non-Goals

- **No run-loop presets**: `design-writing`, `character-writing`, etc. are stub-only. V1.55+ gets full LLM-driven production presets.
- **No chapter model**: game-bible does not use `work_chapters` table, `Stories/`, or `Outlines/`.
- **No completion detection**: section status model is defined but not evaluated by daemon (V1.55+).
- **No script profile**: only game-bible in V1.54.
- **No KB extraction integration**: game-bible KB entries are author-curated; LLM extraction from design docs is V1.55+.
- **No auto-chain**: no automatic stage sequencing for game-bible.
