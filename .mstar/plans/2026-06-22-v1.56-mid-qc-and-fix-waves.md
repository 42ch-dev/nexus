## Goal

Make the V1.56 3-wave "merge + tri-review + mid-QA + fix-wave" rhythm explicit and auditable for P0, P1, P2, and P3 across Track A and Track B.

## Scope In

- Track each implement plan's merge to `iteration/v1.56`.
- Ensure QC tri-review and mid-QA happen on the integrated branch checkpoint.
- Track targeted fix-waves and re-review decisions.
- Prevent the next wave from starting before the prior wave's checkpoint is accepted or PM-overridden.
- Enforce the sequential dependency: Wave 2 starts only after Wave 1 (P0+P1) accepted; Wave 3 starts only after Wave 2 (P2) accepted.

## Scope Out

- No product code.
- No replacement for actual QC reports under `.mstar/plans/reports/<plan-id>/`.
- No QA execution by this plan; it records routing and evidence only.

## Acceptance Criteria

- [ ] P0, P1, P2, and P3 each have a recorded integrated checkpoint before formal review.
- [ ] Each checkpoint has three QC reports and one mid-QA report, or a PM-recorded exception.
- [ ] Mid-QC runs in **3 waves** (Wave 1: P0 || P1 parallel; Wave 2: P2 single; Wave 3: P3 single), not four sequential reviews.
- [ ] Wave 2 begins only after both Wave 1 plans are accepted (or PM override recorded).
- [ ] Wave 3 begins only after Wave 2 plan is accepted (or PM override recorded).
- [ ] Fix-waves are targeted and tied to specific findings; no piggyback refactors.
- [ ] P-last does not start until all four implement checkpoints are accepted or explicitly overridden by PM.
- [ ] Review ranges for each checkpoint are reproducible and recorded in assignments/reports.

## QC Strategy

This meta-plan is not itself tri-reviewed. It exists to ensure the four implement plans receive tri-review + mid-QA at integrated checkpoints. P-last QA verifies that the evidence exists.

## Risks

| Risk | Mitigation |
| --- | --- |
| Review overload from 12 QC reviews + 4 mid-QA | Keep scopes per checkpoint narrow; use targeted re-review for fixes; PM spreads reviews across plan timelines. |
| QC reviews wrong branch | Assignments must use `iteration/v1.56` and identical review range/diff basis per checkpoint. |
| Fix-wave drift from findings | Every fix maps to a specific finding; unrelated work becomes a new residual or plan. |
| Wave 2/3 delays cascade | P-mid tracks per-wave gate states; PM can record override for non-blocking findings to unblock next wave. |
