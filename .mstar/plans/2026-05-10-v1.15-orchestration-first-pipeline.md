**Goal:** Deliver V1.15 for the public `nexus` repository by moving writing/research/publish workflows from legacy CLI verbs to orchestration-first presets, embedded skills, ACP session bootstrapping, and the `novel-writing` workspace-to-platform sync story.

**Architecture:** Batch the release as B1 → B2 → B3. B1 establishes embedded skills and ACP session injection, B2 removes obsolete command groups after preset entrypoints exist, and B3 completes novel-writing filesystem policy plus sync contract/tests.

**Tech Stack:** Rust workspace crates (`nexus42`, `nexus42d`, `nexus-orchestration`, `nexus-acp-host`, `nexus-home-layout`, `nexus-local-db`, `nexus-contracts`), JSON Schema wire contracts as SSOT, cargo fmt/clippy/test, GitNexus impact/change analysis.

## Risk Summary

| Rank | Risk | Mitigation |
|---|---|---|
| 1 | Skill sync overwrites user files. | Use content hashes or `.new` suffix policy, emit explicit operator report, and test conflict paths. |
| 2 | Symlink portability across OS/permissions. | Provide a fallback copy policy behind preset/config feature selection; test symlink target validation. |
| 3 | Sync module drifts from platform API. | Keep JSON Schema/contracts as SSOT, add contract tests, and enforce platform companion gate before Done. |

Additional plan-level risk: command removal may break undocumented local workflows. V1.15 is pre-release, so no compatibility layer is required, but help text and preset entrypoints must make the replacement path explicit.
