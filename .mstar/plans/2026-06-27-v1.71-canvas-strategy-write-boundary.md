## Scope (β depth — `wire_contracts_changed: TRUE`)

- **A1 — Schemas + codegen**: New Local API DTOs in `schemas/local-api/canvas/strategy/`:
  - `StrategyPatchStateRequest` / `StrategyPatchTransitionRequest` / `StrategyPatchPromptTemplateRequest`
  - `StrategyPatchResponse`
  - `StrategyConflictError` (extends existing error envelope)
  - `StrategyValidationError`
  - Generate TS (`apps/web/src/api-types/`) and Rust (`crates/nexus-contracts/src/generated/`); commit regenerated output.
- **A2 — graphRevision tracking + persistence**: Add revision tracking to Strategy/preset (storage decision confirmed by `@architect` Phase 2b: default option (a) `revision:` key in preset YAML header; option (b) `strategy_revisions` DB table if architect selects). On every patch, increment revision atomically; read endpoints return current revision; reject stale `base_revision` with `StrategyConflictError`.
- **A3 — Local API routes**: New `POST` routes following V1.64 conventions:
  - `/v1/local/strategies/{strategy_id}/states/{state_id}/patch`
  - `/v1/local/strategies/{strategy_id}/transitions/patch`
  - `/v1/local/strategies/{strategy_id}/states/{state_id}/prompt/patch`
- **A4 — Daemon validation + atomic persistence** (`crates/nexus-daemon-runtime/src/handlers/strategy.rs`):
  - Validate ids, reachability, condition syntax, prompt template refs
  - Atomic YAML write (temp + rename + fsync) + revision update in same DB tx
  - Return structured conflict error on stale `base_revision`
- **A5 — NexusClient methods**: `strategyPatchState`, `strategyPatchTransition`, `strategyPatchPromptTemplate`; track `lastKnownRevision`; refetch canonical preset after patch.
- **A6 — Canvas UI write affordances**:
  - State inspector: edit label/description
  - Edge inspector: edit condition/target
  - In-node TipTap for prompt template editing
  - Conflict modal UX: structured error → View current / Reapply / Merge / Discard
- **A7 — Tauri WKWebView parity smoke**: write affordances + conflict modal + in-node TipTap save work in Tauri shell.
- **A8 — Accessibility**: conflict modal ARIA live region, keyboard reapply/merge/discard, reduced-motion support.
- **A9 — Tests**: unit (validation + revision + conflict), integration (3 routes), e2e (canvas edit → save → refetch), regression (concurrent daemon write causes 409).
