## 1. Problem Statement

`creator memory pending-*` works, but `creator memory review` calls `POST /v1/local/memory/review` which is **not registered** on the daemon. `GET /v1/local/memory/fragments` is similarly missing, breaking assemble-moment keyword collection and fragment UX.

---

## 2. Goals

### A1 — `POST /v1/local/memory/review`

1. Handler calls `promote_to_long_term` (or thin wrapper) with creator scoping.
2. Request/response shapes aligned with CLI expectations.
3. Auth: active creator from request context.

### A2 — `GET /v1/local/memory/fragments`

1. List/query `memory_fragments` for creator (and optional filters).
2. Response field names match CLI parser (`fragments` vs `pending_reviews`).

### A3 — CLI alignment

1. `creator memory review` succeeds against running daemon in integration test.
2. `platform context assemble-moment` fragment keyword path no longer 404s (or degrades gracefully with log only if empty).

### A4 — Tests

1. Daemon handler unit tests.
2. CLI integration test with test daemon fixture.
