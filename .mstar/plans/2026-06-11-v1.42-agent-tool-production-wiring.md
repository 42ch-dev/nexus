## 1. Problem Statement

V1.34 shipped the adapter; V1.35–V1.41 deferred **production caller wiring** (DF-47). Agents in real schedules do not yet invoke daemon `nexus.*` tools end-to-end through `worker/agent_tool_request`.

---

## 2. Goals

1. Wire **one** production path: real schedule tick → worker upcall → `HostToolExecutor` → handler for a single chosen tool (e.g. `nexus.work.status` or minimal read-only tool per spec §5.4 grill-me).
2. Hermetic E2E test with mock worker confirming request/response round-trip.
3. Update spec § Production caller wiring with shipped tool name + constraints.
4. Close or narrow DF-47 row (full DF-46 parity remains out).

---

## 3. Non-goals

- Full `nexus.*` capability matrix (DF-46)
- CLI subprocess bridge (DF-48 rejected)
- MCP server (DF-49)

---

## 4. Acceptance Criteria

1. One tool callable from a running schedule without manual CLI invocation.
2. Authz/locks: read-only tool respects completion-lock; mutating tools follow runtime_lock (P0).
3. Test proves E2E path in CI hermetic mode.
