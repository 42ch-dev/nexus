## 1. Problem Statement

`RunCommand` is a ~2000-line clap enum. Each new author preset requires Rust changes. V1.45 needs a registry-driven runner.

---

## 2. Goals

1. Replace enum dispatch with `creator run <preset_id> [<work_id>]`.
2. Resolve preset from embedded + user + system dirs (**no CLI whitelist**).
3. Optional positional `work_id` → pool `active` (same as `works status`).
4. Add **`preset.yaml` `cli_args`** minimal schema; ship fields for audit + review-master presets.
5. For FL-E default presets (`research`, `novel-writing`, `reflection-loop`, `kb-extract`), preserve stage-advance semantics (gate check + Work stage PATCH + schedule enqueue) inside runner.
6. Global flags: `--json`, `--force-gates --reason` only (no `--force` from stage advance).

---

## 3. Non-goals

- Deleting old subcommands (P1/P2 — same merge).
- `creator bootstrap` (P2).
- Quickstart rewrite (P3).

---

## 4. Acceptance Criteria

1. `nexus42 creator run novel-review-master [<work_id>] [--finding-id ID] [--auto-schedule]` enqueues schedule.
2. `nexus42 creator run novel-manuscript-audit-review [<work_id>] --chapter N` works.
3. `nexus42 creator run research [<work_id>]` advances FL-E research stage.
4. Unknown preset id fails with actionable error from orchestration.
5. Hermetic tests for runner + cli_args parsing.
