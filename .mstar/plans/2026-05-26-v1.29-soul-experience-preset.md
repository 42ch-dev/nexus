## 1. Problem Statement

`creator soul` can init/edit SOUL and push personality to memory, but **`## Experience` is not automatically aggregated** from long-term memories. Authors lack a scheduled or preset-driven refresh path aligned with platform creator-memory lifecycle specs.

Grill decision: **Preset + Schedule** for ongoing aggregation; **CLI `refresh-experience`** as deterministic one-shot (no LLM in CLI).

---

## 2. Goals

### A2.1 — Deterministic CLI entry

1. Add `SoulCommand::RefreshExperience` → `nexus42 creator soul refresh-experience`.
2. Behavior: merge/summarize from long-term memory files into `## Experience` using **deterministic rules** (template + ordering); no provider/LLM call from CLI.
3. Preserve normative warning: hand-edits under `## Experience` may be overwritten on next refresh.

### A2.2 — Orchestration capability + preset

1. Register capability e.g. `soul.experience.aggregate` (name frozen in implementation PR; document in orchestration spec).
2. Add embedded or `_system` preset fragment (or extend existing maintenance preset) that:
   - Reads long-term memory via creator-memory APIs
   - Optionally calls LLM via **`acp_prompt`** only inside capability (if LLM polish needed — prefer deterministic first; LLM optional behind preset flag)
3. Wire schedule trigger documentation: `daemon schedule` can run preset on cron (reuse V1.5 schedule infrastructure).

### A2.3 — Spec touch

1. Update [creator-schedule-and-core-context.md](../knowledge/specs/creator-schedule-and-core-context.md) § Experience aggregation (surgical paragraph).
