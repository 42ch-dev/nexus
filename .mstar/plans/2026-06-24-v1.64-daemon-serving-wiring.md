# P3: Daemon Web UI Serving Wiring

Backend bridge: serve the built `apps/web` SPA from the `nexus42` daemon. Release = embed via `rust-embed`; the daemon gains a static-asset route. Small CLI convenience command. Pure Rust (daemon + CLI); no frontend code.

## Tasks

- [ ] T1: **Embed strategy** (compass §5 item #3) — add `rust-embed` dep to `nexus-daemon-runtime` (or `nexus42`); embed `../../apps/web/dist/**` at compile time. Macro rebuilds on dist change. Fallback decision: if embedding creates release-pipeline friction, document serve-from-disk (`~/.nexus42/web/`) as the alternative — but default to embed (single-binary story).
- [ ] T2: **`tower-http::ServeDir` route** — wire into the daemon's top-level router (`crates/nexus-daemon-runtime/src/api/mod.rs`): SPA fallback serving embedded assets (index.html fallback for client-side routes). Unauthenticated route (the UI handles auth UX; the Local API endpoints behind it remain keyless on loopback — V1.20 model; the `require_api_key` middleware only gates in keyed-all / non-loopback mode).
- [ ] T3: **Release build pipeline** — `apps/web` dist build before `cargo build --release` of `nexus42`; document the release sequence (or wire a build.rs check). Ensure `rust-embed` assets are present in CI release artifacts.
- [ ] T4: **CLI entry** (compass §5 item #4) — default: `nexus42 daemon start` logs the UI URL (`http://localhost:<port>/`). Optional low-cost: add `nexus42 ui` (or `nexus42 web`) convenience command that starts the daemon (if not running) + opens the OS browser (`open`/`xdg-open`/`start`). PM decides final shape.
- [ ] T5: `cli-spec.md` amend — Web UI access section (PM/architect co-own the spec text; P3 implementer supplies the empirical surface).
- [ ] T6: `daemon-runtime.md` amend — static-asset serving section (ServeDir + rust-embed).
- [ ] T7: Verify — `cargo build --release -p nexus42` produces a binary that serves the UI at `http://localhost:<port>/`; client-side routes deep-link; Local API endpoints remain keyless on loopback (V1.20 model); daemon logs UI URL on start.
- [ ] T8: `cargo clippy -p nexus-daemon-runtime -p nexus42 -- -D warnings` + `cargo test -p nexus-daemon-runtime -p nexus42`.

## Done when

- `nexus42 daemon start` serves the built SPA at `http://localhost:<port>/` from the embedded assets (release binary, zero external files).
- Client-side routes deep-link correctly (SPA fallback).
- Local API endpoints remain keyless on loopback (V1.20 model); UI static route is unauthenticated (auth UX handled in-browser).
- CLI logs the UI URL; optional `nexus42 ui` convenience command works (if added).
- `daemon-runtime.md` + `cli-spec.md` reflect the serving model.
