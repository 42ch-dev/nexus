# P3: Local API DTO Promotion — F1+F2+F3 Orchestration/Preset Schemas + Surface Audit

Track C depth: promote ~20 additional Local API DTOs for orchestration sessions READ (F1), schedules + capabilities READ (F2), and preset_management full surface (F3). Follow the per-resource subfolder + re-export pattern established by P1. Conduct a Local API surface audit (pagination / error-shape / filter consistency) across all promoted resources; produce audit report; fix-waves for high-value inconsistencies.

## Tasks

- [ ] T1: Promote F1 orchestration sessions READ DTOs → `schemas/local-api/orchestration/sessions/` (list-response, detail-response, status enums). READ only — no create/signal/cancel write-side DTOs.
- [ ] T2: Promote F2 DTOs → `schemas/local-api/orchestration/{schedules,capabilities}/` (list-response shapes for schedule list + capability registry).
- [ ] T3: Promote F3 preset_management full surface DTOs → `schemas/local-api/preset-management/` (list/show/create/update/delete/validate request + response shapes). This is a WRITE surface but preset configs are declarative YAML/JSON (stable shape).

- [ ] T5: **Surface audit**: review all promoted resources (P1 F0 + P3 F1-F3) for:

  - Error-shape consistency (error envelope, status codes, stable error codes).
  - Filter/sort parameter consistency (naming, defaults, validation).
  - Produce audit report at `.mstar/plans/reports/2026-06-24-v1.63-local-api-orchestration-and-preset-dtos/surface-audit.md`.
- [ ] T6: Fix-waves for high-value inconsistencies surfaced by audit (low-risk, in-scope fixes). Defer structural changes as residuals with V1.64+ target (§5 item #5).
- [ ] T7: Boundary docs: update `schemas-directory-layout.md` inventory + `schemas/README.md` for the new subtrees.
