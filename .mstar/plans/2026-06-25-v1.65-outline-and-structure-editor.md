## Overview

The V1.65 lead slice. Extend the V1.64 `apps/web` SPA from control-room (read + setup) to an **authoring entry surface**: authors plan, review, and restructure chapter outlines and structure in the UI, with body rendered read-only. Consumes the P0 chapter-content Local API; builds on the P1 test baseline; consumes the DESIGN.md increment (editor/table/context-menu tokens).

Architecture stays Tauri-ready: all new screens route through `NexusClient` → `BrowserClient` (no browser-only APIs in editor core); V1.66 `TauriClient` swaps transparently.

## Acceptance Criteria

- Author can: switch Work → see chapter structure → edit a chapter outline (markdown round-trip) → save → change structure (slug/wc/volume; `title` displayed read-only) → advance status `not_started→outlined` → read body read-only → copy body path.
- Soft-concurrency warning renders when editing outline of `draft`/`finalized` chapter.
- `finalized`/`published` structural edits require confirmation; deletion hard-blocked.
- No browser-only API leaks in editor core (architect audit); `NexusClient` boundary intact.
- DESIGN.md increment tokens consumed (light + dark).
- `pnpm --filter web test` + `build` + `typecheck` clean; CI `web-build` + `web-test` green.
