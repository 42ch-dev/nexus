## 1. Problem Statement

Per-Work cron config is not implemented. `works.schedule_json` column does not exist. `creator works cron` subcommands do not exist. Authors have no way to set staggered schedules for brainstorm/write/review.

---

## 2. Goals

1. Add `works.schedule_json TEXT` column (nullable; empty = defaults).
2. DAO: `set_schedule_json(work_id, json)` / `get_schedule_json(work_id)`.
3. CLI: `creator works cron set/show/list` per spec §3.
4. Default schedule table constant (`DEFAULT_SCHEDULE` in `cron.rs`).
5. Hermetic tests for migration (forward + back-compat) + DAO + CLI smoke.

---

## 3. Non-goals

- Cron firing into auto-chain (T-A P1).
- Review cron (T-A P2).
- Auto-chronology (T-A P3).

---

## 4. Acceptance Criteria

1. Migration `202606180001_works_schedule_json.sql` adds the column; existing Works get NULL (= defaults).
2. `cargo test -p nexus-local-db --test works_schedule_migration` passes (forward + rollback).
3. `cargo test -p nexus42 --test cron_cli` covers: set with flags, show with TZ display, list across workspace.
4. `creator works cron set my-work --brainstorm "0 3,9,15,21 * * *" --tz Asia/Shanghai` round-trips through DAO and re-renders correctly on `show`.
5. Invalid cron expression returns `ValidationError` with stable error code; invalid TZ returns `ValidationError`.
