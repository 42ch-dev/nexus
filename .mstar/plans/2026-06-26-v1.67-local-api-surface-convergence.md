- AC1: All orchestration list/detail handlers return the canonical `ErrorResponse` envelope (no ad-hoc tuples). Verified by handler-level tests.
- AC2: All emitted `ErrorResponse.code` strings match the casing convention. Convention documented + asserted.
- AC3: All list endpoints emit `{ items: [...], pagination: {...} }` (per-resource array keys removed). `pnpm run codegen` regenerates TS + Rust; `check-wire-drift` clean.
- AC4: List endpoints accept the documented `?sort=<grammar>` (additive; server-side sort where cheap, deferred-with-doc where expensive).
- AC5: `apps/web` consumers adapted to `.items`; `@42ch/nexus-contracts` version bumped per `@architect` Phase 2b verdict.
- AC6: `cargo test --all` + `cargo clippy --all -- -D warnings` + `cargo +nightly fmt --all --check` + `pnpm --filter web typecheck/build/test` clean.
- AC7: Regression tests cover error-envelope mapping, casing, `items` shape, sort.
