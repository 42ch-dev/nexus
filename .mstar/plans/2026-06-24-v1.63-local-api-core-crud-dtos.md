# P1: Local API DTO Promotion — F0 Core CRUD Schemas + Codegen + Drift Detection

Track C foundation: promote ~35 Local API DTOs for the 6 core CRUD resources (works, kb, findings, schedule, workspace, creators) from hand-written Rust (`crates/nexus-contracts/src/local/`) to cross-language JSON Schema (`schemas/local-api/<resource>/`). Regenerate codegen. Establish the per-resource subfolder pattern + re-export migration strategy that P3 will follow for orchestration/preset DTOs.

## Tasks

- [ ] T1: Survey existing DTOs in `crates/nexus-contracts/src/local/` for the 6 core resources (request/response/response-list/error shapes). Produce an inventory mapping (local Rust path → target schema path).
- [ ] T2: Create `schemas/local-api/{works,kb,findings,schedule,workspace,creators}/` subtrees; author JSON Schema files for each DTO (per-operation files: e.g. `<resource>-list-request.schema.json`, `<resource>-list-response.schema.json`, `<resource>-create-request.schema.json`, etc.). Resolve §5 item #4 (naming convention).
- [ ] T3: Update `$id`/`$ref` URIs; ensure `common/` shared types are referenced via relative `$ref`.
- [ ] T4: Update `tooling/codegen/src/schema-loader.ts` to register new `local-api/<resource>/` paths.
- [ ] T5: `pnpm run codegen` → regenerate Rust (`generated::local_api::{works,kb,...}`) + TypeScript (`packages/nexus-contracts/src/generated/local-api/`).
- [ ] T6: Migration strategy (§5 item #6): re-export promoted types from `generated::local_api::*` at the existing `local::*` import path (zero consumer call-site churn) OR update import paths. Implement chosen strategy.
- [ ] T7: Update `crates/nexus-contracts/tests/schema_drift_detection.rs` — add `entry!` rows for all new schemas (`CheckMode::Strict`).
- [ ] T8: `./tooling/check-wire-drift.sh` + `cargo test --test schema_drift_detection` → fix drift.
- [ ] T9: `cargo check -p nexus-contracts` + `cargo clippy -p nexus-contracts -- -D warnings` + `cargo test -p nexus-contracts`.
- [ ] T10: Boundary docs update: `schemas-directory-layout.md` (add new subtrees to normative tree + inventory), `schemas/README.md` (layout/product-lines tables), `schemas/AGENTS.md` (codegen note), per-folder READMEs.

## Done when

- ~35 new schemas under `schemas/local-api/{works,kb,findings,schedule,workspace,creators}/` pass `pnpm run validate-schemas`.
- Codegen produces matching Rust + TS types; no drift.
- All consumers compile without manual type duplication.
- `schema_drift_detection` Strict mode passes for all new schemas.
- Boundary docs reflect the new tree.
