# P2: Track A — Control Room + Setup Screens

The MVP product surface: author-facing Web UI screens consuming the hardened Local API (P0) via the scaffold (P1). This is the largest plan; may split (P2a Control-Room READ ‖ P2b Setup CRUD) if @product-manager/@architect advise in Phase 2b review.

## Tasks

- [ ] T1: **TanStack Query data layer** — per-resource query hooks (`useWorks`, `useWork`, `useSessions`, `useSchedule`, `useCapabilities`, `useFindings`, `usePresets`, `usePreset`) consuming `NexusClient` (P1). Cursor-pagination handling; `ErrorResponse` → toast notifications. **F-P3 adapter**: thin transformer normalizing `works`/`schedules`/`sessions`/`capabilities` arrays → `items` (compass §5 item #2). **F-F1 adapter**: client-side sort for small lists.
- [ ] T2: **Works dashboard** (Control Room) — cursor-paginated list (status badges, completion %), filters, work-detail view (chapters/status for novel; sections for game-bible/script; draft for essay). Empty/loading/error states.
- [ ] T3: **Orchestration sessions view** (Control Room) — list sessions (running/completed/failed), per-session status detail, stage progression. READ-only.
- [ ] T4: **Schedule view** (Control Room) — cron roles per Work (brainstorm/write/review) + next-fire (UTC) + local-time display. Parity with CLI `creator works cron` output. READ-only.
- [ ] T5: **Capability registry browser** (Control Room) — list `nexus.*` capabilities + descriptions + admission gates. READ-only.
- [ ] T6: **Findings view** (Control Room) — list findings (post-F-P2 endpoint) with status/severity filtering; link to source work. READ-only.
- [ ] T7: **Work CRUD** (Setup) — create work (profile enum: novel/game-bible/script/essay), patch (status, stage), archive. Form validation against `@42ch/nexus-contracts` types.
- [ ] T8: **Preset management** (Setup) — full CRUD: list / show / create / update / delete / validate. Schemas already promoted (V1.63). Editor form for preset YAML/JSON; validate action surfaces errors.
- [ ] T9: Responsive layout + a11y (keyboard nav, ARIA, color-contrast per DESIGN.md tokens).
- [ ] T10: Verify — `pnpm --filter web build` + typecheck; manual smoke against running daemon (each screen loads, CRUD operations succeed, errors surface as toasts).
- [ ] T11: Capture screenshots for the `web-ui.md` spec (Shipped promotion evidence in P-last).

## Done when

- All 7 screen groups render against a running daemon via `BrowserClient`.
- Cursor pagination works on Works + Findings (consuming P0 F-P1/F-P2).
- Work CRUD + Preset management CRUD round-trip the Local API.
- `ErrorResponse` from any endpoint surfaces as a toast (consuming P0 F-E1).
- F-P3 + F-F1 adapters documented (deferral notes for P-last spec).
- DESIGN.md tokens applied consistently; dark mode works; a11y baseline met.
