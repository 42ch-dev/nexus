## 1. Problem Statement

V1.47 P0 persists findings via a **placeholder synthesis** (`kind=craft`, `severity=info`, empty `rule_suggestion`). Authors cannot trust finding severity or kind for routing. `REVIEW_PRESET_ID` is duplicated across three modules.

---

## 2. Goals

1. Parse `Works/<work_ref>/Logs/review/review-report.md` when present to populate `kind`, `severity`, `body`, and optional `rule_suggestion`.
2. Define safe fallback when report missing or parse fails (degrade to current placeholder with tracing).
3. Centralize `novel-chapter-review` preset id constant (R-V147P0-06).
4. Close R-V147P0-05 if hotfix H-1 not already on `main`.
5. Hermetic tests: parsed report → finding row shape; fallback path.

---

## 3. Non-goals

- Findings → draft prompt (P1).
- `AGENTS.md` write (P2).
- Retention policy (P3).
- New wire JSON schemas.

---

## 4. Acceptance Criteria

1. Review terminal with valid `review-report.md` creates findings with parsed fields per overlay §1.
2. Missing/malformed report still creates ≥1 finding with documented fallback + `tracing::warn!`.
3. Single SSOT constant for review preset id shared by auto_chain, validation allowlist, supervisor hook.
4. R-V147P0-01 and R-V147P0-06 closed in `status.json`.
5. R-V147P0-05 closed (hotfix or P0 T0).
