## Objective

Wire `nexus42` CLI commands to invoke `nexus-daemon-runtime` directly instead of spawning `nexus42d` binary. Implement foreground and background process models per spec Section 4.

## Scope

### In Scope

1. Add `nexus-daemon-runtime` dependency to `crates/nexus42/Cargo.toml`
2. Wire `daemon start --foreground` to runtime invocation in-process
3. Add hidden internal command: `nexus42 __internal daemon-run ...`
4. Implement background mode: self-spawn into `__internal daemon-run`, parent exits after startup gate
5. Wire `stop` / `restart` / `status` through runtime health and process coordination
6. Keep `nexus42d` temporarily compiling (but no longer spawned)

### Out of Scope

- Removing `nexus42d` (Batch 3)
- Naming/log unification (Batch 4)
- API/transport changes
