## 1. Problem Statement

Legacy code and prompts write to `Stories/<story_ref>/` at workspace root. Sync module scans that tree and treats all `.md` files as chapters. The novel profile requires `Works/<work_ref>/` with chapter scan limited to `Stories/` subtree. Pre-1.0: **full migration**, no dual defaults.

---

## 2. Goals

1. Migrate sync module + contract tests to `Works/<work_ref>/Stories/`.
2. Ship embedded templates: `work-status`, `chapter-outline`, `chapter-body`, `volume-outline` (minimal).
3. Update `acp-capability-set.md` path policy for `nexus.workspace.paths` (novel profile).
4. Remove normative `Stories/<story_ref>/` from prompts in `novel-writing` preset (P2 may patch paths; P3 completes pipeline).
5. Migration note in AGENTS.md / orchestration AGENTS if needed.

---

## 3. Non-goals

- Chapter drafting logic (P3)
- Data migration tool for existing user workspaces (pre-1.0 wipe acceptable per AGENTS.md)

## 5. Acceptance

1. Sync scans **only** `Works/<work_ref>/Stories/*.md` for chapters; per-chapter metadata is derived from `work_chapters` table (T13), not from a `work-status.md` file.
2. `README.md`, `Outlines/**` (incl. `foreshadowing.md`, `event-index.md`), `Logs/**` never appear in bundle chapters. No per-Work `Worldbuilding/` subtree ships or is scanned (T10/T13 + per novel-workflow-profile §3.5).
3. No runtime fallback to workspace-root `Stories/`.
4. Template files render with `work_ref` variable substitution.
5. `Works/<work_ref>/Outlines/foreshadowing.md` and `Outlines/event-index.md` ship as empty templates; `novel-project-init` (T11) writes them when scaffolding a new Work.
6. Embedded chapter body template renders with optional `volume` + `world_refs` variables (V1.36 leaves blank).
7. `work_chapters` table migration applies cleanly; `nexus-local-db` exports the typed module; queries return correct rows for a sample Work.
8. `creator run start --init-preset novel-project-init --idea "..."` creates `Works/<work_ref>/` tree, seeds `work_chapters` rows, sets `work.world_id` (if bound), and does **not** enqueue `novel-writing` automatically.
9. `creator run reconcile-chapters <work_id>` rebuilds `work_chapters` from filesystem; idempotent.
