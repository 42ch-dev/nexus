## 1. Problem Statement

Legacy code and prompts write to `Stories/<story_ref>/` at workspace root. Sync module scans that tree and treats all `.md` files as chapters. The novel profile requires `Works/<work_ref>/` with chapter scan limited to `Stories/` subtree. Pre-1.0: **full migration**, no dual defaults.

---

## 2. Goals

1. Migrate sync module + contract tests to `Works/<work_ref>/Stories/`.
2. Ship embedded templates: `work-status`, `chapter-outline`, `chapter-body`, `volume-outline` (minimal).
3. Update `acp-capability-set.md` path policy for `nexus.workspace.paths` (novel profile).
4. Remove normative `Stories/<story_ref>/` from prompts in `novel-writing` preset (P2 may patch paths; P3 completes pipeline).
5. Migration note in AGENTS.md / orchestration AGENTS if needed.

---

## 3. Non-goals

- Chapter drafting logic (P3)
- Data migration tool for existing user workspaces (pre-1.0 wipe acceptable per AGENTS.md)

## 5. Acceptance

1. Sync scans **only** `Works/<work_ref>/Stories/*.md` for chapters.
2. `README.md`, `work-status.md`, `Outlines/**` (incl. `foreshadowing.md`, `event-index.md`), `Worldbuilding/**`, `Logs/**` never appear in bundle chapters.
3. No runtime fallback to workspace-root `Stories/`.
4. Template files render with `work_ref` variable substitution.
5. `Works/<work_ref>/Outlines/foreshadowing.md` and `Outlines/event-index.md` ship as empty templates; `novel-project-init` (P1) writes them when scaffolding a new Work.
6. Embedded chapter body template renders with optional `volume` variable (V1.36 leaves blank).
7. `work-status.md` template renders 6-column chapter table.
