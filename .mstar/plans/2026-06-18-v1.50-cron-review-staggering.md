## 1. Problem Statement

Brainstorm + write cron work (T-A P1). Review cron does not yet fire. `novel-review-master` only runs on explicit `creator run` invocation. Authors cannot let review stagger every 30 minutes.

---

## 2. Goals

1. Cron evaluator handles `review` role with default `0,30 * * * *` (configurable per Work).
2. Review fire invokes `novel-review-master` preset; result feeds `open_findings_block` (existing V1.48).
3. Review cron respects per-Work gating (same as brainstorm/write).
4. T-B P1 hook stub: review-time extraction call site marked; T-B P1 fills with `kb_extract_jobs.pending` writer.

---

## 3. Non-goals

- T-B P1 full extraction logic (separate plan).
- Auto-chronology (T-A P3).

---

## 4. Acceptance Criteria

1. Cron evaluator handles all three roles; review fires on `:00` and `:30`.
2. Review result lands in `Logs/review/`; findings flow into `open_findings_block` per V1.48 contract.
3. T-B P1 hook stub is in place (function signature only; TODO marker).
4. Hermetic test: review cron fire with no chapters to review skips gracefully.
