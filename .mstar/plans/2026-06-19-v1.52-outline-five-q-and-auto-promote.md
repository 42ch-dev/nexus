## 1. Problem Statement

V1.36 shipped a **finalize-time** 五问 quality gate that blocks weak chapter drafts before `finalized`. V1.52 needs the same protection **before drafting**: an outline-time 五问 gate that rejects weak chapter outlines before `draft_chapter` runs, preserving the finalize gate unchanged.

Separately, V1.50/V1.51 review-time KB extraction always leaves candidates in `pending` for manual `creator world kb adopt`. Authors now see high-confidence, validation-clean, provenance-backed candidates that could safely promote on the happy path. V1.52 T-A P0 adds an explicit `--auto` promotion path with a full audit trail, while ambiguous candidates remain pending for human review.

This plan also closes V1.51 P-last residuals:

- **R-V151Q3-W001**: `extract_kb_candidates_for_review` and `LlmExtractTask::evaluate` contain duplicated LLM→`KbCandidate` parsing. Unify into a single helper.
- **R-V151Q3-W002**: `WorkerUnavailable` is currently surfaced as an empty `Vec` contract; make the unavailable state explicit so callers handle it cleanly.

## 7. Design Notes

### 7.1 Outline 五问 dimensions

The outline-time gate evaluates five dimensions that differ from finalize-time:

| Dimension | Heuristic signal |
|-----------|------------------|
| **structure** | Outline has clear beat-level sections (e.g., `##`, `-`, numbered beats). |
| **arc** | A character or situation change is described (conflict/revelation/stakes). |
| **foreshadow** | At least one `F###` reference or explicit future setup is present. |
| **pacing** | Outline length is within sane bounds (not empty, not a full draft). |
| **hook** | Final line ends with tension, question, or unresolved action. |

The pure function returns GO only when all five dimensions pass. In production the `llm_judge` template performs the authoritative evaluation; the pure function is used for deterministic tests and as a no-worker fallback signal.

### 7.2 Auto-promote threshold

A candidate is auto-promoted only when **all** of the following hold:

- `llm_confidence >= 0.95`
- `llm_source_quote` is non-empty (provenance_backed)
- `source_chapter_id` is non-NULL (provenance_backed)
- Adopt-time `ValidationMode::Novel` passes on the constructed `KeyBlockBody` (validation_clean)
- No duplicate `canonical_name` exists in the World (existing filter already applied by `kb_adopt`)

On failure of any condition, the row stays `pending` and the reason is recorded in the `--json` `skipped` array.

### 7.3 WorkerUnavailable explicit handling

Introduce a shared outcome enum:

```rust
pub(crate) enum LlmExtractOutcome {
    Candidates(Vec<KbCandidate>),
    WorkerUnavailable,
    CapabilityError(String),
}
```

`extract_kb_candidates_for_review` and `detect_missing_kb_on_finalize` fall back to the heuristic only on `WorkerUnavailable` or capability error, never on an empty-vec contract. `LlmExtractTask::evaluate` will return `Option<Vec<KbCandidate>>` or the same outcome type so that callers can distinguish "no candidates" from "worker missing".
