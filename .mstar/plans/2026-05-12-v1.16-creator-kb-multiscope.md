## Purpose

Implement the `creator kb` multi-scope minimum model: local `work` scope is operational, `world` scope is accepted but deferred. This establishes the scope plumbing that Plan 8 (creator identity/auth compat) and future plans will build upon.

**Delivery compass ref**: `creator kb` multi-scope 基线 — 本地默认 `work`，并预留 `world`/扩展 scope 的对接语义。

**Done definition check**: `creator kb` 默认 `work` 可用，`world` 映射可用。

## Current state (post Plan 4)

- `KbCommand` has `List` and `Search` stubs (both print "Coming soon")
- No scope concept exists
- `KbEntry` type exists in `nexus-domain::context_assembly` (entry_id, title, content) — read-only reference type for context assembly, NOT a storage model
- No local DB table for KB entries
- `nexus-home-layout` defines workspace paths but no KB-specific paths

## Design

### Scope model

```rust
#[derive(Debug, Clone, clap::ValueEnum, Default)]
pub enum KbScope {
    /// Local workspace knowledge (default)
    #[default]
    Work,
    /// Platform/world-scoped knowledge (future)
    World,
}
```

- `--scope` flag on all `kb` subcommands (default: `work`)
- `work` scope resolves to active creator's active workspace
- `world` scope is accepted but returns a deferred message

### KB local path layout (work scope)

```
~/.nexus42/creators/<creator_id>/workspaces/<workspace_slug>/kb/
├── entries/          # individual KB entry files
│   └── <entry_id>.md
└── index.json        # entry index (entry_id, title, created_at)
```

This follows ADR-014 pattern (flat files + JSON index, same as SOUL.md approach).

### Command surface changes

```
creator kb list [--scope work|world]    # list KB entries in scope
creator kb search <query> [--scope work|world]  # search entries
creator kb show <entry_id> [--scope work|world]  # show single entry
creator kb add --file <path> [--title <title>] [--scope work]  # add entry from file
```

### Implementation approach

1. **Scope enum + flag** — pure CLI plumbing, no I/O
2. **`kb list`** — for `work`: scan `kb/entries/` directory, read `index.json`; for `world`: deferred message
3. **`kb search`** — for `work`: grep title/content in index; for `world`: deferred message
4. **`kb show`** — for `work`: read entry file; for `world`: deferred message
5. **`kb add`** — for `work`: copy file into `kb/entries/`, update index; for `world`: deferred message
6. **Home layout helper** — add `creator_kb_dir()` and `creator_kb_entries_dir()` to `nexus-home-layout`

## Non-goals

- Full KB CRUD API (edit, delete, update) — deferred to post-V1.16
- `world` scope implementation — accepted as parameter, returns deferred message
- DB-backed KB storage — file-based baseline (same as SOUL.md pattern)
- KB sync with platform — out of scope
