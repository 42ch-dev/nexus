## 1. Problem Statement

V1.26 residual **R10**: `run_assemble_moment` uses `InMemoryKnowledgeStore::new()`, so the User knowledge domain is empty after restart even when narrative and World KB are persisted.

There is no CLI to author User-scoped knowledge entries locally. DF-43 left **KnowledgeEntry SQLite** open while narrative + World KB shipped in V1.26.

---

## 2. Goals

### Theme U — User knowledge persistence

1. Migration `knowledge_entries` in workspace `state.db` (`user_id`, `entry_id`, tags JSON, content, timestamps).
2. `SqliteKnowledgeStore` in `nexus-local-db` implementing `nexus_knowledge::KnowledgeStore`.
3. CLI `creator knowledge`:
   - `add` — store entry (default `--user-id user_default`; optional override)
   - `list` — paginated list for user
   - `search` — text search for user
4. Update [local-db-schema.md](../knowledge/specs/local-db-schema.md) §4 with `knowledge_entries`.
5. Close residual **R10**; update DF-43 tracker note (KnowledgeEntry portion shipped).

### Theme C — Context + bootstrap

1. Replace `InMemoryKnowledgeStore` in `run_assemble_moment` with `SqliteKnowledgeStore` (same pool as other CLI context paths).
2. `creator demo seed` — one command writes demo world + event + key block + knowledge entry (calls Plan 1 write APIs + knowledge store).
3. Persistent E2E or integration test: seed → new process → `assemble_moment` / CLI `assemble-moment` shows User knowledge section (closes V1.26 QC gap on in-memory-only demo test).
4. Optional cli-spec §6.2F note: User knowledge uses `user_default` until platform `usr_*` mapping.

---

## 3. Non-goals

- Daemon routes for User knowledge.
- Merge `nexus-knowledge::ReferenceSource` production ownership (DF-43 reference portion unchanged).
- Change `assemble-local` / Stage0 semantics.
- Platform HTTP for `ContextCommand::Assemble`.
