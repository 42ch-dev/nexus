## 1. Problem Statement

V1.37 P3 defined findings lifecycle roadmap-only. V1.39 needs a local `findings` table, API, and routing to write/brainstorm/none/master executors integrated with auto-chain review stage.

---

## 2. Goals

1. DDL migration + DAO for `findings` per novel-workflow-profile §5.5.1 sketch.
2. Local API: list/create/update findings by work_id (+ optional chapter).
3. Severity enum: `info | minor | major | blocker`; status: `open | resolved | wont_fix`.
4. Review stage writes findings from `reflection-loop` or successor; route `target_executor` field.
5. `creator run status` Findings section per §5.5.6.

---

## 3. Non-goals

- `novel-brainstorm` / `novel-review-master` preset bodies (P2).
- Rules files (P3).
- 96h daemon task (P4).

---

## 4. Acceptance Criteria

1. Finding CRUD via daemon API with creator isolation.
2. Review stage completion can create at least one finding row linked to work/chapter.
3. Status lists open findings with severity and suggested next action (write/brainstorm/none/master).
4. Findings do not fork auto-chain driver schedule.
