### KB naming sub-grill resolution

**Decision**: Use `nexus.kb_snapshot.read` for P1 DF-46 read slice.

**Rationale**:
1. The ACP capability catalog (`acp-capability-set.md` §4.2) already lists `nexus.kb_snapshot.read`.
2. The existing naming convention uses `nexus.<domain>.<action>` (e.g. `nexus.context.whoami`, `nexus.workspace.info`) or `nexus.<compound-domain>.<action>` (e.g. `nexus.orchestration.schedule_status`). "kb_snapshot" follows the compound-domain pattern.
3. "kb.keyblock.read" would introduce an unnecessary third hierarchy level inconsistent with existing ids.

**Recorded in**: `capability-registry.md` §2 row definition; this plan §7.

### No temporary parallel paths

All Sub-phase transitions are atomic (single commit per sub-phase). No lingering dual paths exist at plan close. The old `dispatch_tool()` match table was removed in Sub-phase 3. No deferred residuals recorded.
