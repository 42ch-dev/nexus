## 1. Scope

Deliver Mode 1 spec governance (bridge→Master draft + acp §4 roster rewrite) and consolidate the capability::Registry as the SSOT for all 35 implemented catalog IDs. Migrate handler bindings from duplicated `execute_X` functions in `host_tool_executor.rs` into `capability/builtins/`; each ID gets a test vector.

### In scope

- **Bridge Master draft**: Update `agent-nexus-tool-bridge.md` header (document class → Master, `Status: Master (V1.57 P-last promote)` draft-ready body). Coordinate cross-references in `acp-capability-set.md`, `capability-registry.md`, `acp-client-tech-spec.md`. Final promotion happens in P-last.
- **acp §4 roster rewrite**: Replace the current minimal capability set table with a complete roster of all 36 catalog IDs. Each row: ID, one-line description, status tag (shipped / scaffold-equivalent / OUT / deferred-to-V2.0+), which plan shipped it, which registry row binds it.
- **Registry consolidation**: Move all 35 implemented `nexus.*` handler bindings from `host_tool_executor.rs::execute_X` into `capability/builtins/`. Each ID gets a `CapabilityRegistryRow` with: `id`, `access`, `admission` gate chain, `handler`, `ACP wire` shape, `failure_mode`, and `handler_test_vector`.
- **Test vector per ID**: At minimum one success + one failure test per registered ID, dispatched through `CapabilityRegistry::dispatch()` (not by calling handlers directly).
- **Cross-validation**: Catalog ↔ registry invariant test that verifies every registry `id` has a catalog row and every shipped catalog ID has a registry row.
- **Cross-cut**: `R-V156P3-S003` (field drops) — remove unused capability fields during registry consolidation.

### Out of scope

- `host_tool_executor.rs` refactor (P1 owns the god-file split).
- `host-call` CLI subcommand (P1 owns it).
- Worker IPC 35-ID extension (P3 owns it).
- Bridge final Master promotion (P-last owns the header rename).
- `R-V156P1-M001` schema rename (P2 owns it).
- `R-V156P1-M002` CdnConfig injection (P1 owns it).
- New capability implementations beyond the existing 35 shipped IDs.
