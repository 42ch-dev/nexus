**Predecessor**: [v1.67-local-api-surface-convergence-and-derisk-delivery-compass-v1.md](v1.67-local-api-surface-convergence-and-derisk-delivery-compass-v1.md) (Shipped via PR #91, merged 2026-06-26 @ `4606395e`)
**Successor**: TBD (V1.69 — Canvas Strategy Surface *implement* per V1.67 §1.2 roadmap, conditions permitting)
**North star**: **Make the monorepo's product topology self-evident.** `nexus42` is the product (CLI + integrated daemon runtime); the other `crates/*` are its building blocks. Today the product binary is buried among 15 library crates, and `apps/` — the directory a contributor instinctively opens for "what can I run" — lists only the TS clients (`desktop`, `web`) and omits the CLI entirely. V1.68 unifies the **"all product surfaces under `apps/`"** mental model: relocate `crates/nexus42` → `apps/nexus42` so `apps/` becomes the **polyglot product-surfaces directory** (Rust CLI/daemon + TS desktop + TS web), `crates/` returns to **pure Rust libraries**, and the three-tier model — `packages/` (contracts) → `crates/` (libs) → `apps/` (products) — becomes legible at a glance. The relocation is low-risk (Cargo resolves deps by crate **name**, not path → **zero source changes**; CI references crate names → **CI-invisible**), and is paired with documentation that **pins the rule durably**: any future product surface of any language lands in `apps/`; Rust libraries land in `crates/`.
**Effort**: **S** (single implement plan, ~6 live reference updates + 4 doc artifacts; the ~90 `crates/nexus42` mentions are ~85% historical records under `.mstar/archived/` which are intentionally left as-is). Intentionally the smallest iteration in the V1.6x series — a structural hygiene step that *enables* cleaner contributor navigation for every subsequent feature iteration (V1.69 canvas onward).

## 6. Layout reference (before → after)

```text
BEFORE                                  AFTER
apps/                                   apps/
  desktop/   (TS + src-tauri Rust)        desktop/   (TS + src-tauri Rust)  consumer
  web/       (TS SPA)                    nexus42/   (Rust CLI + daemon) ◀── producer  [MOVED]

  nexus42/   (CLI — the product)        crates/

                                          (no binary)
```

**Mental model pinned (durable rule, in `apps/AGENTS.md`):**

> `apps/` = **product surfaces** (runnable things you can install/use), polyglot.
> `crates/` = **Rust libraries** (building blocks).
> `packages/` = **publishable npm libraries** (wire contracts).
> A new product surface of *any* language → `apps/`. A new Rust library → `crates/`.
> `nexus42` is the **producer** (daemon+CLI composition root); `desktop`/`web` are **consumers** (clients over the Local API / IPC boundary).
