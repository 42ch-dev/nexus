The third loop in the author-in-command vision is the **creator self-loop**: the runtime captures session-end digests as *pending reviews* (what the author just did), the author *reviews* them (triggering summarization into long-term *fragments*), and the resulting **SOUL** (the creator's persistent memory / personality) *internalizes* the experience for future moments. > **Terminology note**: This iteration uses three names at different levels: **(a) "creator self-loop"** ‚ÄĒ the conceptual loop (capture ‚Üí review ‚Üí internalize), the third author-in-command loop; **(b) "Creator Memory / SOUL review-loop UI"** ‚ÄĒ the full product-surface name (schemas + codegen + normalization + UI); **(c) "Memory review-loop"** ‚ÄĒ the implementation scope within Track A.
