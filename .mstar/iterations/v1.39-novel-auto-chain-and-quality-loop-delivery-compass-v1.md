## 0. Position

V1.38 shipped DF-62's first slice: DB-selected chapters 2..N, status UX, and `novel-writing` parameterization — but **explicitly without** auto-reenqueue or full FL-E chaining. V1.39 completes the product loop ordinary users expect:

> **While the daemon is running, the novel Work advances through intake → research → produce → review → persist per chapter without manual `stage advance` at every boundary; daemon restart resumes from the last checkpoint. Users may inject inspiration and research updates without forking the chain.**

V1.39 also implements the V1.37 P3 quality-loop roadmap (DF-64/65/66/67) including `novel-brainstorm` and `novel-review-master` presets.

When this compass conflicts with a later locked implementation plan, the active plan wins for delivery batching only. Normative behavior changes still require spec updates before implementation.

## 2. Target User Journey

### 2.0 Chapter outer loop (daemon online)

```text
creator run start --idea "..."     # --auto-chain default true
  → intake completes → research → produce ch01 → review → persist ch01
  → auto enqueue produce ch02 → review → persist ch02 → ... → Work completed
```

User may run `creator run continue <work_id> --note "..."` at any time; chain continues after current state finishes.

### 2.1 Daemon-attached continuity

| State | Behavior |
| --- | --- |
| Daemon online + `auto_chain_enabled` | Stage/chapter boundaries auto-enqueue next driver schedule |
| Daemon shutdown mid-run | Checkpoint records `stage`, `chapter`, `driver_schedule_id`, `auto_chain_interrupted` |
| Daemon restart | Boot auto-resumes **only** schedules tied to checkpointed auto-chain Works; others stay paused (today's safe default) |
| `creator run status` | Shows `daemon: online/offline`, `chain: running/paused_at_<stage>_ch<N>`, `pending_resume` |

Opt-out: `--no-auto-chain` on start; Work-level disable; explicit `creator run resume <work_id>` when auto-resume skipped.

### 2.2 Side-input lane (non-breaking insertions)

1. **No fork**: insertions must not create a second active FL-E driver schedule per Work.
2. **No cancel**: inspiration / research completion must not cancel the current driver.
3. **Visibility**: status shows pending inspiration count and research KB refresh hints.
4. **Consumption**: inspiration merges at next state transition; research KB available to produce/review via existing query paths (P0.5).
