## 0. Position

V1.40 closed **World KB continuity** for a single active novel Work. The next **product completeness gap** for prolific authors is **multi-book lifecycle** (DF-60 + DF-61):

> A user can auto-chain chapters on one Work, but cannot reliably **finish a book**, **set a default novel for CLI**, or **track queued ideas** across Works without manual filesystem surgery. novels-system solves this with Redis + 8-step ceremony; OSS must map to **SQLite + `Works/`** with a **Medium** ceremony (grill-me locked). **Concurrent** multi-Work execution is allowed; there is **no** global switch mutex.

V1.41 implements DF-60 (P0) and DF-61 (P1) per grill-me decisions, plus aggressive hygiene (P-last). **Multi-volume PK** (DF-62 continuation) is explicitly deferred to V1.42.

When this compass conflicts with a later locked implementation plan, the active plan wins for delivery batching only. Normative behavior changes still require spec updates before implementation.

## 2. Target User Journey

### 2.0 Complete book and start next (primary)

```text
nexus42 creator works status          # Work A nearing completion (default = pool active)
  → last chapter finalize + intake complete
  → mark_work_completed: Work A.status = completed
  → Works/<work_ref_A>/.completion-lock.json written
  → pool row for A → completed; pool active slot cleared
  → auto-chain stopped on A only; Work B may still run concurrently
nexus42 creator works pool promote <entry_id> --set-default
  → queued → bind/scaffold Work B; pool active = B
  → OR: nexus42 creator run start --from-work <work_id_A> --idea "..."
  → OR: nexus42 creator works use <work_id_B>
```

### 2.1 Track ideas without starting a Work

```text
nexus42 creator works pool inspiration add --title "Cyberpunk heist"
  → DB row + Works/_pool/灵感池/cyberpunk-heist.md scaffolded
nexus42 creator works pool list
  → shows 在写 / 排队开坑 / 已完结 rows from state.db (SSOT)
```

### 2.2 novels-system mapping (reference only)

| novels-system | V1.41 OSS |
| --- | --- |
| Redis `novel:active` | `novel_pool_entries.status = active` (CLI default only) |
| 8-step switch + 2h lock | **OUT** — completion-lock + runtime lock |
| Global switch mutex | **OUT** — concurrent multi-Work OK |
| `选题库.md` file SSOT | **DB SSOT**; optional markdown export |
| `灵感池/` Obsidian tree | `Works/_pool/灵感池/*.md` + DB path ref |

See [deferred-features-cross-version-tracker.md](../knowledge/deferred-features-cross-version-tracker.md) §3.6.1.
