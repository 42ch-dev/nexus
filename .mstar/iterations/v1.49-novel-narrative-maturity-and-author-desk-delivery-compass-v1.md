## 0. Position

V1.48 closed the findings **produce→consume→rules** chain and hardened serial §4.5.7 #4/#5. The **largest remaining novel product gaps** are:

1. **Findings lifecycle** still three-state (`open` / `resolved` / `wont_fix`) while roadmap promised extended workflow (V1.48 OUT F6).
2. **Narrative indexes** (`Outlines/foreshadowing.md`, `event-index.md`) are scaffolded and referenced in prompts but have **no runtime** to allocate F###/E### ids or promote outline declarations into the index files.
3. **Author desk UX** gaps: no intake re-trigger on existing Work (R-V147P1-01); `reconcile-chapters` mutates without preview (R-V148P4-W2).
4. **Serial reliability** follow-ups: reconcile holds runtime lock for full filesystem walk (R-V148P4-W3); findings prune DAO has no CLI (§9.4); review-report path guard deferred (R-V148P0-W1).

V1.49 addresses these in P0–P3 with bounded P-last hygiene on 8–10 V1.46 lows.

When this compass conflicts with a later locked implementation plan, the active plan wins for delivery batching only. Normative behavior changes still require spec overlay updates before implementation.
