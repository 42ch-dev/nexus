## 0. Position

V1.36 shipped the first complete novel-writing vertical slice: a generic Work with `work_profile: novel`, a `Works/<work_ref>/` artifact tree, `novel-project-init`, `work_chapters` as the chapter state SSOT, a chapter outline/draft/finalize pipeline, and a completion stop. That was intentionally a **single-chapter MVP**.

V1.37 locks the next axis:

> **Make the V1.36 novel-writing path reliable for ordinary users, then prepare the next serial-writing slices without over-claiming implementation.**

The immediate product risk is not that Nexus lacks more advanced novel features. It is that a normal user can still hit declaration-only or partially wired foundation pieces:

- `novel-project-init` grill-me answers do not fully reach `preset.input`.
- `gates:` are declared in specs/YAML but runtime enqueue evaluation is still a V1.37 residual.
- Scaffold DB writes are recoverable but not yet one atomic transaction across `work_chapters` seed + `works` PATCH.
- Multi-chapter rows are seeded, but `novel-writing` remains effectively a chapter-1 path.

When this compass conflicts with a future locked implementation plan, the active plan wins for delivery batching only. Normative behavior changes still require spec updates before implementation.

## 2. Target user journey (V1.37 direction)

```text
creator workspace init && daemon start && acp agent use
creator run start --idea "..." --work-profile novel
  → if scaffold missing:
       either run/suggest novel-project-init with clear remediation
       or fail fast with preset_gates_failed and exact next command
  → novel-project-init grill-me answers:
       work_ref, title, total_planned_chapters, optional world_id
       answers become preset.input.* and schedule input
       scaffold + work_chapters seed + works PATCH commit atomically
  → creative-brief-intake completes
  → novel-writing enqueue gates pass
  → chapter production starts from the next not_started work_chapters row
creator run status <work_id>
  → shows reliable progress and gate/remediation state
```

V1.37 P0 must make the front door trustworthy even if P1+ multi-chapter implementation is still pending.
