## 0. Position

This compass is the **third full audit**. V1.24 closed **normative doc drift** (Cargo graph, Local API table, work-KB semantics, context/assemble retirement, crate `AGENTS.md`). V1.25 focuses on what remains:

1. **Product wiring** — domain crates reachable in Cargo but not in CLI/daemon handlers.
2. **Spec/code split** — claims that four-domain context assembly is shipped when `nexus42` only calls `Stage0Assembly` / `TwoStageAssembly`.
3. **Residual tails** — JSON schema text, clap help, deferred-tracker accuracy, CLI `daemon orchestrate *` vs shipped `daemon schedule`.

**Normative long-term rules** stay in `knowledge/specs/`. This file drives the **next** locked plan under `.mstar/plans/`; it does not replace `entity-scope-model.md` or `local-cloud-crate-architecture.md`.

When this compass conflicts with an active implementation plan, the active plan wins for delivery.

## 5. Design issues (normative)

### 5.1 Context assembly mode split (KCA-25-003)

**Problem:** [cli-spec.md](../knowledge/specs/cli-spec.md) capability row (§6.2F area) states `platform context assemble` reads `nexus-creator-memory`, `nexus-narrative`, `nexus-kb`, `nexus-knowledge`. Implementation in `crates/nexus42/src/commands/platform/context.rs` builds **`Stage0Assembly`** from local SOUL/memory/fragments only. `assemble_moment` in `nexus-moment-context-assembly` aggregates four domains but has **no** `nexus42` caller.

**Risk:** Agents and contributors assume World KB / narrative slices are in production context paths.

**Recommended direction:**

- **Option M1 (spec-first):** Split cli-spec into **Stage-0** (shipped) vs **Moment four-domain** (library-ready, not CLI-shipped).
- **Option M2 (product):** Add `platform context assemble-moment` (or flag on assemble) calling `assemble_moment` with configured stores — Theme B below.

### 5.2 Work KB vs World KB (closed policy, open product)

V1.24 **KCA-003 C2** remains correct: `/v1/local/kb/*` and `creator kb` default = **work-scope file index**. World KB (`nexus-kb`) and `creator kb --scope world` remain deferred (**DF-42**). No regression found.

### 5.3 References vs User knowledge (KCA-25-006)

Daemon `GET /v1/local/references` → `nexus-local-db` SQLite. `nexus-knowledge::reference_source` parallel module not product-wired. **DF-43** remains the persistence strategy tracker.

### 5.4 Orchestration CLI parity (KCA-25-005)

| Surface | Spec | Code |
| --- | --- | --- |
| `daemon schedule *` | cli-spec | Wired to `/v1/local/orchestration/schedules/*` |
| `daemon orchestrate list\|run\|…` | cli-spec §6.3 | All **Coming soon** in `commands/daemon/mod.rs` |
| Daemon orchestration sessions API | orchestration-engine | **Shipped** under `/v1/local/orchestration/sessions/*` |

**Recommended direction:** Either thin CLI wrappers over existing session/schedule HTTP (**Theme D**) or mark orchestrate subcommands **Deferred** in cli-spec until implemented.
