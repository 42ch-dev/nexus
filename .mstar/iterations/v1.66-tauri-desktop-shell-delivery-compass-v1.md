## 1. Scope / non-goals / out-of-scope

### 1.1 In scope

**Track A ‚Äî Tauri Desktop Shell (V1.66 lead slice)**

Ship `apps/desktop` ‚Äî a Tauri v2 application that wraps the V1.65 `apps/web` dist and delivers the desktop experience. Screen data access stays **transport-agnostic** (clean `NexusClient` boundary); V1.66 adds only **desktop-gated affordances** inside `apps/web` (context-menu entries + capability detection + daemon-status indicator ‚Äî hidden when not in desktop mode) + the `TauriClient` factory selection ‚Äî no new browser-build UI features. (Phase 2b architect F-2 precision: the literal claim "SPA unchanged" is overstated; the accurate statement is "screen data access remains transport-agnostic; only desktop-gated affordances + client factory selection are added.")

- **`apps/desktop` Tauri v2 scaffold**: new pnpm workspace member under `apps/*`; `tauri.conf.json` (productName, macOS bundle id, window config, webview source = bundled `apps/web` dist); Rust `src-tauri/` (Tauri app entry, plugin registration, sidecar wiring). macOS-only target in V1.66 (`aarch64-apple-darwin` + `x86_64-apple-darwin` universal; Windows/Linux targets deferred).
- **`TauriClient` impl** (`apps/web/src/lib/nexus/tauri-client.ts` ‚Äî replace the V1.65 stub): implement the 21-method `NexusClient` interface. **Architecture decision (Phase 2b LOCKED ¬ß5 #1)**: thin desktop-augmentation over `BrowserClient` ‚Äî the 21 data methods reuse the same HTTP transport to the localhost daemon (Tauri's webview can `fetch http://127.0.0.1:<port>/v1/local/*` directly, identical to `BrowserClient`), and `TauriClient` adds only the desktop-only methods that the browser sandbox cannot perform. This reuses all of V1.64/V1.65 HTTP work and keeps the new surface small.
- **Desktop-only `NexusClient` extensions** (the actual new surface): `openWith(path)` / `revealInFinder(path)` (Tauri `@tauri-apps/plugin-opener` / `shell.open` with scope whitelist) / `copyPath(path)` (already in V1.65 browser; reuse) / daemon lifecycle probes (`getDaemonStatus`, `startDaemon`, `stopDaemon` ‚Äî see P1). These are exposed to `apps/web` via the `NexusClient` interface only when running in desktop mode (capability detection at boot).
- **Q5 desktop actions ‚Äî right-click context menu on chapter body/outline paths**: wire the V1.65 "Copy path" + new "Open with‚Ä¶" (system MD editor picker) + "Reveal in Finder" into the chapter body read-only view + outline editor surfaces. Browser build keeps "Copy path" only (Q5 from V1.65); desktop build unlocks the native actions. Scope whitelist (`tauri.conf.json` `opener` scope) restricts `openWith`/`reveal` to paths under the active workspace root (W-002-equivalent path guard, mirroring `host_tool_handlers.rs`).
- **Bundled `nexus42` sidecar** (P1): the Tauri app **bundles** the `nexus42` binary (Tauri `externalBin` + target-triple rename convention) and **lifecycle-manages** it (auto-start on app launch via `@tauri-apps/plugin-shell` Sidecar API; stop on app quit; health probe + restart-on-crash). The user double-clicks the `.app` and the daemon comes up transparently ‚Äî no terminal, no `nexus42 daemon start`. **Sidecar mechanism decision (Phase 2b, architect-owned)**: lean **Tauri `externalBin` + plugin-shell Sidecar**; reject PATH-discovery (defeats the desktop-app DoD) and defer in-process `nexus-daemon-runtime` lib link to V1.67+ (per V1.65 compass).
- **Port discovery** (P1, Phase 2b architect-owned): how `TauriClient` finds the sidecar's port ‚Äî lean **fixed default 8420 with override + health probe** (matches existing `NEXUS_DAEMON_PORT` env); dynamic port allocation + handshake deferred unless the architect finds a concrete need.
- **macOS CI unsigned `.app` build leg**: extend `.github/workflows/` with a `desktop-build` job (macOS runner, `pnpm --filter desktop tauri build`, unsigned `.app` + `.dmg` artifacts uploaded to the workflow run). **No signing, no notarization, no auto-update, no GitHub Releases publish** in V1.66 (T1 DoD).
- **DESIGN.md increment** (Prepare gate, architect-owned Phase 2b): window chrome / app menu / native dialog / system-tray (if any) / desktop-context-menu tokens appended to `apps/web/DESIGN.md` (or a new `apps/desktop/DESIGN.md` ‚Äî architect decides). Standard+ ‚Üí Standard+ (desktop supplement); Production still V1.67+. Required before P0/P1 implement.

### 1.2 Non-goals / out-of-scope (durable V1.67+ roadmap)

- **Body full-text editor + per-chapter edit lock** ‚Äî **V1.67 lead authoring slice**. Requires lock design (UI claims chapter N ‚Üí orchestration skips/queues; lock expiry policy), MD‚Üîrich-text lossless round-trip, frontmatter/status sync, conflict policy. The single highest-risk authoring surface; lands only after the locking design is reviewed. V1.66 renders body read-only (inherited from V1.65). This is the explicit reason Q1 split the body editor out of V1.66.
- **UI productivity wave** ‚Äî **V1.67**. Drag-to-reorder chapters, bulk status/volume operations, manual reconcile trigger, outline template library. Pure `apps/web` features that pair naturally with the V1.67 body editor (the "authoring iteration").
- **Windows + Linux desktop builds** ‚Äî **V1.67+**. Tauri v2 supports tri-OS, but the CI matrix + per-OS webview deps + testing is the biggest complexity multiplier. V1.66 proves macOS first.
- **Code signing + Apple notarization + Windows Authenticode** ‚Äî **V1.67+**. Requires Apple Developer ID + notarization CI leg + Windows signing cert. Signing without identities ready is a classic iteration-killer.
- **GitHub Releases distribution + auto-update (Tauri updater)** ‚Äî **V1.67+**. The unsigned `.app` from CI is the V1.66 deliverable; distribution + auto-update follow once signing lands.
- **In-process `nexus-daemon-runtime` lib link** ‚Äî **V1.67+ optimization** (per V1.65 compass). V1.66 uses the external `nexus42` sidecar binary. The in-process lib link removes the IPC boundary but couples the daemon lifetime to the app process ‚Äî defer until the sidecar pattern is proven.
- **System tray / menu bar app / global hotkeys / native notifications** ‚Äî **V1.67+**. V1.66 ships a standard windowed app; richer shell chrome deferred.
- **Mobile (Tauri v2 mobile targets ‚Äî iOS/Android)** ‚Äî **post-V1.67**. Mobile is premature before the desktop shell proves the Tauri pattern.
- **`R-V165-QC3-VIRT` ‚Äî chapter table virtualization for large chapter counts** ‚Äî V1.67 (UI change; pairs with body editor's large-chapter-count story; premature without real chapter-count data).
- **`R-V165-QC-SUGG-DX` ‚Äî DX/UX polish grab-bag** ‚Äî V1.67+ (deferred as a unit; non-UI subset can be cherry-picked in V1.67 if cheap).
- **Track-B big sweeps** ‚Äî V1.67+: `R-V164-QC1-CASING` (error-code casing convention, coupled to FE1-ORCH), `R-V164-FE1-ORCH` (orchestration ~1300 LoC error-envelope sweep), `R-V163P3-SURF-003` (global F-P3 array rename ‚Üí `items`), `R-V163P3-SURF-005` (F-F1 sort params), `R-V164-P2-G3` (CapabilityInfo admission-gate UI).
- **DESIGN.md ‚Üí Production completeness** ‚Äî V1.67+ alongside the body editor + UI productivity.
- **Platform/cloud features** ‚Äî DF-13/16/55/59 remain platform-gated (PD-05); platform remains paused.
- **agent-host sessions/operations/events(SSE) DTO promotion** ‚Äî long-lived stateful connections; remains deferred (carried from V1.63 non-goal).

### 1.3 Wire contracts note

**`wire_contracts_changed: false` (CONFIRMED Phase 2b by @architect)**. The Tauri shell is a **packaging/delivery layer**, not a wire-layer change. (Phase 2b architect F-6: sidecar lifecycle + open-with/reveal are **Tauri IPC / app-side process control**, not Local API HTTP ‚Äî explicit to prevent schema creep.)

1. `TauriClient` (thin-over-`BrowserClient`, ¬ß5 #1 LOCKED) reuses the **identical HTTP transport** to the localhost daemon ‚Äî the same V1.64/V1.65 Local API routes, the same `@42ch/nexus-contracts` TS types, the same `ErrorResponse`/cursor conventions. No new request/response shapes.
2. The desktop-only methods (`openWith`, `revealInFinder`, daemon lifecycle) are **Tauri invoke / IPC**, not Local API HTTP ‚Äî they don't touch `schemas/`.
3. Track B hygiene is test/refactor/hardening ‚Äî no schema changes (`PaginationInfo` dedup is a Rust-side type consolidation, not a wire shape change; the serialized form is identical).
4. `@42ch/nexus-contracts` npm version is **unaffected** by V1.66.

## 5. Open design items ‚Äî LOCKED (Phase 2b @architect, 2026-06-25)

All 10 items resolved by `@architect` Phase 2b. Decisions below are the locked answers (the original leans were confirmed, with the noted refinements).

1. **`TauriClient` architecture ‚Äî LOCKED: thin desktop-augmentation over `BrowserClient`.** The 21 `NexusClient` data methods delegate to the existing HTTP transport with an explicit desktop `baseUrl` (`http://127.0.0.1:<resolvedPort>`); a separate desktop-only capability surface (`DesktopNexusClient extends NexusClient`, or equivalent capability object) adds the native methods. **Not** a full Tauri-plugin IPC rewrite. Preserves the `types.ts` adapter boundary; reuses the `BrowserClient` transport core; keeps `tauri-client.ts` from becoming a second handwritten Local API stack. (Method count reconciled Phase 2b: 21 methods in `types.ts` ‚Äî health + 20 data; the preset-CRUD methods from R-V164-P2-G2 are additive routes not yet promoted to the interface, so 21 is the V1.66 truth.)

2. **Sidecar mechanism ‚Äî LOCKED: Tauri `externalBin` + `@tauri-apps/plugin-shell` Sidecar.** `bundle.externalBin` + `Command.sidecar(...)`; reject PATH-discovery (fails T1 DoD); defer in-process `nexus-daemon-runtime` lib link to V1.67+. Tauri v2 requires `shell:allow-execute` capability with `sidecar: true` (Context7-verified).

3. **Port discovery ‚Äî LOCKED: fixed default `8420` + `NEXUS_DAEMON_PORT` override + health probe.** Resolution order: explicit configured port ‚Üí `NEXUS_DAEMON_PORT` (if valid) ‚Üí `8420`. App passes `--port <resolved>` to the sidecar so CLI args + env cannot diverge. No dynamic port handshake in V1.66. Readiness = `GET /v1/local/runtime/health` returning healthy (not stdout parsing). Matches existing `boot.rs` default + CLI `daemon start --foreground --port`.

4. **Asset serving ‚Äî LOCKED: Tauri serves `apps/web/dist` via `build.frontendDist`; daemon rust-embed stays for browser-tab flow.** No daemon static fallback inside desktop mode. Cleanly separates Tauri-packaged UI from daemon-served browser UI; preserves V1.64 browser-tab behavior. (`build.frontendDist` is the correct Tauri v2 key, under `build` ‚Äî Context7-verified.)

5. **`wire_contracts_changed` ‚Äî LOCKED: false.** No schema needed for desktop packaging, sidecar lifecycle, path opening, or the 3 residuals. Desktop-native methods are Tauri IPC/app-side contracts, not `/v1/local/*` wire shapes. Existing `DaemonHealth` already typed + consumed via `/v1/local/runtime/health`. No `@42ch/nexus-contracts` version bump.

6. **DESIGN.md increment ‚Äî LOCKED: append to `apps/web/DESIGN.md`.** Single design-system SSOT (already the token SSOT with light/dark values). Do **not** create `apps/desktop/DESIGN.md` in V1.66. Scope: window chrome, app menu, native dialogs, desktop context menu, daemon-status indicator. **System tray: none in V1.66.** Production polish (custom title bar, animated transitions, tray/menu-bar app) deferred V1.67+.

7. **Capability detection ‚Äî LOCKED with refinement: `NEXUS_DESKTOP` flag + Tauri API detection, checked once at client factory.** Primary signal = injected build/runtime flag; sanity check = Tauri API presence. If using `window.__TAURI__`, set `app.withGlobalTauri: true` explicitly; otherwise prefer `@tauri-apps/api/core`'s `isTauri`. Avoid scattering detection across screens (Context7-verified drift: `window.__TAURI__` only exists with `withGlobalTauri`).

8. **`openWith`/`revealInFinder` scope-whitelist path guard ‚Äî LOCKED with security refinement (Phase 2b F-1).** **Runtime canonicalize + prefix-check against the active workspace root is authoritative** ‚Äî it must mirror the W-002 guard intent. **Tauri capability scope is defense-in-depth only** ‚Äî Tauri permissions are *static* capability scopes and **cannot** encode a *dynamic* active workspace root. **Prefer custom Tauri commands** (`open_with`, `reveal_in_finder`) that validate the path (canonicalize + prefix-check) before calling opener functionality, over relying solely on the static opener scope. Coordinate with P-sec `R-V165-QC-SUGG-DEFENSE` (`host_tool_handlers.rs:1362-1458` body write path needs parity hardening). Tauri v2: `@tauri-apps/plugin-opener` exposes `openPath()` + `revealItemInDir()` (Context7-verified; "Reveal in Finder" maps to `revealItemInDir()`).

9. **`apps/desktop` workspace membership ‚Äî LOCKED: pnpm workspace sibling; Rust `src-tauri/` standalone.** `pnpm-workspace.yaml` already admits `apps/*`; `apps/desktop` joins as a pnpm workspace package (shared lockfile + `@42ch/nexus-contracts` workspace dep). `src-tauri/` stays Tauri-managed, **not** a root Cargo workspace member.

10. **CI runner + universal binary + artifact retention ‚Äî LOCKED.** `macos-13` x86_64 runner (deterministic V1.66 CI); install both Rust targets; attempt `--target universal-apple-darwin` if stable, else upload separate `aarch64-apple-darwin` + `x86_64-apple-darwin` artifacts (no hand-rolled `lipo` first). Retain **90 days**. Path filter: `apps/web/**`, `apps/desktop/**`, `packages/nexus-contracts/**`, `crates/nexus42/**`, `crates/nexus-daemon-runtime/**`, `pnpm-lock.yaml`, `package.json`, `pnpm-workspace.yaml`, `Cargo.toml`, `Cargo.lock`, `.github/workflows/**`.
