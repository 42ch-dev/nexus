## 0. Position

V1.36–V1.42 delivered the **novel-writing capability stack** (scaffold → multi-chapter → auto-chain → quality loop → World KB → multi-work desk → multi-volume → minimal agent tools). The largest gap for **ordinary authors** is **discoverability**: no end-user quickstart, CLI errors still assume harness knowledge, and completion/findings state is hard to read without reading specs.

V1.43 closes the **ongoing serial happy path** in human-facing docs and aligned CLI copy, improves author-visible quality-loop signals, and pays down author-path tech-debt without reopening agent/orchestration depth.

When this compass conflicts with a later locked implementation plan, the active plan wins for delivery batching only. Normative behavior changes still require spec updates before implementation.
