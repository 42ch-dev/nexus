## 2) V1.14 positioning

V1.14 is a **stability and determinism** cycle after OSS-forward V1.13: tighten template execution, test harness boundaries, and close or formally accept V1.13 QC follow-ups—**without** opening new major feature themes.

### Core principles

1. **No wire or schema breaking changes** without coordinated contracts release (out of scope for this compass).
2. **Medium-first**: `R-V113-002` (test harness API pollution) is the highest-priority closure candidate.
3. **Evidence-bound deferrals**: any item left open needs an updated `decision`, `target_date`, and `tracking_link` in `status.json`.
4. **Cross-repo honesty**: integration depth (DF-14 extensions) is optional in V1.14 unless explicitly added in a future compass revision.
