## 0. Grill decisions (PM-locked, 2026-06-23)

Canvas-driven clarify phase resolved 9 architecture decisions. Each row records the question, locked answer, and rationale.

| # | Question | Locked answer | Rationale |
|---|----------|---------------|-----------|
| Q1 | WASM runtime | **wasmtime** (Bytecode Alliance) | Strongest Rust integration, WASI p2 + Component Model, fuel/memory/timeout sandbox primitives. Aligns with ACP agent SDK potential runtime. |
| Q2 | Module distribution | **Embedded + User (two-layer).** `modules/` repo dir (non-workspace) + `build.rs` copy pipeline. No System layer. | System layer is for daemon ops presets only; compute modules are product features. Embedded = shipped with binary; User = `~/.nexus42/modules/`. |
| Q3 | ABI scope | **V1 envelope Schema.** 4 JSON Schema files in `schemas/compute/`. V2 full interop protocol. | Standard envelope avoids ecosystem fragmentation; module-declared fields keep flexibility. |
| Q4 | KB structured layer | **attributes (static) + state (dynamic) + computable flag.** per-BlockType JSON Schema. | attributes = immutable compute params (max_hp, base_atk); state = mutable runtime (current_hp, status_effects); computable = filterable marker. |
| Q5 | KB state granularity | **A: nested by block_type.** `state.character.current_hp`. | Same KeyBlock used by different module types; nesting avoids field name collisions. |
| Q6 | Security model | **Per-invocation sandbox.** Stateless pure function. Fuel + memory + wall-time limits. | Instance-per-call (μs creation). No cross-call state pollution. Reproducible compute. |
| Q7 | Work profile | **New combat-engine preset + narrative.compute Capability.** | Independent profile for user-triggered combat; reusable capability for embedded use in future presets. |
| Q8 | I/O schema | **Standard envelope (4-part output). Module-declared fields via manifest.json.** | state_delta, timeline_events, new_key_blocks, battle_report. Combat module has `casualties`; economy module has `market_prices`. |
| Q9 | Sample module | **basic-combat** in `modules/basic-combat/`. Embedded in wasm-host for integration tests. | Simple ATK−DEF resolution. Triple role: integration test, ABI validation, reference implementation. |
| Q10 | Repo structure | **`modules/` at repo root (non-workspace) + `crates/nexus-wasm-host/embedded-modules/` (include_dir!).** | Source → build.rs → compiled .wasm → embedded binary. Mirror of preset `embedded-presets/` pattern. |
| Q11 | Wire change impact | **key-block.schema.json extended** (state, computable). **4 new schemas** in `schemas/compute/`. | key-block change cascades through codegen → Rust + TS generated types → npm. Compute schemas are standalone — no ripple. |
| Q12 | Effort estimate | **XL iteration** (6 plans, 3-wave dispatch). Largest single-feature iteration since novel-writing (V1.36–V1.50). | New crate + 2 extended crates + schema changes + new preset + new sample module. Post-V1.60 dependency: execution gate must wait for V1.60 `iteration/v1.60` retirement. |

### 1.3 Wire contracts note

`wire_contracts_changed: true` (planned). Concretely:

- **key-block.schema.json**: extended — `KeyBlockBody` gains `state` and `computable` fields.
- **New schema family**: `schemas/compute/` — 4 new JSON Schema files for compute ABI contracts.
- **Codegen cascade**: schemas → `pnpm run codegen` → Rust generated types → TypeScript generated types → npm `@42ch/nexus-contracts` major bump.
- **No DB migration** — `body_json` is a TEXT column storing serde JSON; new fields serialize transparently.
- **Additive only** — no breaking changes to existing wire contracts. Existing KeyBlocks without `state`/`computable` are valid (fields are optional).

## 5. Open design items (resolve during implement)

These are tracked as per-plan design decisions, not iteration-level unknowns:

1. **State delta merge semantics** — exact merge algorithm for `+/-/set` operations (resolve in P3 T3).
2. **`computable` flag placement** — inside `body_json` (no migration) vs new column (indexed). Default: `body_json` for V1 simplicity (resolve in P1 T1).
3. **Module `manifest.json` exact format** — which fields are required vs optional (resolve in P2 T6).
4. **Host function ABI whitelist** — exact host function API surface exposed to modules (resolve in P2 T4).
5. **Computable BlockType mapping** — which BlockTypes get structured schema by default (resolve in P1 T2).
6. **build.rs strategy** — pre-compile + commit `.wasm` vs compile-on-build (resolve in P2 T7/T8).
