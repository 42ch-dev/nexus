## 0. Position

V1.39 closed the novel **automation loop**: auto-chain, quality loop, daemon continuation, and multi-chapter serial writing. The largest remaining **product completeness gap** for ordinary users is **World KB continuity** (DF-63):

> A user can bind a novel Work to a World during init, but cannot create Worlds, receive prompt-time World context, validate `world_refs`, or promote chapter facts back into the World KB.

V1.40 implements the five DF-63 slices defined in V1.37 P2 roadmap ([`2026-06-07-v1.37-novel-world-kb-roadmap.md`](../plans/2026-06-07-v1.37-novel-world-kb-roadmap.md)) and closes targeted V1.39 hygiene residuals in P4.

When this compass conflicts with a later locked implementation plan, the active plan wins for delivery batching only. Normative behavior changes still require spec updates before implementation.

## 2. Target User Journey

### 2.0 World-bound path (primary)

```text
nexus42 creator run start --idea "..." --init-preset novel-project-init
  → grill-me: create new World "Neon River"
  → creator world create --name "Neon River" --kind narrative
  → world_id bound; Works/<work_ref>/ scaffolded
  → auto-chain: intake → research → produce ch01 (World context in prompt) → review → persist
  → kb-extract promotes chapter facts to World KB under world_id
  → next chapter inherits updated World context
```

### 2.1 Bind existing World

```text
nexus42 creator world list
nexus42 creator run start --world-id wld_<uuid> --init-preset novel-project-init
  → world_id validated at PATCH; scaffold proceeds
```

### 2.2 Worldless path (unchanged)

```text
nexus42 creator run start --idea "..."   # no --world-id; grill-me: stay worldless
  → no World context block; README.md setting notes only
  → kb-extract skipped for World promotion
```

### 2.3 Validation UX

| Check | Timing | World-bound | Worldless |
| --- | --- | --- | --- |
| `world_id` exists | Work create/PATCH | Error + remediation | N/A (NULL ok) |
| `world_refs` ids valid | Outline | Warning | Warn if present, no fail |
| `world_refs` ids valid | Finalize | Error (unless `--force`) | Warn if present |
