## 0. Position

V1.27 closed the **local authoring loop** (CLI writes to `state.db`, four-domain persistent `assemble-moment`, `acp agent use`). V1.28 shifts to **author-facing context quality** and **Agent Host correctness** under pre-release breaking-change policy (no external users yet):

> **`platform context assemble-moment` is the single local assembly SSOT.** Stage0 / degradation / optional two-stage behavior are flags on that command, not a separate `assemble-local` subcommand.

> **World fork is a platform (community) capability — not a local client feature** (see [deferred-features-cross-version-tracker.md](../knowledge/deferred-features-cross-version-tracker.md) §3.0 PD-01).

When this compass conflicts with an active locked plan, the active plan wins for delivery batching.

## 2. Architecture (assemble-moment SSOT)

```text
nexus42 platform context assemble-moment
        │
        ├── build Stage0 (SOUL, memory, fragments) from local files
        ├── optional: degradation guard + TwoStage (local_first / cloud_enhanced; platform still deferred)
        │
        ▼
run_assemble_moment ──► assemble_moment(request, narrative, kb, knowledge)
        │
        ├── narrative: world state + timeline (branch/event aware)
        ├── kb: KbQuery (not list_by_world dump)
        └── knowledge: KnowledgeQuery with limit

nexus-daemon-runtime
        │
        └── GET /v1/local/narrative/*  (read-only; unchanged)
```

**Invariant**: No daemon POST for domain writes. No local world fork CLI.
