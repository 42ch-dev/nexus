## 6. Consumer invariant & risk notes

### 6.1 The token-name preservation invariant (Track A risk mitigation)

The DESIGN.md migration is **format-only**. The single most important invariant:

> **Token names are preserved verbatim.** `background-100`, `gray-1000`, `blue-700`, `gray-alpha-400`, `radius-control`, `copy-14`, `space-6`, … — every token name in the current MD-table DESIGN.md carries over to the YAML frontmatter unchanged. Only the *container format* changes (MD table → YAML map).

Verified consumers (direct read, 2026-06-27):
- `apps/web/src/index.css` — references tokens as `--color-<token>` (e.g. `--color-gray-1000: #111111;`); the `.dark` overrides swap values, not names.
- `apps/web/tailwind.config.ts` — `cv('<token>')` helper resolves to `var(--color-<token>)`; token names mapped 1:1.

**Consequence**: zero source edits to `index.css` or `tailwind.config.ts` are required for the migration to succeed. The only `index.css` edit is the header comment citing the new dark source file (`DESIGN.dark.md` instead of the inline MD table). If QC finds a name drift, that is a defect to fix before merge — not an expected change.

### 6.2 The `radius-*` → `rounded.*` naming remap (the one structured change)

The new `mstar-design-md` spec uses `rounded:` as the frontmatter key with semantic sub-keys (`sm`, `md`, `lg`, `full`). The current DESIGN.md uses `radius-control`, `radius-card`, `radius-popover`, `radius-fullscreen`, `radius-pill`. **Decision**: the frontmatter `rounded:` map uses the *current* semantic names as sub-keys to preserve the token identity — `rounded.control: 6px`, `rounded.card: 8px`, `rounded.popover: 12px`, `rounded.fullscreen: 16px`, `rounded.pill: 9999px` — rather than the spec's example `sm/md/lg/full`, because `index.css`/`tailwind.config.ts` reference `radius-control` etc. This is a `@architect` Phase 2b confirmation item: adopt the spec's `sm/md/lg/full` (and update consumers) OR keep the current semantic names under the `rounded.` namespace (zero consumer change). PM recommends the latter (zero consumer change) for V1.69; the rename to `sm/md/lg/full` can be a future token-hygiene iteration if desired.

### 6.3 Canvas Draft is non-binding on V1.70 scope

The Canvas Draft (Track B) is a **design de-risk artifact**, not a scope commitment.
