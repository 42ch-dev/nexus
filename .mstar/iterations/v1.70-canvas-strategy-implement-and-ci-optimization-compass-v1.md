## 6. Risk notes

### 6.1 React Flow in Tauri WKWebView (Track A risk)

React Flow is standard DOM/SVG/HTML pointer + keyboard work, but macOS WKWebView can differ from Chromium in gesture details (drag, wheel/pinch zoom, clipboard). A6 parity smoke is mandatory before merge. If a WKWebView blocker is found that cannot be resolved in-iteration, the canvas ships browser-tab-first with a documented Tauri known-issue (non-blocking — the browser SPA is the primary surface; Tauri is a convenience shell).

### 6.2 Path-filter regression risk (Track B risk)

`R-V168P0-QC1-W001` precedent: V1.68's path filter omitted `apps/nexus42/**` after CLI relocation, causing CLI-only PRs to silently skip desktop-bundle CI. B1 *narrows* the PR filter further (removes `crates/**` etc.) — the regression direction is "too narrow" (silently skipping a surface that affects the bundle). QC must verify: (a) no path that affects the Tauri bundle is dropped from the PR trigger; (b) the `push:main` + release triggers remain broad. The `ci.yml` non-packaging test gate is untouched, so Rust/TS test coverage is not reduced by B1; only PR-time desktop packaging coverage is intentionally narrowed.

### 6.3 A5 overlay depth is intentionally bounded (Track A risk)

The V1.69 Canvas Draft (`canvas-strategy-surface.md`) is the *input* to V1.70, not a frozen contract. Phase 2b locks A5 to **client-side projection from existing endpoints**: raw preset YAML is sufficient for the static Strategy graph, but the current session detail endpoint is summary-only.
