## 1. Scope / non-goals / out-of-scope

### 1.1 In scope

**Track A — Local API Surface Convergence (V1.67 lead slice)**

The headline sweep. Make the Local API surface consistent and consumable.

- **`R-V164-FE1-ORCH` — orchestration error-envelope consolidation** (low residual, **L effort**): `crates/nexus-daemon-runtime/src/api/handlers/orchestration/schedules.rs` (~1300 LoC) + peer handlers (`sessions.rs`, `presets.rs`) still return ad-hoc error tuples instead of the shared `ErrorResponse` envelope established in V1.64 (F-E1). Consolidate onto the canonical envelope. **Refactor approach is a Phase 2b `@architect` design item (§5 #1)** — mechanical sweep vs. extraction of a helper, and whether `NexusApiError` already covers these cases.
- **`R-V164-QC1-CASING` — error-code casing convention** (low): `local-api-surface-conventions.md` §3.2 mandates a casing convention (lowercase-snake vs runtime drift). Align runtime error `code` strings to the convention. **Couples to FE1-ORCH** (same files) — do together. **Wire-impact is a Phase 2b design item (§5 #2)**: if `ErrorResponse.code` values change, this is a wire-visible (pre-1.0 breaking) change.
- **`R-V163P3-SURF-003` — response array-key rename → `items`** (low residual, **pre-1.0 breaking wire change**): list responses use inconsistent array keys (`works` / `schedules` / `sessions` / `capabilities` / `chapters` …) vs the V1.65+ convention `items`. Rename to `items` across all list handlers + generated TS contracts + UI consumers. **This is the one wire-breaking change in V1.67** — `@architect` confirms the breaking-change classification + `@42ch/nexus-contracts` version bump in Phase 2b (§5 #3). Coordinates with `R-V164-QC1-S3-P1` (UI adaptation).
- **`R-V163P3-SURF-005` — sort params (F-F1)** (low, additive): no list query schema defines sort parameters. Add a consistent sort-param contract (additive; no breaking change). **Server-side sort design is a Phase 2b design item (§5 #4)**.
- **`R-V164-QC1-S3-P1` — F-P3 UI adaptation** (low): adapt `apps/web` consumers to the `items` array key (companion to SURF-003). Frontend track.
