## 1. Scope / non-goals / out-of-scope

### 1.1 In scope

**Track A — Essay Profile Closure (M effort, S-A)**:
- Essay profile scaffold (`work_profile: essay` — Works/<work_ref>/ layout: Outlines/ + Drafts/ + Logs/, distinct from novel Stories/).
- `essay-writing` embedded preset (intake → outline → draft → revise → finalize stage chain).
- Essay-specific 4-dimension quality rubric (thesis clarity / evidence support / coherence / ending takeaway) — advisory-or-blocking per plan evidence; explicitly NOT the novel 五问 gate.
- Completion detection (`Drafts/draft.md` frontmatter `status == finalized` AND intake complete → works.status = completed).
- Optional KB extraction (reuse `nexus.llm.extract` capability; single character/location context block per spec §6).
- Spec promotion: `essay-profile.md` Draft → Shipped (Feature line, post-implement evidence).
- `creator run essay-writing` CLI wiring (parity with game-bible/script run entry).

**Track C — Local API DTO Promotion + Surface Audit (M-L effort, S-B)**:
- Promote ~55 Local API DTOs from `crates/nexus-contracts/src/local/` (hand-written Rust) to `schemas/local-api/<resource>/` (JSON Schema → cross-language codegen):
  - **F0 core CRUD**: `works/`, `kb/`, `findings/`, `schedule/`, `workspace/`, `creators/` — request/response/response-list/error shapes.
  - **F1 orchestration sessions READ**: `orchestration/sessions/` — list + detail-status response shapes (NOT create/signal/cancel write side).
  - **F2 schedules + capabilities READ**: `orchestration/schedules/` (list) + `orchestration/capabilities/` (registry list).
  - **F3 preset_management full surface**: `preset-management/` — list/show/create/update/delete/validate custom preset shapes.
- Regenerate codegen output (Rust `generated::local_api::*` + TypeScript `packages/nexus-contracts/src/generated/local-api/`).
- Update `schema_drift_detection.rs` entries for new `local-api/` subtrees (`CheckMode::Strict`).
- Surface audit: pagination consistency, error-shape consistency, filter/sort param consistency across all promoted resources. Produce audit report; fix-waves for high-value inconsistencies (not mandatory all-fix-this-iteration).
- Update boundary docs (`schemas/AGENTS.md`, `schemas/README.md`, `schemas-directory-layout.md`, `schemas-external-consumer-boundary.md`).

**Track B — Residual Convergence (woven as P-last + fix-waves)**:
- Close all 7 open low-severity residuals (see §6 for the list).
- Target: zero open residuals at iteration close (or explicit V2.0+ deferral with rationale).

**P-1 — Prepare (M effort, `@project-manager`)**:
- T0: This compass (PM-locked via grill-me).
- T1: Plan stubs at `.mstar/plans/`.
- T2: Iteration index update (`.mstar/iterations/README.md` V1.63 entry).

### 1.2 Non-goals / out-of-sscope

- **agent-host sessions/operations/events(SSE) DTO promotion** — long-lived stateful connections + SSE stream shapes; defer to web-ui iteration when client transport is chosen. Roadmap batch.
- **OpenAPI spec generation + TS client SDK** — defer to web-ui iteration (C2); SDK design couples to client framework choice.
- **Essay cron scheduling parity** (D3) — single-artifact profile; per-Work cron stagger is over-engineering at this stage.
- **Novel 五问 gate for essay** — spec §5 explicitly authorizes a distinct lightweight check; essay is not chaptered fiction.
- **Platform/cloud features** — DF-13/16/55/59 remain platform-gated (PD-05); platform remains paused.
- **New Local API endpoints** — this iteration promotes EXISTING DTOs to cross-language schemas; it does not add new daemon routes. Endpoint additions belong with the web-ui iteration.

**`wire_contracts_changed: true`** (additive, non-breaking):

1. **~55 new schemas** under `schemas/local-api/<resource>/` → codegen → npm `@42ch/nexus-contracts` **minor bump** (new TS exports; no existing exports removed or renamed).
2. **Essay profile** additive enum values: `work_profile: "essay"` + any essay-specific `BlockType` if the implement plan introduces one (per spec §6, essay may reuse existing character/location BlockTypes — new BlockType not mandatory). Additive only.
3. **npm package**: minor version bump; `LATEST_SCHEMA_VERSION` may stay `1` (no envelope-level breaking change) or bump if codegen convention requires — implement plan decides.

## 5. Open design items (resolve during implement)

1. **Essay 4-dim rubric: advisory vs blocking?** Spec §5 says "advisory unless the V1.52 implementation plan explicitly chooses a blocking gate." Track A P2 decides based on evidence: game-bible/script both landed blocking completion detection; essay likely follows (blocking with `--force-gates` override), but the rubric dimensions' pass/fail thresholds need implement-time calibration. **Owner: P2 implementer.**

2. **Essay BlockType additions?** Spec §6 says essay may bind a single character/location KB context block. Decision: reuse existing `character`/`location` BlockTypes (no new enum value), OR introduce `essay_subject` if the rubric needs a distinct essay-scoped entity. Lean: reuse existing (closes no `R-V161P0-INFO-001` gap, but `environment` BlockType gap is a separate residual — see §6). **Owner: P2 implementer.**

3. **DTO promotion: per-resource subfolder vs flat under `local-api/`?** V1.62 precedent is `local-api/compute/` (flat module). For ~55 DTOs across 8 resources, per-resource subfolders (`local-api/works/`, `local-api/kb/`, etc.) keep the tree navigable. **Owner: P1/P3 implementer; align with `schemas-directory-layout.md` amendment.**

4. **DTO promotion: request/response naming convention?** Options: `<resource>-list-response.schema.json` / `<resource>-create-request.schema.json` (verbose but explicit) vs grouped `<resource>.schema.json` with `$defs` (compact but harder to codegen per-operation). Lean: per-operation files for clarity (aligns with `platform/http-bff/` precedent). **Owner: P1 implementer.**

5. **Surface audit: fix-in-iteration vs defer?** The audit may surface inconsistencies (e.g., some resources use `cursor` pagination, others `offset/limit`; error shapes vary). Decision per finding: fix if low-risk + within plan scope; defer as residual if structural. **Owner: P3 implementer + PM.**

6. **`nexus-contracts` local/ vs generated/ migration: re-export strategy?** Promoted DTOs leave a hole in `src/local/`. Consumers (daemon handlers, CLI) currently import from `local::`. Strategy: re-export from `generated::local_api::*` at the `local::` path (zero call-site churn) OR update all import paths (explicit but large diff). Lean: re-export for minimal churn. **Owner: P1 implementer.**
