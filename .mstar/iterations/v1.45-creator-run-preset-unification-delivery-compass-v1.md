## 0. Position

V1.44 shipped on-demand audit and review-master CLI surfaces ‚Äî but implemented them as **new hardcoded `RunCommand` variants**, continuing an anti-pattern: every author-facing preset adds Rust enum entries. V1.43 spec already claimed `creator run brainstorm` and flexible stage presets that **never existed** in code.

V1.45 corrects CLI information architecture:

1. **`creator run <preset_id> [<work_id>]`** ‚Äî generic preset dispatch; any embedded/user/system preset id; **no CLI preset whitelist**.
2. **`creator bootstrap`** ‚Äî sole composite entry (create Work + intake/produce chain); replaces `creator run start`.
3. **`creator works <sub>`** ‚Äî strictly **one business function per subcommand** (`inspire`, `reopen`, `resume-chain`, `reconcile-chapters`, ‚Ä¶).
4. **Hard delete** V1.33‚ÄìV1.44 bespoke `creator run` subcommands ‚Äî **no deprecated aliases** (pre-release; aligns V1.41 `run list/status` precedent).

North star: **adding a preset changes YAML + docs, not `run.rs`.**

When this compass conflicts with a later locked implementation plan, the active plan wins for delivery batching only. Normative behavior changes still require spec updates before implementation.
