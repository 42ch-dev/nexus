## 0. Position

V1.34 delivered linear FL-E stages and minimal Agent `nexus.*` tools. Users still hit **CLI cognitive load**: wrong entry commands, ambiguous `kb` vs `knowledge`, manual stage advances, and a first-run spec that assumes platform login.

V1.35 locked axis:

> **Understandable CLI + creator-centric entry + close critical security/correctness debt — without opening FL-D conditional routing.**

Onboarding means **command-surface clarity**, not a new `docs/QUICKSTART.md` in this iteration (listed as post-P2 follow-up).

When this compass conflicts with an active locked plan, the active plan wins for delivery batching.

### A.3 Detailed findings

#### UX-001 — Dual orchestration entry

**Evidence**: [cli.rs](../../crates/nexus42/src/cli.rs) `long_about` → `daemon schedule`; product path is `creator run` (V1.33+).  
**User impact**: New users start wrong command tree; FL-E invisible.  
**Plan**: P2 (help), P3 (ordering).  
**Spec**: [cli-command-ia.md](../knowledge/specs/cli-command-ia.md) §3, [creator-centric-entry-model.md](../knowledge/specs/creator-centric-entry-model.md) §2.

#### UX-002 — Root help text

**Evidence**: Same as UX-001; `--help` does not mention `creator run start`.  
**Plan**: P2.  
**Acceptance**: `long_about` mentions creator run + workspace init.

#### UX-003 — Sync top-level group

**Evidence**: Six groups in cli-spec §6.0B; grill decision: sync belongs under `platform`.  
**User impact**: Local-only users see `sync` as peer command; implies cloud requirement.  
**Plan**: P2.  
**Tracker**: PD-05 (cloud sync not short-term focus — IA still clarifies boundary).

#### UX-004 — Triple knowledge naming (KCA-003)

**Evidence**: [creator/mod.rs](../../crates/nexus42/src/commands/creator/mod.rs) — `Kb` + `Knowledge` subcommands; cli-spec §6.2E–§6.2F documents collision risk.  
**User impact**: Users think `creator kb` is “the knowledge base” for everything.  
**Plan**: P1 audit options; P3 implement chosen strategy (see cli-command-ia §3.2).  
**Deferred**: DF-42 (Local API KB redesign — separate, larger).

#### UX-005 — `run start` flag overload

**Evidence**: [run.rs](../../crates/nexus42/src/commands/creator/run.rs) — 8 flags on `Start`.  
**Plan**: P3 (defaults + hide advanced flags behind `--advanced` or subcommand profile — design in plan).  
**Severity rationale**: Power users OK; novices overwhelmed.

#### UX-006 — Intake → produce chain gap

**Evidence**: QC [v1.33 P2 qc3](../plans/reports/2026-06-04-v1.33-creative-brief-intake-preset/reports/qc3.md); `chain_novel_writing` default false.  
**Plan**: P4.  
**Deferred**: DF-53 (`--auto-chain` / stage auto sequencing).

#### UX-007 — Manual FL-E stage advances

**Evidence**: [creator-workflow.md](../knowledge/specs/creator-workflow.md) §5.1 — four explicit `stage advance` commands.  
**Plan**: P4 (partial DF-53: `run next` or smarter defaults).  
**Residuals**: R-FL-E-* (all **low** — not P0 blockers).

#### UX-008 — Missing `run next`

**Evidence**: `creator run --help` — subcommands: start, continue, list, status, stage (no `next`).  
**Plan**: P3 design / P4 optional ship.  
**Maps to**: DF-53 exploration.

#### UX-009 — First-run platform assumption

**Evidence**: cli-spec §7 User-first 11 steps include `platform auth login` and `sync pull` before creative work.  
**Plan**: P1 (revise §7.1 local / §7.2 platform).  
**Policy**: `platform_integration = paused`.

#### UX-010 — `run` discoverability

**Evidence**: `creator --help` lists 17 subcommands; `run` is mid-list.  
**Plan**: P3 reorder + optional `creator` default subcommand hint in docs.

#### UX-011 — Positional creator refs

**Evidence**: `creator use`, `pair`, `unpair` use positional args with comment "may become flag in future".  
**Plan**: P3 low priority; document in cli-spec if unchanged.

#### UX-012 — Preset maintenance vs creative entry

**Evidence**: DF-52 — no top-level `preset`; `system preset` vs `creator run` split.  
**Plan**: P1 spec clarity; P3 help cross-links.  
**Deferred**: DF-52 (confirmed OUT).
