This promotes the Outline+Timeline canvas surface from Draft to **Shipped β** (read + write + conflict + non-spatial alternate view).

- **A1 — Schemas + codegen (the additive wire-contract work)**: New Local API DTOs in `schemas/local-api/canvas/outline/`:
  - `OutlinePatchStructureRequest` / `OutlinePatchChapterRequest` / `TimelinePatchEventRequest` (each carries `work_id`, target ids, `base_revision`, and the patch payload).
  - `OutlinePatchResponse` (echoes `new_revision`, `validation_summary`, optional `side_effects`).
  - `OutlineConflictError` (mirrors V1.71 `StrategyConflictError` shape: `current_revision`, `node_id`, `conflicting_path`, recovery hint).
  - `OutlineValidationError` (mirrors V1.71 `StrategyValidationError`: structured validation summary).
  - All generated to TS (`apps/web/src/api-types/canvas/outline/`) and Rust (`crates/nexus-contracts/src/generated/canvas/outline/`). `pnpm run codegen` regenerates downstream; CI `verify-codegen` verifies committed artifacts.
> Architect lock note: A2 chooses the lowest-drift storage point: the outline markdown remains the outline structure SSOT, so its frontmatter carries the revision.
- **A2 — outlineRevision tracking + persistence (LOCKED)**: store the outline graph revision as an `outline_revision:` key in the outline markdown frontmatter (V1.65 atomic-save file pattern). Existing outline markdown without `outline_revision:` reads as revision `0`; the first successful patch writes `outline_revision: 1`. On every accepted patch, increment revision exactly once in the same committed outline markdown image; read endpoints (`GET /v1/local/works/{work_id}/outline`) expose the current `outline_revision`; write endpoints reject stale `base_revision` with `OutlineConflictError`. **Migration note**: a sidecar `.outline_revision` file is rejected because it can drift from the file it guards; a V1.72 `outline_revisions` DB column/table is rejected because it creates markdown/DB split-brain for a local single-author slice. A future DB table is deferred until multi-device sync, audit history, or high-frequency collaborative edits require it; if introduced later, it must backfill from outline markdown frontmatter rather than becoming a second source of truth.
> Architect lock note: A3 keeps the route set narrow and path-authoritative, mirroring V1.71 patch routes while respecting V1.65 chapter identity.
- **A3 — Local API routes (LOCKED)**: New routes following V1.64 cursor + ErrorResponse conventions and V1.71 patch-route convention:
  - `POST /v1/local/works/{work_id}/outline/patch` with `OutlinePatchStructureRequest` → `OutlinePatchResponse` (operations: `move_chapter`, `link_event`, `attach_to_volume`)
  - `POST /v1/local/works/{work_id}/chapters/{chapter_id}/patch` with `OutlinePatchChapterRequest` → `OutlinePatchResponse` (fields: title, slug, planned/actual word counts where supported, volume binding, and status using the existing V1.65 wire enum `not_started → outlined → draft → finalized`; `published` remains read/protected unless a future publish-retraction plan authorizes edits)
  - `POST /v1/local/works/{work_id}/timeline/patch` with `TimelinePatchEventRequest` → `OutlinePatchResponse` (operations: `add_event`, `remove_event`, `attach_event_to_chapter`, `link_foreshadow`)
  - All three requests carry `work_id`, `base_revision`, target identifiers, and a minimal patch payload; path identifiers are authoritative and must match any repeated body identifiers if present.
  - Error envelope: `OutlineConflictError` (409) + `OutlineValidationError` (422) + standard envelope (400/500).
> Architect lock note: A4 separates precondition conflicts (409) from domain validation (422) and aligns chapter status with shipped V1.65 vocabulary.
- **A4 — Daemon validation + atomic persistence (LOCKED)** (`crates/nexus-daemon-runtime/src/handlers/outline.rs`):
  - Parse + validate against outline semantics before accepting any patch:
    1. **ID existence**: `work_id` resolves to one Work; `chapter_id`, `volume_id`, `event_id`, `foreshadow_id`, `timeline_event_id` all exist in that Work's outline; path ids match body ids when duplicated.
    2. **Structural integrity**: a patch must not orphan chapters or create multiple parents; every chapter keeps exactly one valid Work/Volume containment path; volume containment remains valid (`belongs_to_volume` invariants); `ordered_after` edges remain acyclic; `foreshadows` edges reference existing event/chapter ids.
    3. **Status transition validity**: chapter status writes use the existing V1.65 lifecycle vocabulary (`not_started → outlined → draft → finalized`; `published` is protected/read-only in V1.72 patch routes). Invalid forward skips, reverse transitions, or `published` edits return structured validation paths unless a future explicit action/reason contract authorizes them.
    4. **Timeline reference resolution**: timeline events reference existing chapter ids; `realizes_event` references existing event ids; `foreshadows` target events that are scheduled at or after the source event.
    5. **Outline revision precondition**: `base_revision` equals the current outline markdown `outline_revision:` before the daemon writes; stale values short-circuit as 409 conflicts, not 422 validation failures.
  - On success: atomic outline markdown write (temp + rename + fsync, established V1.65 durability pattern) with the incremented `outline_revision:` in the same committed file image.
  - On conflict: short-circuit before write; return `OutlineConflictError` with current canonical state.
- **A5 — TS client methods (`NexusClient` in `apps/web/src/api/`)**: `outlinePatchStructure`, `outlinePatchChapter`, `timelinePatchEvent` methods. Refetch canonical outline after successful patch; expose `lastKnownRevision` for client-side conflict anticipation.
- **A6 — Canvas UI write affordances** (`apps/web/src/features/canvas/outline/`):
  - **Outline graph projection** (per `canvas-strategy-surface.md` §3.3 row "Work outline + timeline"): Work → Volume → Chapter → Scene/Beat as nested graph nodes; `belongs_to_volume` and `ordered_after` edges render correctly; volume lane is a sub-flow with `parentId`+`extent:parent` children; chapter cards show `wc`/`slug`/`status` from the outline; tipTap fragment preview is read-only on the canvas (full edit happens in inspector).
  - **Timeline lane**: separate React Flow lane (or sub-flow) showing events across chapters; events are positioned by chapter realization point; `foreshadow` edges link events that resolve later; `realizes_event` edges link chapter nodes to events they realize.
  - **Chapter inspector**: edit title/slug/wc/volume/status; submit `outline.patch_chapter`; show validation errors inline; show conflict modal on 409.
  - **Event inspector**: edit event metadata (title/description/realizes_chapter); submit `timeline.patch_event`; same conflict UX.
  - **Outline structure inspector**: drag-chapter-to-volume gesture triggers `outline.patch_structure` with `move_chapter`; link-event gesture triggers `link_event`; same conflict UX.
  - **Conflict modal UX** (reuse V1.71 pattern, outline-flavored copy):
    - **Headline**: "This outline changed while you were editing."
    - **Body**: "Nexus updated **{node_label}** to revision **{current_revision}** while you were editing **{field}**. Your change is still in the inspector." (PM: `{node_label}` adapts to the domain entity — chapter title for `outline.patch_chapter` conflicts, event title for `timeline.patch_event` conflicts, volume/outline for `outline.patch_structure` conflicts.)
    - **What changed**: a field-level summary of the canonical value that now differs from the user's last known revision.
    - **What you were about to do**: a field-level summary of the user's draft value for the same path.
    - **Actions**: **Use current** (primary/default), **Reapply my edit**, **Review side-by-side** (enabled only when the draft and canonical changes touch non-overlapping fields; disabled for same-field/path or timeline-event content conflicts).
    - **Cancel/close**: returns focus to the originating inspector with the draft intact.
  - **outlineRevision freshness indicator** in command palette: "Outline · revision {revision} · updated {relative time} — refresh now" (reuse V1.71 Strategy revision copy pattern; explicit refresh action added for parity with V1.71 "Strategy last refreshed 3 min ago — refresh now").
  - **Non-spatial alternate view** (Draft §4.4 accessibility): sortable chapter list (chapter title / status / wc / volume / updated) + sortable timeline event list (event / realizes_chapter / foreshadows / updated) — both available as toggle from the canvas toolbar. Both list views are the default for keyboard-only / screen-reader users.
- **A7 — Tauri WKWebView parity smoke**: V1.70 Strategy α + V1.71 Strategy β established the canvas-vs-WKWebView parity pattern; V1.72 extends it to outline drag-to-volume, timeline event pin, inspector modal, and conflict modal. Browser + Tauri shell parity check on outline/timeline interactions.
- **A8 — Accessibility (conflict UX + canvas)**: conflict modal announces the conflict and the current-vs-draft difference via an ARIA live region; focus moves into the modal and is trapped until the author selects an action; closing returns focus to the originating node or inspector control; provide keyboard-accessible paths for **Use current**, **Reapply my edit**, **Review side-by-side**, and **Cancel**; honor `prefers-reduced-motion` for modal entrance, timeline lane animation, and graph-edge animation; non-spatial alternate views are reachable and equivalent to canvas (Draft §4.4 invariant).
> Architect lock note: A9 must include a daemon-side concurrent writer regression because the revision precondition only matters if both UI and daemon/orchestration paths are covered.
- **A9 — Tests**: unit (validation + revision + conflict + structural integrity), integration (3 routes via NexusClient mock + canonical daemon), e2e (canvas edit → daemon save → refetch), regression (concurrent daemon + canvas writes cause 409 — proves outlineRevision catches stale writes from BOTH the canvas and daemon/orchestration-side writers, mirroring V1.71 `R-V152TA-S006` promise gap closure).
