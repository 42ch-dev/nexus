## 0. Position

V1.49 closed the novel quality loop (F-D findings chain + serial hardening) and shipped the F6 extended findings lifecycle + F/E narrative index runtime + author desk UX (intake re-trigger, reconcile `--dry-run`). The **next-mile product gap** has three angles:

1. **Novel-writing daily production rhythm** — the `novel-writing` preset runs only when an author actively invokes it. V1.39 shipped auto-chain but with **single-role sequential cadence**. The novels-system reference (§3.6.1) describes three-role staggering (`brainstorm 03/09/15/21` / `write 04/10/16/22` / `review :00/:30`) that lets authors set the work on autopilot and watch quality-graded chapters accumulate.
2. **Multi-volume continuity** — DF-62 shipped multi-volume PK and seed, but **automatic advancement** from "volume N complete" to "volume N+1 outline + seed" is not implemented. Authors currently must manually `creator works chronology advance` (or recreate the Work).
3. **World KB closure** — V1.40 shipped cross-Work World KB (DF-63) but the **authoring surface is read-only** (`creator world show/list`). V1.49 §1.2 OUT-B listed O5 (World KB auto-promotion) as deferred. Reference body externalization is static (DF-44) — no `creator kb rescan` exists.

V1.50 closes these in Track A (cron + chronology) and Track B (KB loop) with bounded P-last hygiene.

When this compass conflicts with a later locked implementation plan, the active plan wins for delivery batching only. Normative behavior changes still require spec overlay updates before implementation.
