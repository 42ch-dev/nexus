## 0. Position

V1.32 closed the **preset quality gate** (shared validator, embedded preset hardening, `SEC-V131-01`). V1.33 turns the shipped orchestration baseline into a **coherent narrative Work experience**: a creator can start from a small initial idea, pass through a **grill-me-style Creative Brief Intake**, continue adding inspiration and direction tweaks on the same **Work**, run **`novel-writing`** as the primary production preset, and close the loop with memory promotion, KB artifacts, and `assemble-moment` verification.

The locked axis is:

> **Work-first narrative experience loop + real quality gates + discoverable `creator run` entry.**

This is **not** a generic creator workflow iteration (see FL-E in deferred tracker). It is **not** full conditional routing / branching engine work (deferred post-V1.33).

When this compass conflicts with an active locked plan, the active plan wins for delivery batching.

## 2. Architecture

### 2.1 Work vs workspace vs World vs work index

```text
Creator
  └── workspace_slug (operational root, state.db)
        └── Work (work_id)          ← NEW V1.33 product container
              ├── long_term_goal
              ├── creative_brief (structured, post-intake)
              ├── world_id / story_ref (optional binding)
              ├── inspiration_log[] (append-only supplements)
              └── linked_schedule_ids[] / run_history
        └── creator kb --scope work   ← file index only (NOT Work)
World (nexus-narrative + nexus-kb)    ← narrative KB graph
```

Normative detail: [work-experience-model.md](../knowledge/specs/work-experience-model.md).

### 2.2 End-to-end user journey (happy path)

```text
creator workspace init + daemon start + acp agent use
        │
        ▼
creator run start --idea "<few words>"
        │
        ├── create Work (work_id)
        ├── schedule: creative-brief-intake (work_init)
        │     └── ACP multi-turn → structured creative_brief on Work
        └── schedule: novel-writing (work_continue)
              └── gathering → brainstorming → outlining → drafting
        │
        ▼
creator run continue <work_id> --note "<new inspiration>"
        │
        ├── append inspiration_log + core_context
        └── optional reflection-loop / memory-augmented (work_continue)
        │
        ▼
creator memory pending-list → review → LTM
creator kb queue-extract → kb-extract (knowledge_ingest)
platform context assemble-moment  (verify four domains)
```

### 2.3 Preset run intents (taxonomy)

| Intent | User meaning | Example presets |
| --- | --- | --- |
| `work_init` | May start a new Work (only presets with this intent) | `creative-brief-intake` (new), `novel-writing` (if policy allows direct start) |
| `work_continue` | Runs on existing Work; may append context | `novel-writing`, `memory-augmented`, `reflection-loop`, `research` |
| `knowledge_ingest` | Reference/KB pipeline | `research`, `kb-extract` (via queue) |
| `work_maintenance` | Work-adjacent but not narrative production | `soul-experience-refresh` |
| `system_maintenance` | Daemon/system | `_system.*` |

`creator run` **filters** presets by intent + Work state. `daemon schedule` remains the **power-user** surface.

### 2.4 Creative Brief Intake (grill-me)

- Implemented as **preset/orchestration** (not CLI wizard).
- Output: JSON-shaped **creative_brief** on Work (genre, tone, constraints, audience, open_questions resolved, non_goals).
- Minimum bar: intake must not proceed to `novel-writing` until brief passes schema validation (required fields non-empty).
- ACP session uses structured prompts; engine stores result via capability or schedule context hook (P2 architect decision).

### 2.5 `llm_judge` fix (V1.33)

Current gap: `StateCompositeTask` routes `exit_when: llm_judge` through `JudgeTask` stub reading `_judge_rule` only. V1.33 requires:

- Resolve `judge_capability` (default `judge.llm`) and `template_file` from state YAML.
- Invoke capability with schedule-injected `_creator_id` / `_session_id` (same boundary as V1.32 `SEC-V131-01` fix).
- Map GO/NOGO to `Continue` vs `WaitForInput` without conditional `next` (linear graph unchanged).

### 2.6 Memory closed loop

| Step | Surface |
| --- | --- |
| Session end | `session_capture` → `POST /v1/local/memory/pending-review` |
| List/dismiss | `creator memory pending-*` (shipped) |
| Promote | `creator memory review` → **`POST /v1/local/memory/review`** (P4 implements handler) |
| Fragments | `GET /v1/local/memory/fragments` for assemble-moment keyword pass |
| Preset fragments | `creator.write_memory` / `read_memory` (parallel pipeline) |
