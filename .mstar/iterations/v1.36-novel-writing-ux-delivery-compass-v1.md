## 0. Position

V1.33–V1.35 shipped Work, FL-E stages, CLI IA, and a minimal `novel-writing` preset (`gathering → brainstorming → outlining → drafting`). Ordinary users can **start** a Work, but the正文产出 path still behaves like a demo: flat `Stories/<story_ref>/` layout, no structured work status, no per-chapter outline/final pipeline, and sync scans risk treating metadata files as chapters.

V1.36 locked axis:

> **Complete the novel-writing正文产出 journey on generic Work — from project scaffold through one polished chapter — without platform publish and without legacy layout shims.**

Pre-1.0 policy: **no compatibility** with old `Stories/<story_ref>/` as the default artifact root. Specs, prompts, sync module, and tests migrate to `Works/<work_ref>/` in the same iteration waves.

When this compass conflicts with an active locked plan, the active plan wins for delivery batching.

## 2. User journey (target state)

```text
creator workspace init && daemon start && acp agent use
creator run start --idea "..." [--work-profile novel] [--world-id <world_id>] [--init-preset novel-project-init]
  → optional: novel-project-init preset (grill-me) if work_profile scaffold incomplete
       grill-me asks: working title, total_planned_chapters, World binding (existing world_id / create new World / worldless)
       on success: scaffold Works/<work_ref>/ dirs + seed work_chapters rows 1..N
  → creative-brief-intake (if not skipped)
  → novel-writing produce stage
        Works/<work_ref>/README.md
        Works/<work_ref>/Outlines/volume-outline.md (optional V1.36)
        Works/<work_ref>/Outlines/chapters/ch01-outline.md
        Works/<work_ref>/Outlines/foreshadowing.md    # V1.36 empty stub (P2)
        Works/<work_ref>/Outlines/event-index.md      # V1.36 empty stub (P2)
        Works/<work_ref>/Stories/ch01-<slug>.md  (draft → llm_judge 五问 gate → finalized)
        # work_chapters row updates per transition (DB SSOT); chapter frontmatter mirrors
        # for World-bound Works: World KB context block injected at each chapter prompt
creator run status <work_id>   # shows work_chapters + completion + (if bound) world: <name> (<world_id>)
creator run reconcile-chapters <work_id>  # manual: rebuild work_chapters from filesystem
# when completion criteria met (work_chapters 1..N all finalized + intake complete):
#   Work.status → completed; novel-writing schedules stop; message to user
```

Power users may still use `creator run stage advance` for FL-E stages; V1.36 focuses on **produce** stage正文 quality inside `Works/<work_ref>/`.
