# nexus-daemon-runtime — Daemon Runtime Library

`nexus-daemon-runtime` is the reusable runtime layer for daemon lifecycle, subsystem composition,
and local API transport. It is a **library crate only** — no standalone binary.

## Key Rules

- **Library only** — no `[[bin]]` section; this crate is consumed by `nexus42` CLI.
- **This is NOT an ACP Agent/Server** — the runtime is a local supervisor only, client-side.
- **Contract types:** shares generated types from `crates/nexus-contracts`. Do NOT hand-write duplicate DTOs.

## Architecture Reference

See `.agents/knowledge/nexus42-single-binary-daemon-runtime-architecture.md` — Sections 2.2, 3.1, 3.3
for the layering, crate structure, and workspace cleanup governing this crate.

## sqlx Compile-Time Macros (Mandatory)

All new sqlx queries **MUST** use compile-time checked macros (`sqlx::query!`, `sqlx::query_as!`,
`sqlx::query_scalar!`). **Do NOT** use runtime `sqlx::query()` or `sqlx::query_as::<T>()` for static SQL.

### When runtime `sqlx::query()` is acceptable

1. **DDL statements** (`CREATE TABLE`, `CREATE INDEX`, `ALTER TABLE`) — macros cannot validate DDL.
2. **PRAGMA statements** — no table schema to validate.
3. **Truly dynamic SQL** — query string constructed at runtime; each usage MUST include a
   `// SAFETY: dynamic SQL — compile-time macro not applicable.` comment.

### DATABASE_URL & `.sqlx/` conventions

- `DATABASE_URL="sqlite:.sqlx/state.db?mode=rwc"` (repo-relative, auto-creates).
- **Commit** all `*.json` metadata under `.sqlx/` (generated by `cargo sqlx prepare`).
- **Gitignore** `state.db`, `state.db-wal`, `state.db-shm`.
- After adding/changing queries: run `cargo sqlx prepare --workspace --all -- --all-targets`
  and commit updated `.sqlx/`.
